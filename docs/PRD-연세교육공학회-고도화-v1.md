# 연세교육공학회 플랫폼 고도화 PRD v1.0

> **작성일**: 2026-04-11
> **대상 프로젝트**: yonsei-edtech (https://yonsei-edtech.vercel.app)
> **스택**: Next.js 16 + Firebase + Vercel + bkend.ai

---

## 1. 현황 분석 (Executive Summary)

### 1.1 플랫폼 성숙도 평가

| 영역 | 완성도 | 상태 |
|------|--------|------|
| 회원 관리 (가입/승인/역할) | 95% | 안정적 운영 중 |
| 세미나 라이프사이클 (생성→출석→리뷰→수료증) | 90% | 핵심 워크플로우 완성 |
| 게시판/커뮤니티 | 85% | 기본 기능 완비, 고도화 여지 |
| 뉴스레터 | 70% | 제작 가능, **배포 기능 미완** |
| 활동 관리 (프로젝트/스터디/대외활동) | 80% | CRUD 완성, 협업 도구 부재 |
| AI 기능 (홍보 생성/챗봇) | 50% | 홍보 생성 작동, **챗봇 준비중** |
| SEO/검색 노출 | 40% | OG 태그만 설정, robots/sitemap 미구성 |
| 알림/소통 | 20% | 이메일 알림 일부만 구현 |
| 분석/리포팅 | 10% | 대시보드 기초 통계만 존재 |

### 1.2 핵심 강점
- 세미나 8단계 워크플로우 (생성→설정→홍보→등록→출석→리뷰→수료증→분석) 완전 통합
- 6단계 역할 체계 (admin/president/staff/advisor/alumni/member)
- AI 기반 홍보 콘텐츠 자동 생성 (보도자료/SNS/이메일/카카오톡/해시태그)
- QR 출석 체크 + 셀프 체크인

### 1.3 핵심 과제
학회 **일상 운영**에 필요하지만 현재 빠져 있거나 미완성인 기능들이 있음:
- 회원에게 소식을 **전달**하는 수단 (알림/이메일 배포)
- 학회 활동의 **성과를 측정**하는 도구 (분석/리포트)
- 회원 간 **소통과 참여**를 촉진하는 장치 (투표/설문/캘린더)

## 2. 고도화 로드맵

### Phase 1: 소통 인프라 (Priority: Critical)
> 학회가 회원에게 정보를 전달하고, 회원이 참여하는 기본 채널 확보

### Phase 2: 운영 효율화 (Priority: High)
> 반복 업무 자동화, 데이터 기반 의사결정 지원

### Phase 3: 참여 활성화 (Priority: Medium)
> 회원 체류시간과 재방문율을 높이는 기능

### Phase 4: 확장 및 대외 노출 (Priority: Low)
> 외부 유입, 검색 노출, 브랜드 강화

---

## 3. Phase 1: 소통 인프라

### 3.1 인앱 알림 시스템
**문제**: 회원 가입 승인, 새 세미나 등록, 게시글 댓글 등의 이벤트를 회원이 인지하려면 직접 사이트에 접속해야 함.

**구현 범위**:
- `Notification` 모델: `{ userId, type, title, message, link, read, createdAt }`
- 헤더에 알림 벨 아이콘 + 읽지 않은 개수 배지
- 알림 드롭다운: 최근 20개 표시, 클릭 시 해당 페이지 이동
- 알림 발생 트리거:
  - 회원 가입 승인/거절
  - 세미나 등록 확인
  - 내 게시글에 댓글
  - 새 공지사항 등록
  - 수료증/감사장 발급 완료
- "모두 읽음" 처리 버튼

**기술 고려사항**:
- Firestore `notifications` 컬렉션 + `onSnapshot` 실시간 리스닝
- 또는 bkend.ai 테이블 + React Query 폴링 (30초)

**성공 지표**: 알림 클릭률 > 40%, 사이트 재방문 주기 단축

---

### 3.2 뉴스레터 이메일 배포
**문제**: 뉴스레터를 제작할 수 있지만 회원에게 **발송하는 기능이 없음**. 발행해도 직접 방문하지 않으면 볼 수 없음.

**구현 범위**:
- 구독자 관리: 회원 가입 시 자동 구독 (opt-out 가능)
- 뉴스레터 발행 시 "이메일 발송" 버튼 추가
- 이메일 템플릿: 커버 이미지 + 목차 + "자세히 보기" 링크
- 발송 상태 추적: 발송중/완료/실패
- Resend Batch API 활용 (기존 인프라 활용)

**기술 고려사항**:
- Resend 일일 한도 확인 (무료: 100/일, 프로: 50,000/일)
- 대량 발송 시 큐 방식 필요 (Vercel Cron 또는 Edge Function)

---

### 3.3 세미나 사전 알림 (D-3, D-1)
**문제**: 등록한 세미나를 잊고 참석하지 않는 경우가 많음.

**구현 범위**:
- 세미나 등록자에게 D-3, D-1 이메일 리마인더 발송
- 이메일 내용: 세미나 제목, 일시, 장소, 온라인 URL
- Vercel Cron Job으로 매일 09:00 KST 체크
- 관리자 설정: 리마인더 활성화/비활성화

---

## 4. Phase 2: 운영 효율화

### 4.1 분석 대시보드
**문제**: 학회 활동 성과를 정량적으로 파악할 수단이 없음. 임원 보고나 사업 계획 수립 시 데이터 부재.

**구현 범위**:
- **세미나 분석**:
  - 월별/학기별 세미나 개최 수
  - 평균 참석률 (등록 대비 출석)
  - 리뷰 평균 별점 추이
  - 인기 시간대/요일 분석
- **회원 분석**:
  - 월별 신규 가입자 수
  - 역할별 회원 분포
  - 활동 회원 비율 (최근 3개월 내 세미나 참석 또는 게시글 작성)
- **콘텐츠 분석**:
  - 게시판 카테고리별 글 수
  - 뉴스레터 조회수
  - 가장 많이 본 게시글 Top 10
- 차트 라이브러리: `recharts` (이미 설치 여부 확인 필요)
- CSV/Excel 내보내기

**배치**: `/admin/analytics` 신규 페이지

---

### 4.2 회비 관리 고도화
**문제**: `/admin/fees` 페이지가 존재하지만 기본적인 수준. 학회 재정 관리에 필요한 기능 부족.

**구현 범위**:
- 학기별 회비 납부 현황 일괄 조회
- 납부/미납 필터 + 독촉 이메일 발송
- 납부 확인 (관리자 수동 체크 또는 영수증 첨부)
- 학기별 수입/지출 요약
- 회비 면제자 관리 (휴학, 졸업 등)

---

### 4.3 세미나 대기열 (Waitlist)
**문제**: 정원 초과 시 등록이 막히고, 취소자가 생겨도 빈자리가 채워지지 않음.

**구현 범위**:
- 정원 초과 시 "대기 등록" 버튼 표시
- 대기 순번 표시
- 기존 등록자 취소 시 자동 승격 + 알림
- 대기 등록 취소 가능

---

### 4.4 임원 교체 자동화
**문제**: 매 학기 임원진 교체 시 역할 일괄 변경이 번거로움.

**구현 범위**:
- "임원 교체" 위자드: 현 임원진 → alumni 일괄 전환
- 신임 임원 선택 → staff/president 역할 부여
- 변경 내역 로그 기록
- 회장 인사말 자동 갱신 프롬프트

---

## 5. Phase 3: 참여 활성화

### 5.1 설문/투표 시스템
**문제**: 세미나 주제 선정, 활동 방향 결정 등에 회원 의견을 수렴할 도구가 없음.

**구현 범위**:
- `Poll` 모델: `{ title, options[], multiSelect, anonymous, deadline, createdBy }`
- 게시글에 투표 첨부 가능
- 실시간 결과 표시 (막대 차트)
- 마감 시간 설정
- 익명/실명 선택

---

### 5.2 캘린더 통합
**문제**: 세미나, 스터디, 프로젝트 일정을 한눈에 볼 수 없음. 개인 캘린더 연동 불가.

**구현 범위**:
- `/calendar` 페이지: 월간 캘린더 뷰
- 세미나 + 활동 + 게시판 마감일 통합 표시
- 색상 코딩: 세미나(파랑), 스터디(초록), 프로젝트(보라), 대외활동(주황)
- "내 캘린더에 추가" 버튼 (Google Calendar / iCal 내보내기)
- `.ics` 파일 생성

---

### 5.3 챗봇 완성 및 고도화
**문제**: 챗봇이 "준비중" 상태로 방치됨. 신규 회원의 학회 이해를 돕는 도구로 활용 가치가 높음.

**구현 범위**:
- 기존 Q&A 프레임워크 활성화
- FAQ 데이터 자동 구축: 사이트 콘텐츠 기반 RAG
- 주요 질의 패턴:
  - "다음 세미나는 언제인가요?"
  - "회비는 얼마인가요?"
  - "가입 승인은 언제 되나요?"
  - "수료증은 어디서 확인하나요?"
- 답변 불가 시 문의 페이지로 안내
- "준비중" 라벨 제거

---

### 5.4 소셜 공유
**문제**: 세미나나 게시글을 외부에 공유하려면 URL을 직접 복사해야 함.

**구현 범위**:
- 세미나/게시글 상세 페이지에 공유 버튼 추가
- 카카오톡, 링크 복사, 트위터(X)
- 카카오 SDK 연동 (카카오톡 공유)
- OG 이미지가 이미 설정되어 있으므로 링크 공유 시 프리뷰 자동 표시

---

## 6. Phase 4: 확장 및 대외 노출

### 6.1 SEO 완성
**문제**: OG 태그만 설정됨. 구글 검색에 노출되려면 robots.txt, sitemap 필수.

**구현 범위**:
- `robots.txt`: 크롤링 허용 범위 설정 (admin 제외)
- `sitemap.xml`: 동적 생성 (세미나, 게시글, 활동 포함)
- 게시글/세미나별 구조화 데이터 (JSON-LD Event, Article)
- 이미지 alt 텍스트 보강

**기대 효과**: "연세교육공학회", "교육공학 세미나" 등 키워드 검색 유입

---

### 6.2 PWA 지원
**문제**: 모바일 접속 비중이 높을 것으로 예상되나 앱 경험이 없음.

**구현 범위**:
- `manifest.webmanifest` 완성 (이미 라우트 존재)
- 서비스 워커: 오프라인 폴백 페이지
- 홈 화면에 추가 프롬프트
- 푸시 알림 (Phase 1 알림 시스템 연계)

---

### 6.3 다크 모드
**문제**: `next-themes`가 이미 설치되어 있으나 활성화되지 않음.

**구현 범위**:
- ThemeProvider 활성화
- 헤더에 테마 토글 버튼 추가
- Tailwind CSS `dark:` 변형 적용
- 증명서/수료증 프리뷰는 항상 라이트 모드 유지

---

### 6.4 영문 페이지 (선택)
**문제**: 외국인 유학생/교환학생 참여 시 언어 장벽.

**구현 범위**:
- 주요 공개 페이지 영문 번역 (About, Seminars, Contact)
- `next-intl` 또는 URL prefix 방식 (`/en/about`)
- 관리자 UI는 한국어 유지

## 8. Phase 1 상세 구현 계획 (즉시 착수 권장)

### Sprint 1 (1주): 알림 + 뉴스레터 배포

**Day 1-2: 알림 시스템**
1. `notifications` 테이블 생성 (bkend.ai)
2. `notificationsApi` CRUD + `useNotifications` 훅
3. 헤더 알림 벨 컴포넌트 (드롭다운 + 배지)
4. 회원 승인 시 알림 생성 로직 추가

**Day 3-4: 뉴스레터 배포**
1. User 모델에 `newsletterOptIn: boolean` 추가
2. 뉴스레터 발행 페이지에 "이메일 발송" 버튼
3. Resend Batch API로 구독자 일괄 발송
4. 발송 상태 표시 (진행률 바)

**Day 5: 세미나 리마인더**
1. `/api/cron/seminar-reminder` API 라우트
2. D-3, D-1 자동 이메일 발송 로직
3. Vercel Cron 설정 (`vercel.json`)

### Sprint 2 (1주): 알림 트리거 확대 + 테스트

**Day 1-3: 추가 알림 트리거**
- 댓글 알림, 새 공지 알림, 수료증 발급 알림
- 알림 설정 페이지 (마이페이지 내)

**Day 4-5: QA + 배포**
- 전체 알림 흐름 테스트
- 이메일 발송 테스트 (Resend 테스트 모드)
- 프로덕션 배포

## 10. 리스크 및 제약 조건

| 리스크 | 영향 | 완화 방안 |
|--------|------|-----------|
| Resend 무료 한도 (100/일) | 뉴스레터 대량 발송 불가 | 유료 전환 또는 배치 분산 발송 |
| Vercel Cron 제한 (Hobby: 1/일) | 리마인더 주기 제한 | Pro 플랜 또는 외부 스케줄러 |
| Firebase 읽기 한도 | 알림 실시간 리스닝 비용 | 폴링 방식 + 캐시 활용 |
| 1인 개발 운영 | 유지보수 부담 | Phase별 점진 도입, 코드 단순화 우선 |

### 회원 관리
- 회원 가입 (이름/이메일/학번/소속/분야)
- 가입 승인/거절 (관리자)

- 프로필 편집 (사진/소개/연락처 공개범위)
- 비밀번호 변경/재설정 (Firebase + Resend 이메일)
- 회원 검색/디렉토리

### 세미나
- 세미나 CRUD (제목/일시/장소/연사/정원)
- 멀티 세션/프로그램 관리
- 연사 정보 (소속/직위/사진/유형)
- 커스텀 등록 폼 필드
- 타임라인 관리 (D-Day 기반 마일스톤)
- 참가 등록 (회원/비회원 게스트)
- QR 출석 체크 + 셀프 체크인
- 리뷰 수집 (별점 + 커스텀 질문, 참석자/연사/스태프 분리)
- 수료증/감사장 발급 (WYSIWYG 편집, 일괄 발급, PDF 다운로드)
- AI 홍보 콘텐츠 생성 (보도자료/SNS/이메일/카카오톡/해시태그)
- 자료 업로드/다운로드

### 게시판
- 5개 카테고리 (공지/세미나자료/자유/홍보/보도자료)
- 글 작성/수정/삭제
- 댓글
- 조회수 추적
- 검색 (제목/작성자)

### 뉴스레터
- 매거진형 이슈 제작
- 5개 섹션 타입 (특집/인터뷰/리뷰/칼럼/뉴스)
- 게시글 자동 임포트
- 커버 색상 커스터마이징
- 발행/임시저장

### 활동
- 3개 유형 (프로젝트/스터디/대외활동)
- 모집 상태 관리
- 참가 신청/승인
- 리더 지정
- 태그/위치 설정

### AI
- 홍보 콘텐츠 자동 생성 (OpenAI + Gemini)
- 챗봇 프레임워크 (준비중)
- Q&A 데이터베이스

### 기타
- 대시보드 (통계 카드 + 캘린더 + 활동 피드)
- 문의 접수/답변
- 사이트 설정 (인사말/연혁/조직도/분야 등)
- OG 메타태그/JSON-LD 구조화 데이터
