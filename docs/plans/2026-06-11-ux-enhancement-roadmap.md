> 6개 영역 병렬 분석(사용자 UX 심층 / 운영진 콘솔 UX 심층 / 기능 완성도 / 보안 / 참여·운영 / 코드 품질*) 종합 결과.
> *코드 품질 분석은 결과 유실로 리팩토링 단계 직전 재실행 예정.

## 1. 진단 요약

### 사용자 측 핵심 문제
1. **내비게이션 과밀**: 43개 기능이 5개 진입점(헤더/푸터/대시보드/마이페이지/하단네비)에 산재, 중복 진입점 다수
2. **대시보드 ↔ 마이페이지 역할 경계 불명확**: 활동 통계가 3곳 중복 노출
3. **빈 상태/로딩 표현 불일치**: 페이지마다 제각각 — 신규 회원 첫인상 훼손
4. **폼 데이터 유실 위험**: PostForm 제출 실패 시 작성 내용 소실 (draft 저장 없음)
5. **모바일 도달성**: 주요 기능 접근에 4~5탭, 하단네비 5개 외 기능은 헤더 펼침 필수

### 운영진 측 핵심 문제
1. **"오늘 처리할 일" 가시성 부재**: 콘솔 첫 화면이 stat 4종뿐 — 승인 대기/미발급 수료증/타임라인 overdue 확인에 3~4클릭
2. **세미나 운영 분산**: 신청·타임라인·출석·수료증이 6개 라우트로 분산, 신규 세미나 운영 최소 8단계
3. **일괄 처리 부재**: 활동 신청 승인 50명 = 50클릭, 현장 수동 체크인 개별 입력만
4. **자동 승인 미활용**: evaluateSignup 규칙 구현돼 있으나 토글 기본값 false (localStorage)
5. **모바일 현장 운영 불가**: 출석 체크 UI 데스크톱 위주

### 횡단 문제
- **소통 보드 고아화**: 알림·피드·리더보드·타임라인·다이제스트 어디에도 연결 안 됨
- **보안**: agent_workflow_runs rules 누락(Critical), 게시글 img src `javascript:` XSS, resolve-email enumeration
- **기능 완성도**: 알림센터 "(준비 중)" 오표기, 진도 미팅 목록 페이지 부재, 대시보드 위젯 저장 무피드백

## 2. 방향성 (원칙)

1. **추가보다 정리**: 신규 기능 동결, 기존 기능의 통합·노출·연결에 집중
2. **역할 분리**: 대시보드 = "오늘의 액션" / 마이페이지 = "내 기록·설정 관리"
3. **운영 1-화면 원칙**: 운영진은 콘솔 첫 화면에서 당일 업무를 모두 인지하고, 핵심 워크플로(세미나)는 한 페이지에서 완결
4. **일괄 처리 기본화**: 목록형 운영 작업엔 항상 다중 선택 + 일괄 액션
5. **신기능은 루프에 연결 후 종료**: 소통 보드를 알림/피드/다이제스트에 연결해야 "완성"으로 간주
