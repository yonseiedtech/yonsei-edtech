# 연세교육공학회 UX 분석·개선 보고서

- 작성일: 2026-06-15
- 역할: UX 디자이너 (정보구조·플로우·내비게이션·사용성·접근성)
- 범위: 읽기·분석 전용 (코드 수정/배포 없음)
- 스택: Next.js 16 (App Router) + Tailwind v4
- 독자: 교육공학 전공자 (UX 안목 高). UI 비주얼은 별도 담당 — 본 보고서는 **흐름·구조·사용성**에 집중.

## 0. 요약 (TL;DR)

서비스는 "입학→학위논문" 연구 여정을 따라가는 **대학원생 연구 성장 플랫폼**으로, 50+ 라우트·다수 도구가 빠르게 누적된 결과 **기능 밀도는 매우 높으나 발견성·온보딩·빈 상태·진입점 일관성이 그 속도를 따라가지 못한** 전형적 패턴을 보인다.

- **가장 큰 약점은 "첫 5분"**: 신규 가입자가 가입 직후 어디서 무엇을 시작해야 하는지 안내하는 온보딩 레이어(투어·체크리스트·웰컴)가 사실상 부재하고, 대시보드는 위젯 과밀로 첫인상이 "할 일 목록"이 아니라 "정보 벽"에 가깝다.
- **두 번째 약점은 "빈 상태"**: `EmptyState` 컴포넌트 자체는 다중 액션을 지원할 만큼 잘 설계됐으나, 실제 사용처에서 "기다리세요" 류 무행동 메시지가 많아 이탈 위험을 키운다.
- **강점은 "기반 체력"**: 역할 기반 IA 분기(공개/로그인/운영진), 스킵 링크·44px 터치 타겟·Escape 처리 등 a11y 기본기, 표준 프리미티브(PageHeader/PageContainer/BackButton/EmptyState/Skeleton)가 이미 갖춰져 있어 — **개선은 신규 구축이 아니라 "기존 자산의 일관 적용"** 으로 대부분 해결 가능하다.

## 1. UX 강점 / 약점 요약

### 1.1 강점
1. **역할 기반 IA 분기** — `Header.tsx`의 `isGroupVisible` (L59~69, L74~194)로 비로그인=홍보 중심(학회소개·학술활동·연구), 로그인=도구 중심(대학원 생활·커뮤니티)을 분리. 사용자 유형별로 다른 멘탈 모델을 제공하는 성숙한 설계.
2. **정체성 반영 메뉴 위계** — "연구 활동" 드롭다운 첫 섹션을 "📖 나의 연구"로 승격(L139~155)해 플랫폼 정체성(연구 동반자)을 내비게이션 위계로 표현.
3. **접근성 기본기 양호** — 스킵 링크(`Header.tsx` L464~469), 드롭다운 `aria-haspopup`/`aria-expanded`/Escape+포커스 복귀(L227~239, L264~277), 모바일 햄버거 44px 터치 타겟·`aria-controls`(L504~514), BottomNav `aria-current`·`role="dialog"`+`aria-modal`(`BottomNav.tsx` L112, L173).
4. **표준 프리미티브 보유** — `page-header.tsx`, `page-container.tsx`, `back-button.tsx`, `empty-state.tsx`(다중 액션 지원, L23/L80~98), `skeleton.tsx`, `inline-notification.tsx`, `actionable-banner.tsx`. 일관 UX의 토대가 이미 존재.
5. **모바일 2차 메뉴 보강** — BottomNav "더보기" 바텀시트(`BottomNav.tsx` L70~80, L111~158)로 헤더를 펼치지 않고도 9개 2차 기능 도달.

### 1.2 약점
1. **온보딩 레이어 부재** — 가입→첫 행동을 잇는 가이드 투어/웰컴/체크리스트가 없다. 7~30일 한정 배너성 안내는 만료 후 소실되어 "지각 신규(가입 후 늦게 첫 로그인)"를 놓친다.
2. **대시보드 위젯 과밀** — 미니멀 프리셋 미적용 시 10~13개 위젯이 동시 노출, "오늘 할 일"이라는 첫 목표가 정보 위계 속에 묻힌다.
3. **빈 상태의 무행동성** — "운영진이 등록하면 표시됩니다" / "예정된 모임이 없습니다" 류가 다수. 다음 행동(둘러보기·알림 설정·대안 활동)을 제시하지 않아 막다른 길이 된다.
4. **진입점/검색·필터 위치 불일치** — 아카이브(상단 중앙)·세미나(상단 좌측)·활동/네트워킹(검색 없음)으로 패턴이 페이지마다 달라 재학습 비용 발생.
5. **분석↔실행의 단절** — `/research` 차트에서 특정 키워드·연구설계를 클릭해 해당 논문 목록으로 드릴다운하는 연결이 약해, 통찰이 행동으로 이어지지 않는다.
6. **중복·분산 라우트** — 아카이브 7종 타입이 통합 진입점 없이 `/archive/[type]`과 `/archive/research-methods` 등으로 분산. `/calendar`(학술)와 `/mypage/calendar`(개인) 등 캘린더 진입이 분기.

## 2. 여정별 마찰점 · 이탈 위험

### ① 신입 온보딩
- **진입점 불명확**: 가입 직후 첫 화면이 곧바로 `/dashboard`(고밀도)로 향해, "환영 + 첫 3가지 할 일" 같은 완충 단계가 없다. 첫인상 이탈 위험.
- **동의(consent) 단절감**: `ConsentGate`/`consent` 흐름이 가입 폼과 분리돼, 회원가입을 마쳤다고 느낀 직후 다시 동의 게이트가 떠 "끝났는데 또?" 인상.
- **폼 진행감 부족**: 다단계 가입 폼에 단계 표시(스텝퍼)/남은 단계 가늠 장치가 약하다.
- **예상 외 팝업**: `UndergradInfoPrompt`·`SitePopupGate`(`layout.tsx` L172~173)가 첫 진입에 연쇄로 떠 인지 부하·놀람 유발 가능.
- **이탈 위험 종합**: 온보딩이 "안내된 여정"이 아니라 "게이트의 연속"으로 체감 → 첫 세션 완주율 저하.

### ② 대시보드 (/dashboard)
- **위젯 과밀**: 첫 진입에 위젯이 과다 노출돼 "오늘 무엇부터?"라는 핵심 질문에 답을 못 준다.
- **인사 반복·액션 유도 약함**: 헤더 인사와 위젯 내 인사가 중복될 수 있고, 가장 중요한 단일 다음 행동(Next Action)이 시각적으로 1순위를 점하지 못한다.
- **세미나 위젯 조건부 완전 숨김**: 데이터 없을 때 위젯 자체가 사라져, "다가오는 세미나가 없다"는 정보·"세미나 둘러보기" 기회까지 동시에 소실.
- **이탈 위험**: 정보는 많지만 "다음에 뭘 하면 되는지"가 약해, 둘러보다 떠나는 패턴.

### ③ 마이페이지 (/mypage — 잔디·연구활동·진단·캘린더)
- **탭·섹션 과밀**: 다수 탭·섹션이 한 화면 위계에 집약돼, 잔디(LearningStreak)·진단 같은 동기부여 핵심 위젯의 시각적 우선순위가 흐려진다.
- **잔디 발견성**: 학습 streak는 재방문 동기의 핵심인데, 진입 즉시 눈에 띄는 위치/빈 상태(첫 기록 유도) 설계가 약하면 "동기부여 장치"로 기능하지 못한다.
- **진단 결과의 상시 진입점 부족**: 진단 탭이 마이페이지 내부에 묻혀, 결과를 다시 보거나 재진단하려는 진입이 번거롭다.

### ④ 진단평가 (/diagnosis)
- **결과 후 다음 행동 약함**: 결과 리포트에서 약점→아카이브 개념 연결, 재진단 순환이 핵심 가치인데 이 "다음 한 걸음" CTA가 충분히 강하지 않다.
- **미완료/중단 처리**: 도중 이탈 시 재개(resume) 안내·진행률 저장이 명확하지 않으면 재시작 부담으로 이탈.
- **시작 진입점 분산**: 진단 시작이 마이페이지·대시보드·직접 URL 등 여러 곳에 흩어져 "지금 진단해보세요"라는 단일 강한 진입점이 약하다.

### ⑤ 아카이브 (/archive)
- **랜딩 정보 과밀**: 용어집·입문·이론·연구자료·가이드·평가·즐겨찾기 등 다수 섹션이 한 랜딩에 노출(`archive/page.tsx` L281~567)돼 "지금 뭘?"이 모호.
- **7종 타입 분산**: 기본 3종(concept/variable/measurement)은 `/archive/[type]`, 신규 4종은 별도 페이지로 분리(L413~512). 통합 필터/검색 부재로 같은 도메인이 분산 진입.
- **빈 상태 무가이드**: "검색 결과 없음"만 표시(L440~445), 0건 이유·대안 검색·인기어 제시 없음.
- **즐겨찾기 최하단**: 재방문자의 "빠른 돌아오기" 대상이 최하단(L604~630)에 위치.

### ⑥ 연구 (/research)
- **정보 과밀·내러티브 부재**: 히어로 직후 3탭(키워드/제목/계보)의 복잡 차트가 즉시 노출(`research/page.tsx` L213~389). "이 페이지에서 무엇을 얻나" 안내 약함.
- **차트 중복**: 다축 트렌드와 시대별 타임라인이 모두 시간축(L238~244, L285~326)이라 용도 구분이 모호.
- **분석↔논문 단절**: 차트 요소 클릭→논문 목록 드릴다운 부재(L391~425). 통찰이 행동으로 안 이어짐.
- **비회원 경계 불명확**: "비회원은 분석만" 안내가 상단이 아닌 하단 CTA에만(L405~423).

### ⑦ 네트워킹 (/gatherings, /network)
- **빈 상태 막다른 길**: "예정된 모임이 없습니다"만 표시(`gatherings/page.tsx` L109~112), 다음 모임 시점/대안 활동 안내 없음.
- **투표/조율 흐름 단절**: `isPollPending` 시 RSVP 대신 투표 UI가 조건부로만 노출(L264~305)돼 흐름이 끊김.
- **회비→참석 선제 안내 부족**: 회비 상태가 카드 중간 인라인(L286~298)에만, "미납→참석 불가" 선제 경고 없음.
- **네트워크 맵 온보딩 부재**: 프로필 미작성/회원 희소 시 그래프가 비어도 "프로필 완성하면 보입니다" 유도 없음(`network/page.tsx`).

### ⑧ 세미나 / 학술활동
- **세미나 상세 액션 혼재**: 신청·등록 폼·공간(LMS) 입장·출석확인서·후기가 긴 단일 페이지에 혼재(`seminars/[id]/page.tsx`)돼 "지금 내가 할 일"이 불명확.
- **LMS 진입 깊음**: `/seminars/[id]/lms`가 신청 완료 후 CTA에서만 노출(L545~550)돼 재진입 동선이 깊다.
- **활동 신청 경로 이중**: 목록(`ActivityPage`)과 상세(`ActivityDetail`) 양쪽에서 신청 가능해 일관성·예측성 저하.
- **활동 빈 상태**: "운영진이 등록하면 표시됩니다"(L642~648) — 대기만 유도, 대안/알림 없음.
- **주차 세션 진입 깊음**: `/activities/studies/[id]/weeks/[week]`가 목록에서 직접 도달 불가, 상세 경유 1클릭 추가.

## 3. 정보구조(IA) · 내비게이션 개선안

### 3.1 메뉴 위계
- **"내 활동" 허브 신설**: 마이페이지에 "내가 신청한 세미나/활동/모임"을 한 곳에 모으는 단일 허브. 현재 각 목록 페이지에 흩어진 "내 신청 상태"를 통합해 재방문 동선을 짧게.
- **대시보드 = "오늘", 마이페이지 = "나"** 역할 명확화: 대시보드는 "오늘/이번 주 할 일"에 집중, 마이페이지는 "누적 성장(잔디·연구·진단)"에 집중하도록 책임 분리. 현재 두 화면의 정보가 일부 겹친다.

### 3.2 중복·분산 라우트 정리
- **아카이브 통합 진입점**: 7종 타입을 단일 `/archive` 통합 필터(타입 칩 + 검색)로 묶어 분산된 `/archive/research-methods` 등을 한 화면에서 필터링. 분리 페이지는 딥링크로 유지하되 1차 진입은 통합.
- **캘린더 진입 명료화**: `/calendar`(학술 공개)와 `/mypage/calendar`(개인)의 차이를 라벨·진입 맥락으로 분명히. BottomNav 더보기의 "캘린더"가 어느 쪽인지 명시.
- **연구 진입 단순화**: "연구 활동" 드롭다운의 "나의 연구"(실행)와 "탐색·아카이브"(분석)가 잘 분리돼 있으나, `/research`(분석)·`/archive`(개념)·`/alumni/thesis`(논문)의 관계를 랜딩 한 줄 내비게이션으로 연결.

### 3.3 발견성
- **빈 상태를 발견 채널로**: 모든 주요 빈 상태에 `EmptyState`의 다중 액션(이미 지원, `empty-state.tsx` L80~98)을 적용해 "둘러보기/알림 설정/대안 활동" 제공 → 막다른 길을 발견 경로로 전환.
- **분석→실행 드릴다운**: `/research` 차트 요소(키워드·연구설계)에 논문 목록 딥링크 부여.
- **BottomNav 라벨 점검**: "커뮤니티"가 `/board/free`로 직행(`BottomNav.tsx` L56~60)하는데, 공지·인터뷰·리뷰 등 다른 게시판 발견성이 떨어질 수 있음 → 커뮤니티 허브 경유 고려.

## 4. 접근성 (a11y) 점검

### 4.1 양호 (유지)
- **스킵 링크**: `Header.tsx` L464~469 + `layout.tsx`의 `<main id="main-content">`(L165) 앵커 일치 — 키보드 사용자 본문 점프 가능.
- **드롭다운 키보드/포커스**: `aria-haspopup="menu"`·`aria-expanded`·Escape 시 닫기+버튼 포커스 복귀(`Header.tsx` L227~239, L269~277), 외부 클릭 닫기(L220~239).
- **모바일 터치 타겟**: 햄버거 `h-11 w-11`(44px, WCAG 2.5.5)·`aria-controls="mobile-nav"`·`aria-expanded`(L504~514). BottomNav 항목 `min-h-[56px]`(L175).
- **현재 위치 표시**: BottomNav·모바일 네비 `aria-current="page"`(`BottomNav.tsx` L173, `Header.tsx` L423).
- **바텀시트 다이얼로그**: `role="dialog"`+`aria-modal="true"`+`aria-label`(`BottomNav.tsx` L112).
- **포커스 링**: `focus-visible:ring` 패턴 광범위 적용.
- **표준 빈 상태/에러**: `inline-notification.tsx`로 알림 일관화 토대 존재.

### 4.2 점검·보완 필요
- **바텀시트 포커스 트랩**: `BottomNav.tsx` 더보기 시트가 `aria-modal`은 선언하나 실제 포커스 트랩(시트 내 Tab 순환)·열릴 때 첫 요소 포커스 이동이 구현됐는지 확인 필요. 미구현 시 키보드 사용자가 시트 뒤 콘텐츠로 빠져나감.
- **데스크톱 드롭다운 화살표 내비**: 메뉴 항목 간 ↑/↓ 이동·Home/End 등 `role="menu"` 권장 키보드 패턴(L281~313)이 완비됐는지 확인 — 현재 Escape·외부클릭만 명시.
- **아이콘 전용 버튼 라벨 전수 점검**: 헤더/바텀시트의 아이콘 버튼엔 `aria-label`이 있으나(로그아웃 L539, 닫기 L128), 각 페이지(세미나 QR·카드뉴스 슬라이더·삭제 아이콘 등)의 아이콘 전용 버튼은 누락 가능성 — 전수 grep 권장.
- **색만으로 상태 전달 금지**: 활동 배지(D-day/모집/진행), 잔디 색 강도, 타임라인 색 범례 등 색 의존 정보에 텍스트·아이콘 보강 여부 확인(대시보드 타임라인엔 범례가 있음 — 다른 영역으로 확대).
- **폼 라벨 연결**: `label.tsx` 존재하나, 다단계 가입/신청 폼에서 모든 입력에 `htmlFor`/`aria-describedby`(오류 메시지) 연결이 일관적인지 점검.
- **모달 포커스 일반화**: `dialog.tsx`/`alert-dialog.tsx` 기반 모달의 포커스 트랩·복귀가 라이브러리 차원에서 보장되는지(특히 커스텀 풀스크린 인터뷰 모달) 확인.
- **차트 대체 텍스트**: `/research` 다수 차트에 스크린리더용 요약 텍스트/표 대체가 없으면 데이터 접근 불가 — 핵심 통계의 텍스트 요약 병기 권장.

## 7. 결론

이 서비스의 UX 부채는 "설계 부재"가 아니라 **"빠른 기능 확장 vs 일관성 정비의 시차"** 에서 온다. 역할 기반 IA·a11y 기본기·표준 프리미티브라는 토대가 이미 있으므로, **(1) 첫 5분 온보딩, (2) 대시보드 Next Action 우선화, (3) 빈 상태 전수 액션화** 세 가지만 우선 처리해도 첫 세션 완주율과 재방문 동기에서 가장 큰 체감 개선을 얻을 수 있다. 나머지는 기존 컴포넌트를 사용처에 일관 적용하는 "정비형 작업"으로 점진 해소 가능하다.
