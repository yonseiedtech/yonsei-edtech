## 변경 파일 (3종)

### 1. `src/app/api/cron/content-draft-generator/route.ts`

**변경 내용**: Q&A 초안 블록 추가 (세미나 블록 뒤에 이어서 실행)

- 상수 추가: `QNA_BOARD_SCAN=20`, `QNA_TOP_PER_BOARD=5`, `QNA_MAX_BOARDS_PER_RUN=5`, `QNA_MIN_LIKES=1`
- `comm_boards(contextType=seminar)` 스캔 → 보드당 우수 Q&A(채택 or 좋아요≥1) 집계
- 멱등 docId: `${boardId}__qna-newsletter` (보드당 1회, 재실행 시 건너뜀)
- `content_drafts` 문서 필드:
  - `kind: "newsletter"`, `status: "pending"`, `source: "cron"`
  - `contentGapSource: "seminar_qna"` — 갭 대시보드 배지·다이제스트 집계 식별자
  - `payload`: 우수 Q&A 목록을 포함한 NewsletterSection JSON
- 실행당 최대 5보드 처리 (타임아웃 방어)
- 응답에 `qnaGenerated`, `qnaBoards` 추가

### 2. `src/app/api/cron/weekly-digest/route.ts`

**변경 내용**: 운영진 전용 갭 하이라이트 블록 추가

- `loadContentGapSummary()` 함수 추가:
  - `content_drafts` where `contentGapSource == "seminar_qna"` (단일 필드, 복합 인덱스 불필요)
  - client-side 필터로 `status == "pending"` 카운트
  - `search_misses` orderBy count desc limit 3
- `buildStaffGapHtml()` 순수 함수 추가:
  - Q&A 초안 검수 대기 건수 + 검색 실패 상위 검색어 → HTML 블록
  - `/console/content-drafts` 및 `/console/archive/content-gaps` 딥링크
- `usersById` 맵에 `role` 필드 추가
- `buildHtml()` 파라미터에 `staffGapHtml?: string` 추가
- 수신자 분류:
  - 기존 개인화 수신자: staff이면 갭 블록 추가
  - 비개인화 staff: genericEmails에서 제외 → 개별 발송(갭 블록 포함)으로 전환
- **회원 발송분에는 갭 블록 불포함** (운영진 role 판정: staff/president/admin/sysadmin)

### 3. `src/app/console/archive/content-gaps/page.tsx`

**변경 내용**: Q&A 자동 초안 생성 상태 배지

- `where` import 추가 (firebase/firestore)
- `qnaDraftQ` useQuery:
  - `content_drafts` where `contentGapSource == "seminar_qna"` 조회
  - pending 건수 클라이언트 필터
- "세미나 우수 Q&A" SectionCard 상단에 배지:
  - 대기 건수 > 0 일 때만 렌더 (`/console/content-drafts` 딥링크)

## 설계 결정

| 결정 | 이유 |
|---|---|
| Q&A 초안 docId: `${boardId}__qna-newsletter` | 기존 `${seminarId}__newsletter` 와 충돌 없이 보드 단위 멱등 보장 |
| `contentGapSource: "seminar_qna"` 필드 | 단일 필드 쿼리만으로 식별 가능 → 복합 인덱스 불필요 |
| 갭 블록 운영진 전용 | 회원 발송분 노이즈 방지 (계획서 명시) |
| 비개인화 staff → 개별 발송 전환 | BCC 묶음에 갭 블록을 넣으면 비운영진에게도 노출되므로 분리 |
| `loadContentGapSummary` graceful 실패 | 갭 블록 실패 시 발송 자체는 계속 진행 |
