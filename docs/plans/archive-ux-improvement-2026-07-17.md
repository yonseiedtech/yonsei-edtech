## 1. 정보구조(IA) 진단

### 1-1. 랜딩 과밀 — 세로 스크롤 15+ 블록, 점프 수단 없음
`src/app/archive/page.tsx` 는 위에서부터 ①통계 마법사 CTA ②연구방법 마법사 CTA ③Anchored Instruction 배너 ④통합검색 ⑤ArchiveStartHere(4스텝) ⑥ArchiveConceptRecommend(6카드) ⑦EduTechOverview ⑧진단평가 CTA ⑨즐겨찾기 ⑩연결 흐름 카드 ⑪그래프 CTA ⑫그룹1(용어집·입문 3카드) ⑬그룹2(라이브러리 3 + 가이드 7) ⑭사용 가이드 를 **단일 세로 스택**으로 나열한다. 스크롤이 매우 길고, 원하는 섹션으로 바로 가는 **인덱스/탭/스티키 서브내비가 없다**. "연결 흐름 카드(l.343)"·"사용 가이드 카드(l.761)"·"ArchiveStartHere"는 안내 성격이 겹쳐 **중복 인지 부하**를 만든다.

### 1-2. 두 리스트 UX 체계의 불일치
`/archive/[type]/page.tsx` 는 태그 AND 필터·3종 정렬·즐겨찾기만·(변인) 구인영역 필터를 제공하지만, `/archive/research-methods/page.tsx`·`foundation-terms/page.tsx`·`statistical-methods/page.tsx` 는 **검색어+kind/category 그룹핑만** 제공한다(정렬·즐겨찾기·태그 필터 없음). 같은 아카이브에서 "연구방법 리스트에는 왜 즐겨찾기 별이 없지?" 같은 기대 불일치가 생긴다.

### 1-3. 사전류 3종의 경계 모호 (개념 / 기초 용어 / AECT 용어)
"정의를 찾는" 목적의 리소스가 **개념(concept)·기초 용어(foundation-terms)·AECT 용어사전(terminology)** 3개로 병존한다. ArchiveStartHere(`ArchiveStartHere.tsx`) 는 2단계=기초 용어, 3단계=개념으로 안내하지만, terminology 는 별도 그룹(그룹1)에 있어 **"어떤 사전을 먼저 봐야 하나"** 가 첫 방문자에게 불명확하다. 세 리소스의 역할 구분(입문 용어 / 표준 역어 인덱스 / 이론·구성개념 심화)이 화면상 명시되지 않는다.

### 1-4. 진입점별 도달 경로 점검
- **홈/헤더/바텀내비**: `BottomNav.tsx`(l.76), `Header.tsx`(l.154) 모두 `/archive` 로만 진입 → 항상 과밀 랜딩을 거친다. 특정 계열(연구방법·통계)로의 직접 진입은 커맨드팔레트에만 있다.
- **커맨드팔레트**(`command-routes.ts`): "연구 · 아카이브" 그룹에 14개 라우트가 평면 나열 — 발견성은 좋으나 `/archive/concept`·`/archive/variable`·`/archive/measurement` **라이브러리 타입 직접 링크는 누락**(개념/변인/측정도구 리스트로 바로 못 감).
- **진단 오답**: `weak-concept-links.ts` 는 약점 개념→`/archive/concept/{id}`(없으면 terminology 폴백)·theory-map·flashcards 로 잇는 단일 루프가 잘 설계됨. 다만 오답→**연구방법/통계방법 보강** 경로는 없음.
- **연구 여정**: 개념 상세(`[type]/[id]/page.tsx` l.393)에서 `JOURNEY_STAGES` 역링크로 `/mypage/research` 로 나감. 반대로 여정→개념은 있으나 여정→연구방법/측정도구 딥링크는 확인되지 않음.

## 2. 콘텐츠 소비 UX

### 2-1. 상세 페이지 가독성·위계 — 대체로 우수
`/archive/[type]/[id]/page.tsx` 는 `max-w-[65ch]` 본문폭, `ConceptLinkedText` 자동 개념 하이라이트, 스티키 ToC(`ArchiveStickyToc`), 순화어·AECT 병기 배지, `scroll-mt-24` 앵커까지 갖춰 **읽기 경험의 기본기가 탄탄**하다. 연구방법 상세(`research-methods/[id]/page.tsx`)도 "쉽게 이해하기"(비유)·절차 스텝·강점/약점·가정 표·설계 다이어그램으로 위계가 좋다.

### 2-2. 크로스링크의 비대칭 — 개념↔연구방법/통계/용어 연결 단절
- 개념·변인·측정도구 상세는 서로(개념→변인→측정도구) + 관련 개념 + 관련 졸업논문 + (이론개념)theory-map 으로 촘촘히 연결된다.
- 그러나 **개념/변인/측정도구 상세에서 연구방법·통계방법·기초용어·AECT 사전으로 나가는 링크가 없다.** 예: "자기효능감" 개념을 본 대학원생이 "이 개념을 다룬 연구는 어떤 연구방법·측정을 썼나"로 이어가려면 상세 안에 경로가 없다(관련 졸업논문 카드로만 우회).
- 역으로 연구방법 상세→통계·졸업논문 은 있으나(`research-methods/[id]` l.606·652), **연구방법→해당 개념/변인** 링크는 없다.
- 결과적으로 6종 전체를 잇는 곳은 `/archive/graph` 뿐이고, **상세 페이지 수준에서는 concept-variable-measurement-thesis 클러스터에 갇혀** 있다. 그래프는 탐색엔 좋지만 "지금 읽는 항목에서 다음 학습으로" 잇는 문맥 링크를 대체하지 못한다.

### 2-3. 학습 흐름(읽기→암기카드→진단)의 부분적 단절
- 암기카드 저장 버튼(`flashcardsApi.saveFromConcept`)은 **개념 상세에만** 존재(`[type]/[id]/page.tsx` l.417). 변인·측정도구·기초용어·연구방법 상세에는 암기카드화 경로가 없어 "읽고→외우고" 루프가 개념에 한정된다.
- 진단→개념 추천(landing `ArchiveConceptRecommend`)은 잘 작동하나, **읽은 뒤 "이 개념으로 진단 문항 풀기"** 같은 역방향 마이크로 CTA는 상세에 없음(진단 CTA는 랜딩에만).

## 3. 발견성·재방문 동기

### 3-1. 규모·가치의 첫인상 약함
랜딩 히어로(`PageHeader`, l.203)는 카운트 없이 문구만 노출. 항목 수는 **개념/변인/측정도구 3개 숫자**만 라이브러리 카드(중반부 l.550)에 표시되고, 연구방법 16·통계 13·AECT 186 등은 산문 설명에만 있다. 첫 방문자가 "이 아카이브가 얼마나 크고 값진가"를 **한눈에 인지할 headline 지표(예: 총 N개 연구 자원)가 없다.**

### 3-2. 재방문 장치 = 즐겨찾기 단일, 접근 제한적
- 재방문 유인은 사실상 **즐겨찾기 하나**. 그마저 랜딩에서 로그인+즐겨찾기 보유 시에만 카드로 노출(l.314)되고, **전역 "내 아카이브(북마크)" 허브 페이지가 없다.** 다른 화면에서 저장 목록에 접근 못 함.
- **"이어보기/최근 본 항목" 기능이 전무**(코드 확인: archive 계열에 recentlyViewed/lastViewed 저장 없음). 방금 보던 개념으로 돌아오는 원클릭 동선이 없다.
- 즐겨찾기 별 토글이 **동적 리스트(`[type]`)에는 인라인**으로 있으나, 가이드 리스트(연구방법/통계/기초용어)에는 리스트에서 별 토글이 없어(상세에서만 가능) 저장 접근이 불균일.
- **"신규/업데이트" 신호 없음**: 최근 추가된 항목(개념·연구방법 등)을 알리는 배지·섹션이 랜딩에 없다("최신 추가순" 정렬은 리스트 내부에만 존재). 재방문 시 "새로 뭐가 생겼나"를 알 방법이 없다.

### 3-3. 두 개의 검색 표면 — 중복·상호 안내 부재
랜딩 인페이지 `ArchiveGlobalSearch` 와 전역 헤더 `GlobalSearch`(Cmd+K)가 병존한다. 인페이지 검색은 매 입력마다 7개 컬렉션을 클라이언트 로드(staleTime 5분으로 완화)하고, 결과는 **그룹당 6개 캡·"전체 보기" 링크 없음·키보드 내비 없음·매칭어 하이라이트 없음·aria-live 없음**. 두 검색의 역할 차이가 사용자에게 안내되지 않는다.

## 4. 모바일·접근성

### 4-1. 접근성 — 전반적으로 양호
`focus-visible:ring`, `aria-label`, `aria-pressed`(토글칩), `role="list"`(AECT), 그래프 노드 `aria-label`/`tabindex`, 필터 체크박스 label 연결 등 **기본 접근성 규율이 일관되게 지켜진다.** 타입 배지는 색상 외 텍스트 라벨을 병기해 색 의존 문제도 낮다.

### 4-2. 모바일 개선 여지
- **상세 ToC가 모바일에서 사라짐**: `[type]/[id]` 와 `research-methods/[id]` 는 `lg:grid-cols-[1fr_200px]` 로 ToC를 lg 이상에서만 렌더. 모바일에서는 **섹션 점프 수단이 전혀 없어** 긴 연구방법/측정도구 문서를 끝까지 스크롤해야 한다.
- **밀도 높은 표**: 연구방법 "가정 표"(`research-methods/[id]` l.500)는 `min-w-[640px]` + 가로 스크롤 — 모바일에서 5열(가정·설명·검정·SPSS·R)이 좁게 눌린다. 측정도구 메타는 `grid-cols-2` 고정(l.488)으로 모바일에서도 2열이라 값이 짧게 잘릴 수 있음.
- **AECT 사전은 표→아코디언 전환**으로 모바일 대응이 잘 됨(좋은 사례). 그래프도 70vh SVG + 컨트롤 collapsible + 핀치줌 지원으로 모바일 배려됨.
- **다크모드**: 상세·랜딩·리스트 전반에 `dark:` 변형이 촘촘히 적용됨. 다만 랜딩 그룹1은 배경틴트(`bg-slate-50/60`), 그룹2는 border-bottom만 — **그룹 시각 위계가 두 방식으로 갈려** 스캔성이 떨어진다.

## 5. 개선 백로그

- 개선: 상단에 `[사전·입문] [라이브러리] [연구·통계·글쓰기] [도구·그래프]` 스티키 앵커 탭 추가. "연결 흐름 카드"+"사용 가이드 카드"를 StartHere 로 통합하고 접이식(details)로 축소.

- 난이도: M

**H2. 상세 페이지 크로스링크 확장 — 개념/변인/측정도구 → 연구방법·통계·용어**

- 개선: 개념 상세에 "이 개념을 다룬 연구가 자주 쓴 연구방법·통계" 섹션(졸업논문의 methodIds 역집계) + "AECT 표준 용어에서 보기"(aectTerm 매칭 시) 링크. 연구방법 상세엔 "관련 개념/변인" 역링크.
- 기대: 상세→상세 회유(回遊) 상승, 논문 준비 맥락(방법·측정 선택)에서 이탈 감소.
- 난이도: M

**H3. "내 아카이브" 허브 + 최근 본 항목(이어보기)**
- 문제: 재방문 장치가 즐겨찾기 하나이고 랜딩에서만 접근. recentlyViewed 부재.
- 개선: `/archive/my`(또는 랜딩 상단 위젯)에 즐겨찾기 + 최근 본 5개(localStorage 또는 프로필 필드) + 진단 약점 개념을 모은 개인화 대시. 상세 진입 시 최근목록 갱신.
- 기대: 재방문 복귀 원클릭, 세션 연속성, 리텐션 강화.
- 난이도: M

**H4. 가이드 리스트 UX를 동적 리스트와 정합화(즐겨찾기·정렬 통일)**
- 문제: 연구방법/통계/기초용어 리스트에 즐겨찾기·정렬 없음(검색+그룹핑만) vs 동적 리스트는 풍부 (`research-methods/page.tsx` 등).
- 개선: 공용 `ArchiveListToolbar` 추출 → 검색·정렬·즐겨찾기만·(있으면)카테고리 칩을 전 리스트에 동일 배치. 리스트 카드에 별 토글 인라인화.
- 기대: 조작 모델 통일로 학습비용↓, 저장 접근 균일.
- 난이도: M

### Medium

**H5(M). 히어로 규모 지표 — "N개 연구 자원" 헤드라인 + 최신 추가 섹션**
- 문제: 규모·가치 첫인상 약함, 신규 신호 없음 (`archive/page.tsx` 히어로).
- 개선: 히어로에 총계(개념+변인+측정+연구방법+통계+용어) 카운트업 1줄 + "이번 달 새로 추가된 항목" 캐러셀(createdAt 기준).
- 기대: 첫 방문 가치 인지↑, 재방문 신규 탐색 유인.
- 난이도: S~M

**M6. 상세 페이지 모바일 ToC(앵커 점프)**
- 문제: ToC가 lg 이상에서만 노출, 모바일 섹션 점프 불가 (`[type]/[id]`, `research-methods/[id]`).
- 개선: 모바일에서 상단 고정 "목차 ▾" 드롭다운 또는 가로 스크롤 앵커칩으로 `tocSections` 재사용.
- 기대: 긴 문서 모바일 탐색성↑.
- 난이도: S

**M7. 인페이지 통합검색 고도화(전체보기·키보드·하이라이트·aria-live)**
- 문제: 그룹당 6개 캡, 전체보기 링크 없음, 키보드 내비·매칭 하이라이트·결과수 announce 없음 (`ArchiveGlobalSearch.tsx`).
- 개선: 그룹별 "N개 모두 보기"→해당 리스트 `?q=` 딥링크(이미 리스트는 `?q=` 수신 지원), ↑↓/Enter 내비, 매칭어 굵게, `aria-live="polite"` 결과 수.
- 기대: 검색→목록 전환 매끄러움, 접근성·효율↑.
- 난이도: M

**M8. 사전류 3종 역할 안내(디스앰비규에이션)**
- 문제: 개념/기초용어/AECT 사전 경계 모호 (`ArchiveStartHere.tsx`, 그룹1).
- 개선: 그룹1 상단에 "언제 무엇을 쓰나" 3열 비교(입문 용어=처음 / AECT=표준 역어 확인 / 개념=이론 심화) 마이크로 안내. 서로 간 상호 링크.
- 기대: 첫 방문자 리소스 선택 혼란 해소.
- 난이도: S

**M9. 암기카드화 대상 확장(변인·측정·연구방법·기초용어)**
- 문제: 암기카드 저장이 개념 상세 한정 (`[type]/[id]/page.tsx` l.417).
- 개선: 변인·측정도구·기초용어·연구방법 상세에도 "암기카드로 저장"(`saveFromConcept` 일반화). 카드 뒷면=정의/핵심 메타.
- 기대: 읽기→암기 루프를 아카이브 전체로 확장.
- 난이도: M

**M10. 진단↔학습 역방향 마이크로 CTA(상세→진단)**
- 문제: 진단 CTA가 랜딩에만, 상세에서 "방금 읽은 걸로 점검" 경로 없음.
- 개선: 개념 상세 하단에 "이 영역 진단 문항 풀기"(개념→domain 매핑) 링크. 진단 오답 보강에 연구방법/통계 경로 추가(`weak-concept-links.ts` 확장).
- 기대: 단일 학습 루프 완결, 진단 재응시 유도.
- 난이도: M

**M11. 커맨드팔레트 라이브러리 타입 직접 링크 보강**
- 문제: 팔레트에 개념/변인/측정도구 리스트 직접 항목 없음 (`command-routes.ts`).
- 개선: `r:archive-concept`·`r:archive-variable`·`r:archive-measurement` 3개 라우트 추가(키워드 포함).
- 기대: 과밀 랜딩 우회, 파워유저 도달 단축.
- 난이도: S

### Low

**L12. 랜딩 그룹 시각 위계 통일**
- 문제: 그룹1=배경틴트, 그룹2=border-bottom 로 스캔 단서 불일치 (`archive/page.tsx`).
- 개선: 섹션 헤더 스타일 토큰 1종으로 통일(아이콘+라벨+구분선).
- 기대: 스캔성·일관성↑.
- 난이도: S

**L13. 리스트 카드 "…더보기" 어포던스 정리**
- 문제: 카드 본문 "…더보기" 텍스트가 비클릭, 실제 링크는 하단 "연결 보기" (`[type]/page.tsx` l.560·585) — 이중 신호.
- 개선: 카드 전체를 상세로 링크하거나 "…더보기"를 실제 링크로. 중복 CTA 제거.
- 기대: 클릭 대상 명확화.
- 난이도: S

**L14. 측정도구 메타 모바일 1열 전환**
- 문제: 측정도구 메타 `grid-cols-2` 고정으로 모바일 값 눌림 (`[type]/[id]/page.tsx` l.488).
- 개선: `grid-cols-1 sm:grid-cols-2`.
- 기대: 모바일 가독성↑.
- 난이도: S

**L15. 그래프 발견성 — 상세에서 "이 항목을 그래프에서 보기"**
- 문제: 그래프는 6종 통합 유일 지점이나 랜딩 CTA로만 진입.
- 개선: 각 상세 하단에 "관계 그래프에서 이 노드 중심으로 보기"(`/archive/graph?focus={id}`) 링크(그래프에 focus 파라미터 수신 추가).
- 기대: 문맥→전역 탐색 전환, 그래프 활용도↑.
- 난이도: M

## 6. 요약

- **강점**: 상세 페이지 가독성(65ch·자동 개념링크·ToC·순화어/AECT 병기), 진단↔개념↔암기 단일 루프 설계, 접근성 규율, 그래프·AECT 아코디언의 모바일 대응.
- **최우선 개선**: ①랜딩 과밀 완화(서브내비) ②상세 크로스링크를 6종 전체로 확장 ③재방문 장치(내 아카이브+이어보기) ④두 리스트 UX 체계 정합화.
- **관통 원칙**: "지금 읽는 항목에서 다음 학습(방법·통계·진단·암기)으로" 잇는 **문맥 링크**를 상세 레벨에서 강화하고, **재방문 연속성**(최근 본·신규 신호)을 심어 일회성 탐색을 학습 여정으로 전환한다.
