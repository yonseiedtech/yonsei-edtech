## 1. 인수인계 자산 실측 (신임 운영진 워크스루)

### 1-1. 무엇이 있는가 (자산 지도)
| 자산 | 위치 | 직책 연결 방식 | 최신성 신호 |
|---|---|---|---|
| **인수 메모(handover)** | 조직도 각 직책의 `handover` 필드 (`OrgPosition.handover`, org_chart JSON blob) | 직책=포지션에 직접 저장 | **없음** — org_chart 전체 blob의 `updatedAt`만 존재, 필드별 갱신일 없음 |
| **업무수행철(업무노트)** | `handover_docs` 컬렉션 | `handover_docs.role === position.title` **(문자열 일치)** | `createdAt`/`updatedAt` 있음 → 직책별 최신 갱신일 산출 가능 |
| **인수인계 종합 뷰** | `/console/handover?tab=overview` (`OverviewView`) | 조직도 `handover` 있는 직책만 나열 | 없음 |
| **기수 리포트** | `/console/handover/report` | 조직도 메모 + `handover_docs`(term별) | 출력일만 |
| **운영진 교체** | `/console/handover?tab=transition` | `STAFF_ROLES` 하드코딩 드롭다운 | — |
| **조직도 학기 이월(R3)** | `cron/semester-advance` → `carryOverOrgChart` | 신학기 org_chart 문서 자동 복사 + staff 알림 | 멱등·비파괴 |

### 1-2. 반복 업무: cron이 대신하는 것 vs 사람이 해야 하는 것
- **cron 자동(32종)**: 학기 이월(semester-advance R3)·세미나/스터디/과제 리마인더·뉴스레터 발행·주간 다이제스트·연속실패 watchdog·멘토링 넛지·신입 활성화 시퀀스 등. → 신임자가 **손대지 않아도 도는** 축.
- **사람이 해야 하는 것(미자동)**: 회원 승인·문의 답변·피드백 처리·아카이브 검수·콘텐츠 초안 검토(= 콘솔 랜딩 pending 큐 + 사이드바 "검수 대기" 배너로 가시화됨). → 신임자가 **인수받아야 하는** 축.
- **인수인계 문서 작성 자체는 리마인더 cron이 없음** (아래 §3 G4).

## 2. 직책별 공백 지도 (핵심 산출물)

기준 직책 = 조직도 기본 시드(`DEFAULT_ORG_SEED`) + 업무노트 폴백 직책(`STAFF_ROLES`). 실데이터는 학회별로 다르므로, 아래는 **직책 유형별 구조적 공백**을 표로 정리(라이브 인스턴스는 감사 뷰가 실측 렌더).

| 직책(유형) | 콘솔 접근 권한(제안 매핑) | 업무노트 커버리지 | 인수 메모 | 반복 업무 문서화 | 권한 게이트 정합 | 공백 판정 |
|---|---|---|---|---|---|---|
| 학회장(president) | president → 콘솔 전체 | role="학회장" 매칭 | 조직도 메모 | 승인·교체·발행 총괄 | **OK** (president=4≥staff=3) | 메모/노트 미작성 시 공백 |
| 부학회장(vice_president) | staff → 콘솔 접근 | role="부학회장" | 조직도 메모 | — | **OK**(제안), 단 수동 승격 전엔 member=접근불가 | **승격 지연 시 접근 공백** |
| 조교(TA, direct_aide) | staff(제안) | role="조교" | 조직도 메모 | — | **위험**: 제안일 뿐 자동 아님 → member면 콘솔 진입 불가 | **권한 게이트 불일치** |
| 재학생 전공대표(direct_aide) | staff(제안) | 별도 role 탭 없음 | 조직도 메모 | — | 조교와 동일 | 권한 지연 + 노트 미분류 |
| 졸업생 대표(direct_aide) | staff(제안) | 별도 role 탭 없음 | 조직도 메모 | — | 조교와 동일 | 상동 |
| 자문위원/주임교수(advisor) | advisor(2) → **콘솔 진입 불가** | — | 조직도 메모 | — | advisor<staff → 인수인계 종합조차 열람 불가 | 열람 경로 공백(설계상 의도일 수 있음) |
| 총무·학술부장·홍보부장·대외협력부장·편집부장(STAFF_ROLES 폴백) | 조직도 미시드 → 배정 시 staff 수동 | role 탭은 있으나 조직도 포지션 부재 | **없음**(포지션이 없어 메모 필드 자체 없음) | 업무노트로만 | 포지션 미존재 → 커버리지 뷰에서 누락 | **조직도-업무노트 직책 불일치** |

### 2-1. 공백 4유형 요약
1. **노트 0건·메모 공백**: 배정됐으나 인수 메모도 업무노트도 없는 직책 → 교체 시 지식 0.
2. **낡음(90일+)**: 업무노트 최신 갱신이 90일 경과 → 임기 중 변화 미반영.
3. **핵심 반복 업무 미문서화**: 사람이 해야 하는 반복 업무(승인·검수·발행)가 특정 직책 노트에 없음.
4. **권한 게이트 불일치**: 직책은 배정됐는데 계정 역할이 아직 member → 첫 주 콘솔 진입 불가(수동 승격 필요).

## 4. 적용한 핫픽스 (경량 · 신규 컬렉션 0)

**파일: `src/features/handover/OverviewView.tsx`** (인수인계 종합 탭)

1. **D1 딥링크 수정**: "조직도 설정"·빈 상태 CTA 링크를 `/console/settings` → **`/console/org`**(실제 조직도 편집기)로 교정. 문구도 "운영진 설정(조직도)"으로 정정.
2. **D2 인수인계 준비도(공백 지도) 뷰 추가**: 기존 `handover_docs`·조직도 `positions`만 재사용(신규 컬렉션 없음).
   - **배정된 직책**(담당자 있는 포지션)만 집계 — 공석은 인수 대상이 아니므로 제외.
   - 직책별 판정: **공백**(메모·노트 모두 없음) / **노후**(업무노트 최신 갱신 90일 경과) / **준비**(그 외).
   - 상단에 `준비 N · 노후 N · 공백 N` 요약, 문제 직책은 카드로 나열(공백→노후 순) + **"노트 작성 →"**(딥링크 `?tab=worklog&role=…&compose=1`) · **"메모 편집"**(`/console/org`) CTA.
   - 공백 0건이면 "교체 시즌 준비 양호" 확인 메시지.
   - `print:hidden` — 인쇄/PDF 리포트에는 미포함(리포트는 기존 종합 유지).

> 이로써 "직책별 '인수인계 준비도' 표식(노트 0건 경고·미갱신 표시)"이라는 H3 핫픽스 정의를 충족. 구조 재구성(전용 페이지·데이터모델 변경)은 하지 않음.

## 5. 학기 전환(9/1) 인수인계 흐름 검증

조직도 학기 이월(R3) → 신임 배정 → 업무노트 연결이 실제로 이어지는지 추적:

1. **이월(자동·OK)**: `semester-advance` cron이 9/1 학기 키 전환 시 `org_chart:{신학기}` 부재면 직전(없으면 레거시) 조직도를 **복사 생성**, staff에게 인앱 알림 1회. 멱등·비파괴 확인.
2. **알림 딥링크(OK)**: 알림은 `/console/settings/org-chart`로 링크 → 이 경로는 `/console/org`로 리다이렉트(북마크 보존) → **정상 도착**. (D1과 달리 이쪽은 안전.)
3. **신임 배정(수동)**: 이월된 조직도는 **직전 담당자·메모가 그대로 복사**됨 → 신임자가 조직 설정에서 담당자를 교체해야 함. 교체 시 이전 담당자 승격 권한은 자동 강등 안 됨(안내 toast만) — 의도된 안전장치.
4. **업무노트 연결(끊김 위험)**: 업무노트는 `role=직책명` 문자열로 이월과 무관하게 유지되나, 직책명이 학기 간 바뀌면(D3) 매칭이 끊김. 또 **직전 임기 노트가 그대로 남아** 신임자가 "이번 임기 노트"와 구분하기 어려움(노트에 `term` 필드는 있으나 종합 뷰는 term 필터 없음 — 리포트만 term별).

**결론**: 이월·알림·도착 경로는 이어짐. **끊기는 지점은 (a) 직책명 불일치(D3), (b) 임기 경계에서 노트의 term 구분 부재** — 둘 다 구조 항목.

## 6. 구조 제안 (핫픽스 범위 밖 — 승인 필요, M4 report 자동 채움과 연계)

- **6-A. 직책명 표준/별칭 매핑(D3·D6)**: 조직도 시드와 업무노트 `STAFF_ROLES`·교체 드롭다운을 **단일 소스**(조직도 positions)로 통일하거나, `role` 별칭 매핑 테이블 도입. `TransitionView`도 조직도 포지션 기반으로 전환. → 노트-포지션 매칭 끊김 제거.
- **6-B. 배정↔승격 게이트 가시화(D4)**: 조직도 준비도 뷰에 "계정 역할 미승격(콘솔 접근 불가)" 배지 추가 + 첫 주 온보딩 안내. 자동 승격은 §4 외부 의존(임기/후임 정책) 확정 후.
- **6-C. 인수인계 넛지 cron(D5)**: 교체 시즌(D-N) 또는 노트 90일+ 미갱신 시 담당 직책에 "업무노트 갱신" 인앱 넛지. §4 외부 의존(발송 정책·SLA) 확정 필요.
- **6-D. M4 report 자동 채움 연계**: `handover/report`는 현재 조직도 메모 + `handover_docs`만 렌더, **임기 중 활동 요약(처리 대기 해소·발행·이벤트 운영 건수)은 자동 반영 없음**. 콘솔 랜딩 pending 집계·발행 로그 등 **기존 count를 재사용한 자동 요약 슬롯**을 리포트에 추가하면 인수인계 문서 작성 부하↓·완결성↑. (→ v12-M4 구현 항목)
- **6-E. 종합 뷰 term 구분**: 인수인계 종합/준비도 뷰에 임기(term) 필터 추가 → 이월 후 직전 임기 노트와 신임 임기 노트 구분.
