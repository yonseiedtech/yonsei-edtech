# v7-H5 학습 활동 kudos — 코호트 한정 응원 (구현 설계 · 2026-07-20)

> 계획 원문: `docs/plans/service-enhancement-plan-v7-2026-07-20.md` H5.
> 범위: 온보딩 "우리 기수"(`CohortSection`) 표면만. 대시보드 표면은 후속(다른 에이전트 색상 작업 중).

## 1. 개념

같은 가입 학기(코호트) 동기의 "이번 주 학습 활동"에 보내는 **양성 전용·1클릭 응원**.
순위·비교·게이미피케이션이 아니라 관계성(자기결정성) 지지 레버. 수십 명 규모라
팔로우 피드가 아닌 **코호트 한정**으로 소음을 막는다.

- 응원 소스(활동 사실): `streak_events`(학습 잔디 이벤트) — 이번 주에 활동 이벤트가 있는 동기만 대상.
- **학습 세부 수치 비노출**: 점수·달성 일수 등은 읽지도 보여주지도 않고, "활동이 있었다"는 사실만 사용.
- 대상은 **승인 회원(코호트 동기)** 중 `showInLeaderboard !== false`(잔디 공개 옵트아웃 존중)인 사람만.

## 2. 데이터 스키마 (신규 `kudos` 플랫 컬렉션)

`post_reactions` 의 "결정적 docId + setDoc" 패턴을 재사용. weekly_goal_records 는
rules 상 타인 읽기가 막혀(학습 수치 프라이버시) 활동 소스로 부적합 → 코호트 읽기가 열린
`streak_events` 를 활동 사실 소스로 채택하고, 응원 자체는 신규 `kudos` 컬렉션에 적재.

| 필드 | 타입 | 설명 |
|---|---|---|
| `id` | string | `` `${fromUserId}_${toUserId}_${weekKey}` `` (결정적) |
| `fromUserId` | string | 응원 보낸 회원 |
| `toUserId` | string | 응원 받은 회원 |
| `fromName` | string | 발신자 이름(알림·표시용 denorm) |
| `weekKey` | string | 활동 주의 월요일 `YYYY-MM-DD` (`currentWeekKey()`) |
| `type` | `"cheer"` | 응원 종류(현재 단일, 확장 여지 유니온) |
| `createdAt` | serverTimestamp | 생성 시각 |

- **주 1회 중복 방지**: docId 가 `{from}_{to}_{weekKey}` 라 같은 주 재전송은 create 가 아니라
  update 로 판정 → rules `update: false` 로 거부. 클라이언트도 `sentThisWeek` 로 선차단.
- 타입: `src/types/kudos.ts` (`Kudos`, `KudosType`). `src/types/index.ts` 재노출.

## 3. firestore.rules (신규 `kudos` 블록)

```
match /kudos/{docId} {
  allow read, list: if isAuthenticated()
    && (resource == null
      || resource.data.fromUserId == request.auth.uid
      || resource.data.toUserId == request.auth.uid
      || isStaffOrAbove());
  allow create: if isAuthenticated()
    && request.resource.data.fromUserId == request.auth.uid
    && request.resource.data.fromUserId != request.resource.data.toUserId
    && docId == request.auth.uid + '_' + request.resource.data.toUserId + '_' + request.resource.data.weekKey;
  allow update: if false;                 // append-only(중복 가산 차단)
  allow delete: if isAuthenticated()
    && (resource.data.fromUserId == request.auth.uid || isStaffOrAbove());
}
```

- **프라이버시**: 발신자·수신자·운영진만 read(코호트 전체 공개 아님). 클라이언트는 본인 발신분만
  `filter[fromUserId]==me` 로 조회하므로 list 규칙을 통과한다.
- **본인 create 만**: `fromUserId == auth.uid`, 자기 자신 응원 금지, docId 강제로 스푸핑·중복 차단.

## 4. 표면 (CohortSection.tsx)

기존 M1 구조(진행률 평균·동기 명단·버디 추천)는 그대로 보존하고 하단에 **"이번 주 학습 응원"**
서브섹션만 추가.

- `streakEventsApi.listSince(currentWeekKey())` → 이번 주 활동 회원 id 집합.
- `kudosApi.listSentByUser(me.id)` → 이번 주 이미 보낸 대상(client-side weekKey 필터).
- 대상 각각에 아바타+이름+응원 버튼. 이미 보냈으면 "응원함 ✓"(비활성), 전송 중 스피너.
- 대상이 0명이면 서브섹션 미노출(빈 상태로 방해하지 않음).
