### [C1] altName "자기조절학습" 오연결 — C급 · 즉시 수정 완료

- 마찰: 개념 "자기조절"이 별개 개념 "자기조절학습"(`concept:self-regulated-learning`)의 **정식명과 완전 일치하는 문자열**을 altName으로 보유. 위키 링크(`splitTextByConcepts`, greedy longest+stable order)가 본문의 "자기조절학습"을 잘못된 개념(자기조절)으로 연결할 여지.

## 2. 연구 도구 허브 실사용 마찰 (`/mypage/research/tools` · 18개 카드)

`src/app/mypage/research/tools/page.tsx` 5단계(주제탐색·선행연구·설계·집필·인용심사) 18개 카드 전수 점검.

| 점검 | 결과 |
|---|---|
| 도착지 라우트 실존 | 14개 루트(`page.tsx`) 전부 존재 — **죽은 링크 0** |
| `?tab=` 딥링크 정합 | explore·reading·design·proposal·reportdoc·writing·report·feedback **8개 전부** `MyResearchView`의 유효 `ResearchTab`으로 해석(`isResearchTab`/`normalizeResearchTab`) |
| `?focus=timer` 섹션 착지 | `focus-timer` 스크롤 앵커 존재 — 정상 |
| `/archive/concept/{id}` 딥링크 | `archive/[type]/[id]` 동적 라우트가 type="concept" 처리 — 정상(별도 concept 디렉토리 없음이 정상) |
| 진입 즉시 가치 | 카드마다 라벨+설명+상태뱃지(papers/writing/diagnostic 실데이터) — 양호 |
| 로그인 경계 | 허브 전체 `AuthGuard` — 전부 개인 연구도구라 타당 |

**결함 없음.** 도구 허브는 딥링크·도착지·상태 뱃지 모두 건전. 개념 상세(`archive/[type]/[id]`)도 미연결 빈 상태·오프라인 캐시 폴백·관련 논문/개념/자주 쓴 방법 크로스링크가 우아하게 구현돼 있어 별도 마찰 없음.
