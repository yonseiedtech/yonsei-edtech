## 2. 구현 내용

### 자동 최신성 장치 (과설계 없음 — 파일 기반)

**원칙**: Firestore·cron·신규 컬렉션 없음. `FEATURES` 배열에 `addedAt: string` 필드만 추가하는 최소 변경.

| 장치 | 동작 |
|---|---|
| `addedAt` 날짜 필드 | 모든 Feature 항목에 ISO 날짜("YYYY-MM-DD") 지정 |
| 자동 정렬 | `SORTED_FEATURES`: 모듈 레벨 정렬(addedAt 내림차순) — 새 항목 맨 위 추가 불필요 |
| NEW 배지 자동 만료 | `isNew()`: 60일 이내 항목만 "NEW" 배지 자동 표시, 60일 후 자동 사라짐 |
| 헤더 최종 갱신일 | `LAST_UPDATED = SORTED_FEATURES[0].addedAt` — 신규 항목 추가 시 자동 갱신 |

### 신규 항목 추가 방법 (운영 규약)
```ts
// FEATURES 배열에 추가만 하면 됨 — 위치 무관, addedAt 날짜가 정렬·배지 결정
{
  id: "feature-id",
  addedAt: "2026-09-01",   // ← 이것만 지정하면 자동 최신순·NEW 배지
  icon: SomeIcon,
  badge: "카테고리",
  // ...
}
```
