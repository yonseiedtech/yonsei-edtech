## 수정 파일

### 1. `src/app/api/console/cron-runs/trend/route.ts` (신규)

- **엔드포인트**: `GET /api/console/cron-runs/trend?days=<7~30>` (기본 14일)
- **읽기 상한**: `cron_runs` 최근 1000건 (`orderBy("startedAt", "desc").limit(1000)`)
- **집계 로직**:
  - `startedAt` ISO → KST(UTC+9) `YYYY-MM-DD` 변환
  - kind × 날짜별 `{ total, success, sumMs }` 집계
  - N일 슬롯 전체 생성 — 실행없는 날은 `successRate: -1`
- **열화(degraded) 판정**: 후반 절반 성공률 < 전반 절반 성공률 - 10%p (각 절반 3건 이상 시)
- **정렬**: degraded 우선 → overallSuccessRate 오름차순
- **신규 컬렉션 없음** — 기존 `cron_runs` 재사용

반환 스키마:
```typescript
interface DayBucket { date: string; total: number; success: number; successRate: number; avgMs: number; }
interface KindTrend  { kind: string; days: DayBucket[]; overallSuccessRate: number; degraded: boolean; }
{ ok: true, trends: KindTrend[], days: number }
```

### 2. `src/app/console/cron-logs/page.tsx` (수정)

추가 요소 (기존 섹션 보존):
- lucide-react import에 `TrendingUp`, `TrendingDown` 추가
- `DayBucket`, `KindTrend` 인터페이스 추가
- `KindTrendRow` 컴포넌트 — SVG 스파크라인 (barW=10px, barGap=2px, h=32px)
  - 성공률 100% = `fill-success/80`
  - 80~99% = `fill-warning/70`
  - <80% = `fill-destructive/70`
  - 실행없음 = `fill-muted-foreground/20`
- `CronTrendSection` 컴포넌트
  - `useQuery(["console","cron-trend", windowDays])` → `/api/console/cron-runs/trend`
  - 14일 / 30일 전환 버튼
  - degraded kind 배너 (TrendingDown 아이콘)
  - M5(임계 경보) 범위 외 메모 문구 포함
- `<CronTrendSection />` 렌더 위치: `<CronStatusSection />` 직후
