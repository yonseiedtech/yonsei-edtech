# '연구 설계' 단계·페이지 스펙 (2026-07-13)

사용자 요구: 연구 여정에서 "윤리·도구 확정" 단계를 제거하고, 연구 보고서(3)와 연구 계획서(4) 사이에 **연구 설계** 단계·페이지를 신설. 계획서·논문의 연구방법 파트 상당 부분을 미리 생각·작성해 보게 하는 도구. /console/research에 조회 탭 추가. 여정 전체가 논문 집필 파이프라인으로 연결.

## 1. 여정 개편
- 새 7단계: 주제 탐색 → 문헌 고찰 → 연구 보고서 → **연구 설계(신설, 2~3학기)** → 연구 계획서 → 논문 집필 → 심사 대응
- "윤리·도구 확정" 제거 — 그 내용(연구윤리 체크리스트 링크·도구 확정)은 연구 설계 단계의 하위 요소로 흡수 (지원 도구 점선 카드 유지 여부는 기존 패턴 따름)
- 산출물 파이프라인 문구 갱신: 보고서 → **설계** → 계획서 → 논문

## 2. 연구 설계 페이지 (작성 도구) — 섹션 구조
교육공학 학위논문 연구방법(III장) 전형 구조를 역산한 8섹션. 각 섹션 = 안내(무엇을 왜) + 작성 필드 + 관련 자산 연동.

| # | 섹션 | 필드 | 연동 자산 |
|---|---|---|---|
| 1 | 연구 유형·접근 | 양적(실험/준실험/조사/개발·효과분석)·질적(사례/근거이론/현상학/면담/델파이)·혼합 선택 + 선택 이유 | research-methods(kind) — 선택 시 이후 섹션 프리셋 분기 |
| 2 | 연구 모형 | 기존 연구 모형(research-model) 연결 or 새로 만들기 유도, 변인 관계 요약 자동 표시 | research_models·VariableSyncPanel·research-question-generator |
| 3 | 연구 대상 | 모집단·표본 크기와 산정 근거(검정력/포화)·표집 방법·참여자 보호(동의·익명화) | 연구윤리 체크리스트 링크(기존 윤리 단계 흡수) |
| 4 | 연구 절차 | 선택한 연구방법의 표준 절차 프리필(수정 가능) — 단계별 계획 텍스트 | research-methods.procedures (14+2종) |
| 5 | 연구 도구 | 양적: 아카이브 측정도구 선택(문항수·신뢰도 자동 표시) + 자체 개발 계획(타당화 절차) / 질적: 면담 프로토콜 개요·델파이 패널 구성·질문 초안 | archive_measurements picker(AlumniThesis 패턴) |
| 6 | 프로그램 개발 설계 | 효과분석(개발) 연구일 때만 노출: 처치 프로그램 개요·회기 구성·ADDIE 단계 체크 — 프로그램 개발 가이드 딥링크 | program-development-guide |
| 7 | 자료 수집·분석 | 양적: 수집 절차 + 통계방법 선택(가설별) / 질적: 수집(면담·관찰·문서)·코딩/주제분석·신뢰성 확보(삼각검증·member check) | statistical-methods 가이드 |
| 8 | 연구방법 초안 미리보기 | 1~7 작성 내용을 "III. 연구방법" 아웃라인(1. 연구 대상 / 2. 연구 도구 / 3. 연구 절차(+프로그램 개발) / 4. 자료 분석)으로 자동 조립 — 복사·계획서/논문으로 이어쓰기 안내 | 계획서·논문 작성 탭 연결 |

- 저장: 새 컬렉션 `research_designs` (본인 read/write, staff read — research_proposals 권한 패턴 준수). 자동 저장(디바운스) + 섹션별 완성도 표시(여정 진행 상태 연동).
- 빈 상태·단계별 도움말은 기존 연구 인터뷰 톤(친절·한 화면 하나의 질문 지향은 아님 — 폼형이되 섹션 아코디언).

## 3. 파이프라인 연결
- 여정 카드(3 보고서)→"보고서의 변인·이론을 설계로 가져오기", (5 계획서)→"설계에서 연구방법 채워 넣기" 안내 배지
- /console/research 탭 "연구 설계" 추가 — 회원별 research_designs 조회(기존 4트랙 탭 패턴), MiniProgress(섹션 완성도)
- 진행 판정: 여정의 '연구 설계' 단계 상태 = research_designs 문서 존재·완성도 기준

## 4. 구현 순서
① types + bkend api + firestore.rules ② 작성 페이지(/research 하위, 기존 연구 트랙 라우트 관례) ③ 여정 단계 정의 수정 ④ 콘솔 탭 ⑤ 미리보기 조립 로직(순수 함수, 테스트 가능)
