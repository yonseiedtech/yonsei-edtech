## A. 3관점 교차 최우선 갭 (2개 관점 이상 합치)

### A-1. 해커톤 "코드는 95%, 운영은 30%" — 실전 3주 전 비대칭 (PM GAP-J·K·L + UX IA-5)
- PM: 심사위원·루브릭·장소("추후 공지" 잔존)·모집 콘텐츠 X1~X4 전부 미결, 리마인더 발송 정책 4문서째 이월, 개강 콘텐츠 0건.
- UX: `/hackathon` 자체가 내비 어디에도 없는 고아 페이지 — 대시보드 배너가 유일 진입.
- **종합 판정**: 남은 3주의 병목은 코드가 아니라 **사람의 결정과 콘텐츠**. 코드 측 잔여는 진입점 추가(S)·합류 숏컷(S)뿐.
- **액션**: ① 해커톤 운영 런북 1장(X1~X4 오너·기한 역산표) → **사용자 결정 안건** ② 학술 활동 메뉴/더보기 시트에 행사 기간 조건부 "해커톤" 항목(S) ③ 합류 신청 숏컷(PM GAP-B, S).

### A-2. 신입 첫 24시간 여정 — 리허설 0회 + 착지 불일치 + 검색 유입 공백 (PM GAP-E·F + UX FLOW-1)
- PM: 3개 계획서가 연속 지목한 신입 종단 리허설이 여전히 미실행. OG metadata 16/299(5.4%) — 검색·공유 유입 구조적 손실.
- UX: 가입 완료 화면이 "인지디딤판 안내"를 약속하고 대시보드로 보내는 기대 불일치, 수동승인 대기자에게 알림 안내 부재.
- **액션**: ① 신입 종단 리허설(8/18 전, [기대 vs 실제] 방법론 + **발견 결함은 같은 게이트 내 커밋 완결 조건**) ② 가입 완료 착지/문구 정합(S — 리허설에 포함) ③ OG High 누락 ~10개 공개 페이지 정정(S).

### A-3. 학습 루프의 마지막 마디 단절 — 만든 루프를 세지도, 잇지도 않음 (PM GAP-N·G + UX FLOW-2)
- PM: 진단→암기카드 전환율 등 루프 헬스 집계 뷰 부재 — 8월 초 첫 실데이터 도래 전 준비 필요. 대시보드 신호 경합은 v15 이연 유지.
- UX: 진단 리포트에서 오답 암기카드 복습으로 가는 주요 CTA 불명확 — 루프가 UI에서 끊김.
- **액션**: ① 진단 리포트 "오답 카드 복습" CTA(S) ② 루프 헬스 집계 뷰(기존 컬렉션 count, S~M) ③ v15 편성 규칙 "데이터 없이 High 배정 불가" 명문화.

### A-4. 모바일 1차 동선의 구조 결함 (UX IA-1·2·3·A11Y-1 + UI 터치 관련)
- UX: BottomNav "연구활동" 탭이 개인 연구가 아닌 학회 분석으로 연결(+활성 상태 누락), 대시보드가 1차 탭에 없음, /directory 진입점 전무, 더보기 시트 11개·11px 터치 밀도 과다, 데스크톱·모바일 "더보기" 내용 불일치.
- **액션**: BottomNav 재편 1건으로 묶어 처리(탭 구성: 홈(대시보드)·연구(개인)·학술활동·커뮤니티·마이 + matchPrefixes 정비 + 더보기 6~8개 2열) — **S~M, 단일 파일 중심이라 ROI 높음**.

### A-5. 계획 체계 신뢰성 — "선언 ≠ 완료"와 라운드 번호 충돌 (PM GAP-M + 이번 세션 실증)
- 좀비 병렬 라운드로 v13·v14 계획서가 각 2벌 존재, 커밋 태그 판독 불가. 리허설 "핫픽스 예정" 선언이 미구현으로 남았던 것을 PM 분석이 적발(게이트 59로 해소).
- **액션**: ① 라운드 ID 유일화(날짜+트랙) ② **완료 선언은 커밋 해시 병기 필수** 규칙 ③ 중복 계획서에 상호 참조 헤더 추가. (프로세스 규칙 — 다음 라운드 편성 전 적용)
