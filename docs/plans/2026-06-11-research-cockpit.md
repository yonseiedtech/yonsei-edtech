# 연구 코크핏 (Research Cockpit) — 2026-06-11

## 문제
오늘 구축된 연구 지원 기능들(논문 여정·에디터 v2·지도 노트·연구 타이머·아카이브)이
**서로를 모른다**. 여정은 학기로만 단계를 추정하고, 에디터는 지도 노트의 미반영
항목을 보여주지 못하며, 진척(글자수·계획서·집필 시간)은 어디에도 합산되지 않는다.

## 목표
"내 논문이 어디까지 왔는지"가 **자동 측정**되고, 조각 기능들이 서로를 참조하는 연결 레이어.

## 설계
1. **진행률 엔진** `src/features/research/thesis-progress.ts` (순수 함수 + 단위 테스트)
   - 입력: WritingPaper(sections/chapters), 계획서 존재, 장별 미반영 지도 수, 집필 분
   - 출력: 장별 글자수·레벨(0 미작성/1 시작/2 진행/3 본궤도), 전체 %, 활동 기반 단계 신호(1~5)
2. **지도 노트 ↔ 에디터**: 챕터 탭에 미반영 배지, 현재 장의 미반영 지도 리스트(가이드 아래)
3. **코크핏 스트립** (MyResearchView): 5장 진행 세그먼트 바 + 총 글자·미반영 지도·집필 시간·버전 수
4. **여정 활동 신호**: ThesisJourney 에 "활동 기반으로는 N단계 신호" 보조 표시
   (학기 추정과 다르면 사용자가 단계를 갱신하도록 유도)

## 원칙
- 엔진은 순수 함수 — 컬렉션 추가 없음 (기존 데이터 재해석)
- 신호는 강제가 아니라 제안 (단계 변경은 항상 사용자의 결정)
