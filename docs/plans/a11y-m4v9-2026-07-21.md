## 1. 구현 내역

### 1-1. globals.css — CSS reduced-motion 전역 규칙

**파일**: `src/app/globals.css`

```css
/* 변경 전 */
html {
  scroll-behavior: smooth;
}

/* 변경 후 */
@media (prefers-reduced-motion: no-preference) {
  html {
    scroll-behavior: smooth;
  }
}

/* a11y: 모션 민감 사용자 — 모든 CSS 애니메이션·트랜지션 최소화 (M4-v9) */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}
```

**효과**: `tw-animate-css` 유틸 클래스(`animate-*`)와 모든 Tailwind `transition-*` 유틸의 CSS 레이어 애니메이션이 `prefers-reduced-motion: reduce` 환경에서 즉시 비활성화된다. `html { scroll-behavior: smooth }` 도 reduce 환경에서는 `auto`로 폴백.

### 1-2. MotionProvider — framer-motion 전역 reduced-motion 적용

**신규 파일**: `src/components/layout/MotionProvider.tsx`

```tsx
"use client";
import { MotionConfig } from "framer-motion";

export default function MotionProvider({ children }) {
  return <MotionConfig reducedMotion="user">{children}</MotionConfig>;
}
```

**layout.tsx 추가**: `<body>` 내부 최상단 래퍼로 `<MotionProvider>` 삽입.

**효과**: 프로젝트 전체 framer-motion 사용처 18개 파일(`motion.div`, `AnimatePresence`, `useSpring`, `useAnimationFrame` 등)이 OS의 `prefers-reduced-motion: reduce` 설정을 자동 감지하여 애니메이션을 정지 상태로 렌더링한다. 개별 파일에 이미 구현된 `useReducedMotion` 가드(HeroSection, ResearchJourneySection, ResearchHero — 4파일)와 중복되지 않고 공존한다.
