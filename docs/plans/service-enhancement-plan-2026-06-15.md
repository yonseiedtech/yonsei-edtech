### 1-2. 약점·기능 중복·미흡 영역

**회원 가치 관점**
- 콘텐츠는 풍부하나 발견성·유도가 약함. 신입 회원이 50+ 개념·연구 보고서 3탭·아카이브 7컬렉션 중 "어디서 시작할지" 진입점이 불명확.
- 진단 결과 사장. paperReadiness/analysisReadiness 점수가 마이페이지·대시보드에 표시되지 않고 일회성 리포트로 끝남. 약점→아카이브 링크는 있으나 학습 루프로 이어지지 않음.
- 기능 간 단절. 진단↔타이머 연계 없음, 연구 모형↔보고서 변인 비연동(중복 입력), 심사 연습·진단 결과의 피어 비교/축적 부재.

**운영 부담 관점**
- 의사결정 업무 수작업 잔존. 회원 승인, 콘텐츠(아카이브 시드·카드뉴스·뉴스레터·세미나 안내) 작성, 통계 해석은 자동화율 낮음. 알림/상태 전환은 95%+ 자동화되어 대비됨.
- 아카이브 시드 한국어 KCI 인용 할루시네이션 위험. 운영진 RISS/KCI 수동 검증 부담이 구조적으로 남아 있음.

**기능 중복(구조 정리 대상)**

| 중복쌍 | 현황 | 판정 |
|---|---|---|
| `members` vs `directory` | directory가 전체 기능 포함, members는 불완전 | members 고아 가능성 — 통합/폐기 |
| `board` vs `boards` | board=8카테고리 완성, boards=3파일 용도 불명확 | boards 통합 또는 목적 재정의 |
| `network` vs `gatherings` | network=관계망 시각화, gatherings=행사·회비. 개념은 다르나 "네트워킹" 용어 중첩 | 용어 정리(혼동만 해소) |
| `archive` vs `research` | 보완관계(이론 vs 졸업생 논문 분석). 구조 문제 아님 | 상호 링크 보강만 |

## 3. 운영 효율화 (소수 운영진 부담 경감)

현황: 알림·상태 전환은 cron 18종으로 95%+ 자동화됨. 부담은 의사결정·콘텐츠·검증·통계 해석에 집중.

| 영역 | 현재 수작업 | 자동화/대시보드 제안 | 효과 | 규모 |
|---|---|---|---|---|
| 회원 승인 | 개별 검토·승인 | 일괄 승인 + 자동 매칭 신뢰도·중복 가입 경고 표시(이미 일부 존재 → 통합 강화) | 승인 시간 단축 | S~M |
| 운영 통계 | 콘솔 여러 화면 순회 | 주간 운영 요약 대시보드 1화면(신규 회원·활동 추이·이탈 위험·미응답 신청) + cron 자동 집계 | 순회·해석 시간 절감 | M |
| 콘텐츠 작성 | 카드뉴스·뉴스레터·세미나 안내 수작업 | 템플릿화 + 활동 데이터 자동 채움(세미나/활동에서 초안 생성) | 작성 부담 경감 | M |
| 아카이브 검증 | KCI 인용 수동 검증 | RISS/KCI 검증 보조(document-specialist 패턴) + 검증 상태 배지·미검증 경고 | 할루시네이션 위험 관리 | M |
| 알림 관리 | (대부분 자동) | cron-logs 대시보드 이미 존재 → 실패 알림·재시도 버튼 보강 | 모니터링 안정화 | S |

**빠른 효과 후보(운영)**: 회원 일괄 승인 통합(S), 주간 운영 요약 대시보드(M), cron 실패 알림 보강(S).

## 4. 로드맵 (다음 1~2개월)

원칙: LIVE 기능(진단·잔디·타이머·아카이브·네트워킹) 위에 얹어 quick win 먼저, 큰 투자는 후순위.

### 4-1. Quick Win (1~2주, 즉시 효과)
- M1 구조 정리(members/boards/용어) — S, 혼동·유지보수 즉시 개선
- 운영: 회원 일괄 승인 통합(S) + cron 실패 알림 보강(S)
- H1 1단계: 진단 준비도 점수를 마이페이지 위젯으로 노출(점수 표시만 먼저)

### 4-2. 4주 차 핵심 (동기 루프 완성)
- H1 진단→학습 루프 전체(추천 경로 카드·재진단 유도)
- H2 신입 7일 온보딩 가이드
- H3 회원 가치 통합 대시보드 위젯

### 4-3. 6~8주 차 (큰 투자)
- H4 운영진 인사이트 콘솔 + 주간 운영 요약 대시보드
- M2 연구 모형↔보고서 변인 동기화
- M5 콘텐츠 발견성(아카이브 추천 경로)

### 4-4. 백로그(추후)
- M3 보고서 진입점 통합 · M4 피어 러닝 · L1 졸업생 재참여 · L2~L4

**빠른 효과 vs 큰 투자 구분**
- 빠른 효과: M1, 운영 일괄 승인/알림 보강, H1 1단계 점수 노출, H2 온보딩 골격
- 큰 투자: H4 운영 콘솔, M2 변인 동기화, M3 보고서 허브, L1 졸업생 네트워크

## 핵심 결론 (우선 3가지)

1. **진단 결과를 살려라(H1)**. 가장 잘 만든 기능(진단)이 일회성으로 사장되고 있음. 준비도 점수 상시 노출 + 약점→아카이브→타이머→재진단 루프가 최고 ROI quick win.
2. **발견성·온보딩(H2·M5)**. 콘텐츠는 충분히 풍부하므로 "더 만들기"보다 "안내·발견" 투자가 효율적.
3. **운영 부담은 의사결정·통계에 집중(H4)**. 알림은 이미 자동화됨. 주간 운영 요약 1화면 + 회원 일괄 승인 통합이 소수 운영진에게 직접 효과.
