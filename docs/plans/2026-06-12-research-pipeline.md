# 연구 파이프라인 고도화 기획 — 읽기→보고서→계획서→논문 연결

> 2026-06-12, 교육공학 논문 글쓰기 전문가 관점 기획.
> 핵심 명제: **"읽은 것이 쓰는 곳에서 보이고, 쓴 것이 다음 단계의 초안이 된다."**

## 1. 진단 — 도구는 4개, 흐름은 0개

| 단계 | 도구 | 데이터 | 상태 |
|---|---|---|---|
| 읽기 | 논문 읽기 탭 | `research_papers` — 변인 5종·methodology·findings·insights·**myConnection**·태그·평점 | ✅ 구조화 우수 |
| 보고서 | 인터뷰 모드(5트랙) | `research_reports` — 문제 정의·이론 카드·`priorResearchPaperIds` | ✅ 풍부 |
| 계획서 | 계획서 인터뷰 | `research_proposals` — purpose·scope·method·content·`referencePaperIds` | ✅ 존재 |
| 논문 | 에디터 v2 | `writing_papers` — 섹션·가이드 14종·가정 검정·골격 | ✅ 고도화 완료 |

**문제**: 학술 글쓰기는 본질적으로 *reading-to-write* 과정이고, 교육대학원 5학기 모델(부심 강의 구조와 동일)에서 보고서(2학기)→계획서(3학기)→본문(4~5학기)은 **같은 연구 질문의 점진적 정교화**다. 그런데 현재는—
1. 읽기에서 정리한 변인·발견·통찰이 **글쓰기 화면에서 보이지 않음** → "선행연구 고찰"을 쓰려면 탭을 오가며 다시 찾아야 함
2. 보고서의 문제 정의·이론 카드가 계획서로 **이월되지 않음** → 같은 내용을 처음부터 다시 씀
3. 계획서의 목적·방법이 본문 서론·방법 장의 **모태인데 수동 복사** → 관례상 계획서 1~3장 = 학위논문 1~3장 초안
4. 여정(ThesisJourney)은 단계를 안내하지만 **단계별 산출물 현황**은 본문 진행률(코크핏)만 보임

## 2. 기획 — 5개 스프린트 (우선순위순)

### P1. 에디터 "읽기 서랍" (Reading-to-Write Bridge) — 체감 최대
- 논문 에디터(+보고서·계획서 화면)에 접이식 사이드 패널: 내 `research_papers` 검색·열람
- 카드에 변인·방법론·findings·insights·myConnection 표시, 읽기 상태·평점 칩
- **1클릭 삽입**: "(저자, 연도) 인용 스텁 + 선택 필드 요지"를 현재 단락 뒤에 삽입
- **장별 우선 표시**: 이론적 배경 → findings·variables / 연구 방법 → methodology / 서론 → myConnection
- 근거: "선행연구 고찰은 나열이 아니라 종합"(부심 3주차) — 종합은 원료가 한눈에 보일 때 가능

### P2. 계획서→본문 시딩 (Proposal→Thesis Seeding)
- 본문 에디터의 빈 서론/방법 장에서 "내 계획서에서 가져오기" 배너
- purpose→서론 '연구 목적' 섹션, scope→'연구 범위', method→방법 장 초안 단락으로 1클릭 시딩
- 시딩 전 **버전 자동 백업**(기존 인프라 재사용), 시딩 후 단락은 자유 편집
- referencePaperIds → 읽기 서랍에 "계획서 인용 논문" 필터 연동

### P3. 보고서→계획서 승격 (Report→Proposal Promotion)
- 보고서 완료 화면 + 계획서 빈 상태에 "보고서에서 시작하기"
- 문제 정의(problemDefinition·fieldProblem·진단 필드)→purpose 초안, 이론 카드 요지→content 초안, priorResearchPaperIds→referencePaperIds 복사
- 근거: 소논문(2학기 연습)의 산출이 계획서의 뼈대가 되는 5학기 모델 정합

### P4. 여정 산출물 현황 칩
- ThesisJourney 단계 카드에 실데이터 칩: 1단계 "완독 N편·노트 N", 2단계 "보고서 작성중/완료", 3단계 "계획서 N자", 4~5단계 "본문 N% (코크핏)"
- 단계 CTA를 산출물 상태 기반으로: 보고서 완료+계획서 없음 → "보고서를 계획서로 승격" 버튼

### P5. 인용↔읽기 대조 (2차)
- 본문에서 "(저자, 연도)" 패턴 감지 → `research_papers` 대조: 읽은 논문 배지 / 미등록은 읽기 리스트 추가 제안
- 참고문헌 자동 생성은 서지 데이터 정합성 이슈로 보류

## 3. 구현 노트
- 신규 컬렉션 불필요 — 기존 4모델 + 참조 필드로 전부 구현 가능. rules 변경 없음(전부 본인 데이터 read)
- P1 핵심 컴포넌트: `ReadingDrawer`(공용) — papersApi.listByUser + 장 컨텍스트 prop
- P2·P3 시딩은 **빈 챕터/빈 문서일 때만 제안**(기존 작성분 보호) + 백업 우선 원칙
- 단계: P1 → P2 → P3 → P4 → P5, 각 사이클 독립 배포 가능
