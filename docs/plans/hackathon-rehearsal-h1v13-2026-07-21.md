## 1. 단계별 [기대 vs 실제]

### 1-1. 참가 접수 (registration — 기준선)
| 항목 | 기대 | 실제 | 판정 |
|---|---|---|---|
| 스테퍼 | "현재: 참가 접수" | 동일 + 안내 문구 | ✅ |
| D-day | D-32 카운트다운(초 단위) | 정상 | ✅ |
| 팀 현황/제출/아이디어 보드 빈 상태 | CTA 포함 안내 | 3곳 모두 EmptyState+CTA | ✅ |
| 수상작 섹션 | 발표 예정(8/29) 안내 | 정상 | ✅ |

### 1-2. 접수 마감·제출 오픈 (submission)
| 항목 | 기대 | 실제 | 판정 |
|---|---|---|---|
| 콘솔 전환 | confirm→토스트→상태 갱신 | "단계를 전환했습니다"·수동 지정 배지·자동 기준 병기·**자동으로 되돌리기** 버튼 노출 | ✅ |
| 허브 스테퍼 | "현재: 산출물 제출" | 동일 | ✅ |
| 제출 마감 표기 | 행사 다음날(8/23) 기준 D-33 | "제출 마감 D-33" | ✅ |
| 제출 폼 | 팀명·제목·설명·링크·**팀원 선택(memberIds, H2)** | 전 필드 + memberIds 헬퍼 문구 정상 | ✅ |
| 테스트 제출 | 저장 성공 | "산출물을 제출했습니다" 토스트 | ✅ |
| **참가 신청 폼** | **접수 마감이므로 닫힘 기대** | **"참가 신청하기" 폼이 그대로 노출** | ⚠️ **D-1** |

### 1-3. 심사 (judging)
| 항목 | 기대 | 실제 | 판정 |
|---|---|---|---|
| 콘솔 전환 | 선행 단계 자동 체크 | 접수 마감·제출 마감 ✓✓ 후 심사 단계 | ✅ |
| 현황 카드 | 제출 1건·심사 0/1 반영 | 정상 | ✅ |
| 루브릭 | 4기준(문제 정의·교육공학적 근거·구현 완성도·발표) 0~5 | 정상, 각 기준 설명 병기 | ✅ |
| 점수 저장 | 저장→수정 모드 전환 | "심사 점수를 저장했습니다"→"점수 수정" 버튼 | ✅ |
| 진행률 | 저장 후 100% (1/1) | 정상 | ✅ |
| 수상 지정 | 미수상/대상/최우수상/장려상 + 공개 | "수상 등급을 갱신했습니다"→"수상작 공개"→"공개 해제" 되돌리기 존재 | ✅ |

### 1-4. 수상 발표 (awards)
| 항목 | 기대 | 실제 | 판정 |

| **H2 포트폴리오 1클릭** | 본인에게 버튼→적재→"추가됨" | 토스트("운영진 검증 후 정식 표기")·상태 전환·멱등 시드 정상 | ✅ |
| **H2 아카이브 등록 딥링크** | staff에게 노출 | 정상 노출(실행은 검수 정책상 생략) | ✅ |
| 갱신된 HINT 문구 | 실제 버튼 안내 | 정상 | ✅ |

### 1-5. 복원
| 항목 | 기대 | 실제 | 판정 |

## 2. 발견 결함·관찰

| # | 등급 | 내용 | 조치 |
|---|---|---|---|
| **D-1** | 결함(중) | **접수 마감(submission phase) 후에도 참가 신청 폼·"참가 신청하기" 버튼이 그대로 노출** — 마감 개념이 아이디어 보드에 전파되지 않음. 당일 혼선 소지 | 핫픽스 대상: submission 이후 phase에서 신청 폼을 "접수가 마감되었습니다" 안내로 대체(보드 열람은 유지) |
| **D-3** | 결함(중) | **제출물 삭제 경로 부재** — 소유자(수정만 가능)·콘솔 어디에도 삭제 없음. 오제출·테스트 데이터 정리 불가 | 핫픽스 대상: 콘솔 심사 탭에 staff 전용 삭제(연결 심사 기록 동반 삭제·confirm) |
| D-2 | 관찰(하) | 콘솔 "팀 확정 0팀" vs 허브 "확정 팀 1건" — 집계 소스 상이(콘솔=팀 확정 액션, 허브=제출 존재) | 라벨 정합화 검토(후속 라운드) |
| D-4 | 관찰(하) | 접수 단계에서도 제출 섹션("제출 시작")이 노출 — 조기 제출 허용이 의도인지 불명확 | 의도 확인 후 필요시 phase 게이트(후속) |
| — | 자동화 특이 | 단계 전환 버튼이 데스크톱/모바일 이중 렌더라 ref 클릭이 숨은 쪽을 때릴 수 있음(앱 결함 아님) | 기록만 |

## 3. 테스트 데이터 정리 현황

| 데이터 | 처리 |
|---|---|

| 수상 지정·공개 | 미수상·공개 해제 완료 |
| external_activities "[수상] [리허설]…" | **삭제 완료** (마이페이지 포트폴리오 대외활동 탭) |

## 4. 종합 판정

접수→제출→심사→수상→복원 **전 구간이 의도대로 작동**하며, phase 전환·현황 집계·H2 사후 파이프라인(팀원 크레딧 헬퍼·수상 1클릭·아카이브 딥링크)까지 LIVE에서 실증됐다. 발견 결함 2건(D-1 접수 마감 미전파, D-3 삭제 경로 부재)은 경량 핫픽스로 게이트 59에 포함한다. 8/22 실전 운영 리허설 목적 달성.
