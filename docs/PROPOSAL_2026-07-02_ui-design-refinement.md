# UI 디자인 정제 제안서 — 여백·모바일·가독성·컬러·"AI 티" 제거

작성일: 2026-07-02 · 근거: 코드 전수 감사 5종 (여백/레이아웃 · 모바일 · 타이포그래피 · AI-UI 시그니처 · 컬러 팔레트), src/ 1,131개 파일

## 0. 메타 진단 한 줄

**시스템(DESIGN.md·토큰·PageContainer·브랜드 네이비)은 이미 있는데, 집행이 안 되어 화면마다 재발명되고 있다.** "AI로 만든 느낌"의 정체는 특정 아이콘이 아니라 ① 모든 정보를 칩·배지로 만드는 균질화, ② 라벨마다 붙는 "— 설명" 글로스, ③ 이모지 살포, ④ 파스텔 무지개 틴트의 조합이다. 반대로 이 서비스만의 진짜 자산(연세 네이비 #003378, 한국어 세리프 토큰, 부엉이 마스코트, 잔디 그리드)은 그 소음에 묻혀 있다.

## 1. 영역별 진단 (근거 수치)

### A. 가독성·타이포그래피 — 가장 심각
- 최다 사용 크기가 **text-xs(12px) 2,911회** > text-sm 2,664회. 서브12px 리터럴이 **2,239회**(text-[10px] 996 + [11px] 1,139 + [9px] 104).
- 루트가 `clamp(14px,…)`로 축소되어 **모바일 실측: text-sm≈12.3px, text-xs≈10.5px, [10px]≈8.8px**. 아카이브 본문·게시글 본문·논문 작성 textarea 등 장문 읽기/쓰기 표면 전부 14px.
- 한국어 어절 보호 **keep-all/break-keep 전체 5회뿐** — 제목·문단이 어절 중간에서 꺾임.
- `max-w-prose` **0회** — 본문이 896~1024px 폭으로 흐름(적정 측정폭 무시).
- **muted:foreground = 7:1** (2,231 vs 321) + 임의 불투명도 10단(/40~/90) — 본문까지 회색.
- 제목이 크기 없이 굵기로만 구분(`text-sm font-bold` 등 669회+) — 위계 붕괴.
- 세리프 웹폰트 3종 로드하나 실사용 font-serif 6회 — 죽은 페이로드.
- 하드코딩 색상 라인의 72%(840)가 dark: 미보유(상한 추정) — 다크모드 대비 저하.

### B. "AI 티" 시그니처 — 두 번째 심각
- **이모지 1,847회**. 알림 타입 아이콘 맵(17종)은 일관된 시스템으로 방어 가능 / 나머지(🎓 소제목 프리픽스 32회, 💡 콜아웃, 토스트 🔥📖🎉 13회)는 장식 살포.
- **"— 한글 설명" 글로스 1,039회** — 거의 모든 라벨 뒤에 붙는 LLM 문체의 최대 신호.
- rounded-full **845회** + size 10~13 아이콘 770회 = 모든 메타데이터가 칩. 한 패널에 칩 문법 6종(ResearchQuestionsPanel).
- 아이콘-컬러사각형 공식(h-10 w-10 rounded-xl bg-{색}-100) 33파일 — 대시보드 첫인상을 지배.
- 뱃지 3중 문법(Badge/outline/수제 rounded-full) 공존 화면 **83개**.
- animate-in 109회 + hover:-translate-y-0.5 46회 — 동일 모션 일괄 적용.
- 토스트 1,426건 중 발랄체("함께해요"·느낌표·이모지) 소수 혼입 — 보이스 불일치.
- **증폭할 자산**: 네이비 #003378(AAA 주석까지 있음), Hahmlet/Noto Serif KR 디스플레이 토큰, ReadingMascot(수제 SVG 부엉이), 잔디 스트릭 시스템.

### C. 컬러 팔레트 — 구조적 원인
- 활성 hue **19종**(유채색 15종). 파랑만 blue/sky/indigo/violet 4갈래 1,979회, 초록 3갈래, 노랑 2갈래.
- **status 토큰 부재**(success/warning/info 미정의)가 난립의 근본 원인. 에러는 destructive 토큰·red·rose **3중**.
- amber 하나가 **8가지 의미**(별점·경고·진행중·카테고리…)로 소비. 같은 화면 4+ hue: 캘린더 6종, 라이팅에디터 결과부 5종, 대시보드 5종.
- **카테고리 색 맵 26개+ 독립 하드코딩** — 동일 개념(concept/variable/measurement)이 그래프 vs 아카이브 vs 기초용어에서 서로 다른 색.
- 브랜드: primary 사용 우세(1,033회)이나 제네릭 blue 430회+ 병존. **골드 secondary는 사실상 미사용**, navy-footer 1곳.
- ad-hoc 색의 dark: 커버리지 ≈35%.

### D. 여백·레이아웃
- 카드 패딩 4계층 난립(p-4 226 / p-5 159 / p-6 75 / 비인가 p-3·3.5 89) — 대시보드 한 페이지에 4종.
- radius: rounded-2xl가 표준(69%)이나 rounded-lg 카드가 95파일, xl 60건 — 버튼/카드 경계 붕괴.
- 페이지당 여백 사다리 9~11단(+2.5/3.5 하프스텝), 섹션 간격 mt-6/8 무규칙 교차.
- **PageContainer 미사용 181/291 페이지**(62%), 40페이지는 폭 수제 구현.
- 카드 3~4중 중첩: WritingPaperEditor 결과부(2xl→xl→xl→lg), ActivityDetail(i34 깊이).
- 대시보드: 최상위 섹션 ~17개 + 12위젯 그리드 — "전부 쌓고 아무것도 묶지 않은" 밀도.

### E. 모바일
- 22~28px 칩 버튼(권장 44px 미달) **66개+**, 연구 도구·포트폴리오 검증(파괴적 액션!)에 집중.
- WritingPaperEditor 장 네비: **7버튼 고정 flex, wrap/스크롤 없음** → 375px에서 클리핑.
- 공용 TabsList에 overflow-x-auto 자체가 없음 → MyResearchView 5탭 등 오버플로.
- 기본 grid-cols-2 146곳 중 콘텐츠성 그리드 일부(매거진·폼) 협착, grid-cols-3 다이얼로그 3곳.
- 건강한 부분: BottomNav 여백(루트 pb 보정), 다이얼로그 폭, 와이드 표 전부 overflow-x-auto 처리.

## 2. 디자인 방향 — "학술 에디토리얼"로 수렴

원칙: **칩이 아니라 활자로, 파스텔이 아니라 네이비로, 이모지가 아니라 마스코트로.**

1. 정체성 축 = 네이비(#003378) + 한국어 세리프 디스플레이(Hahmlet) + 부엉이·잔디. 이 4개만 성격을 담당하고 나머지는 조용해진다.
2. 색은 "의미"에만: 브랜드(네이비)·상태 4종(success/warn/danger/info)·카테고리 팔레트 1벌(6색). 그 외 유채색 금지.
3. 텍스트가 1급 시민: 본문 15~16px 풀 콘트라스트, 제목은 크기로 구분, 설명 글로스는 삭제가 기본.
4. 칩은 "필터·태그"에만. 상태는 텍스트+점(dot), 메타데이터는 그냥 텍스트.

## 3. 개선 로드맵

### D1. 글로벌 원라이너·토큰 신설 (효과 大 / 공수 극소 — 반나절)
- `globals.css`: `body { word-break: keep-all; overflow-wrap: break-word; }` — 어절 꺾임 전면 해소.
- 루트 clamp 하한 14px→15~16px 상향(또는 제거) — 모바일 실측 크기 일괄 +7~14%.
- status 토큰 신설: `--success/--warning/--danger(=destructive 재사용)/--info` + Tailwind 확장. 카테고리 팔레트 토큰 `--cat-1~6` 신설.
- 미사용 세리프 2종(Gowun Batang, Noto Serif KR 중 1) 로드 제거 — 페이로드 절감. Hahmlet은 유지(정체성 축).

### D2. 타이포 스케일 정립 (효과 大 / 공수 中)
- 스케일 계약: 본문 text-[15px]/16px · 보조 text-sm · 메타 text-xs(12px 아래 금지, 캡션 예외 1종만) · 제목 h2=text-xl/lg, h3=text-base semibold.
- 장문 4표면(아카이브 상세·게시글·뉴스레터·에디터 textarea): 본문 16px + `max-w-prose(≈65ch)` + foreground 풀 콘트라스트.
- muted 규칙: "메타데이터·타임스탬프·비활성"에만. 문단 단위 muted 금지. 불투명도 계층은 /60 하나로 통일.
- 서브12px 2,239건: 일괄 sed가 아니라 **회귀 방지 lint 규칙**(text-[9~11px] 신규 금지) + 상위 10개 파일부터 수동 승격.

### D3. "AI 티" 제거 (효과 大 / 공수 中)
- 이모지: 알림 타입 맵만 존치(추후 lucide로 교체 검토), **문장 속 장식 이모지·토스트 이모지 전수 제거**. ResearchReport 🎓 프리픽스 32건 우선.
- "— 설명" 글로스 1,039건: 원칙 = 라벨이 자명하면 삭제, 정말 필요한 안내는 최초 1회 노출 후 접힘(또는 물음표 툴팁). 신규 작성 금지 규칙.
- 뱃지 단일화: `<Badge>` 변형 3종(solid/outline/dot)으로 수렴, 수제 rounded-full 칩은 태그·필터에만 허용.
- 토스트 보이스 가이드: "~했습니다" 존댓말 평서, 이모지·느낌표 금지(축하 순간은 부엉이/컨페티가 담당).
- 아이콘 정책: 섹션 제목의 장식 아이콘 제거(제목은 활자), 액션 버튼·내비에만 아이콘. 아이콘-컬러사각형 공식은 대시보드 3카드 등 최상위 3곳만 존치.

### D4. 컬러 통합 (효과 大 / 공수 中~大)
- hue 19종 → **8종**: navy(primary)·gold(절제된 포인트)·success(emerald)·warning(amber)·danger(rose·red/destructive와 통일)·info(sky)·중립(slate 단일)·카테고리 팔레트(6색 고정).
- 상태색 치환: green→emerald, red→rose(destructive 토큰 경유), blue/indigo 상태 표시→info.
- **카테고리 색 맵 26개 → `src/lib/category-palette.ts` 1곳**: 도메인별 매핑은 유지하되 팔레트는 공유(동일 개념 동일 색 보장 — concept/variable/measurement 불일치 해소).
- amber 의미 분리: 별점은 star 전용 토큰(또는 gold), 경고는 warning만.
- dark: 누락은 토큰 경유로 자동 해결(토큰化가 곧 다크모드 수리).

### D5. 여백·구조 (효과 中 / 공수 中)
- 카드 계약: 표준 카드 = `rounded-2xl p-5`(모바일 p-4), 내부 서브블록 = radius/보더 없이 `bg-muted/40 rounded-lg p-3` 1단만 — **중첩 보더 3단 금지**(에디터 결과부·ActivityDetail flatten).
- 여백 사다리 4단 계약: 요소 내 2 · 요소 간 4 · 블록 간 6 · 섹션 간 10. 하프스텝(2.5/3.5) 신규 금지.
- PageContainer 강제: 미사용 181페이지 중 회원 대면 페이지부터 이관(콘솔은 후순위), dashboard 인라인 max-w-6xl 14회 정리.
- 대시보드 그룹핑: 17섹션 → "오늘/이번 주/내 성장/커뮤니티" 4그룹 헤더로 묶고 그룹 간 mt-10.

### D6. 모바일 (효과 中 / 공수 小~中)
- 터치 타겟 최소 계약: 인터랙티브 칩 `min-h-9(36px) px-3`(시각적으론 작아 보여도 히트영역 확보 — before 의사요소로도 가능). 파괴적 액션(검증 거절 등)은 정식 버튼으로 승격.
- TabsList에 `overflow-x-auto scrollbar-none` 기본 내장(프리미티브 1곳 수정으로 전 화면 해결).
- WritingPaperEditor 장 네비: 7버튼 → overflow-x-auto + snap, 또는 모바일에서 셀렉트/드롭다운.
- 기본 grid-cols-2 콘텐츠 그리드(매거진·OrgChart·labs/new)와 grid-cols-3 다이얼로그를 base 1열로.

### 실행 순서 제안
**D1(반나절, 전면 체감) → D3 이모지·글로스 스윕 + D2 장문 4표면(첫인상 급변) → D4 상태색·카테고리 팔레트 → D5/D6 병행.** 대량 항목(이모지·글로스·서브12px)은 "신규 금지 규칙 + 상위 파일부터 수동"의 2트랙으로, 한 번의 위험한 전역 치환을 피한다.

### 성공 기준
- 모바일 본문 실측 ≥14px, 어절 꺾임 0, 장문 측정폭 ≤70ch
- 유채색 hue 15→8, 카테고리 색 맵 26→1, 문장 내 이모지 0
- 카드 패딩 2단·radius 2단, PageContainer 채택률 62%↑→90%+(회원 대면)
- 인터랙티브 요소 최소 히트 36px
