# 연세교육공학회 세미나 시스템 고도화 프로젝트 제안서

> 작성일: 2026-04-12
> 분석 범위: 세미나 관리(seminar-admin), 세미나 상세페이지, 세미나 공간(LMS) 페이지
> 관점: UI/UX 디자이너 + Product Manager

---

## 1. Executive Summary

연세교육공학회 세미나 시스템은 **세미나 기획 → 홍보 → 참석 관리 → 당일 운영 → 사후 관리**의 전체 라이프사이클을 지원하는 종합 플랫폼입니다. 현재 시스템은 풍부한 기능을 갖추고 있으나, 각 단계 간 연결이 수동적이고 운영진의 반복 작업이 많습니다.

본 제안서는 **자동화(Automation)**, **워크플로우 연결(Workflow Integration)**, **데이터 기반 의사결정(Data-Driven Decision Making)** 세 축을 중심으로 고도화 방안을 제시합니다.

### 기대 효과
- 세미나 1회 운영 시 운영진 작업 시간 **약 60% 절감**
- 참석률 및 후기 작성률 **20% 이상 향상**
- 세미나 품질 추적 및 개선을 위한 **데이터 기반 인사이트 확보**

## 2. 현황 분석 (As-Is)

### 2.1 세미나 관리 - 세미나 목록 (SeminarListTab)

**현재 기능:**
- 전체 세미나 목록 조회 (상태별 필터: 예정/진행/완료/취소)
- 세미나 선택 시 하위 탭(홍보, 타임라인, 참석자, 수료증)과 연동

**Pain Points:**
| 문제 | 영향 | 심각도 |
|------|------|--------|
| 세미나 간 비교 기능 없음 | 과거 세미나 참고 시 일일이 클릭해야 함 | 중 |
| 검색/정렬 기능 부재 | 세미나가 늘어날수록 원하는 세미나 찾기 어려움 | 중 |
| 대시보드 부재 | 전체 세미나 운영 현황 파악 불가 | 상 |
| 세미나 복제 기능 없음 | 유사 세미나 생성 시 매번 처음부터 입력 | 상 |
| 상태 전환이 수동 | cron으로 자동화되어 있으나 즉시 반영 불가 | 하 |

### 2.2 세미나 관리 - 세미나 생성 (SeminarForm)

**현재 기능:**
- 기본 정보 입력 (제목, 날짜, 시간, 장소, 온/오프라인)
- 발표자 정보 (이름, 소속, 직함, 사진, 약력)
- 최대 참석 인원 설정
- 일정 충돌 감지 (경고만, 차단하지 않음)
- 라이브 프리뷰 모드

**Pain Points:**
| 문제 | 영향 | 심각도 |
|------|------|--------|
| 임시저장 불가 | 작성 중 이탈 시 모든 내용 유실 | 상 |
| 세미나 복제 불가 | 정기 세미나 시 반복 입력 필요 | 상 |
| 세션 프로그램 생성 분리 | 세미나 생성 후 별도로 세션 추가 필요 | 중 |
| 발표자 DB 없음 | 동일 연사 재초청 시 정보 재입력 | 중 |
| 홍보/타임라인 자동 연동 없음 | 세미나 생성 후 수동으로 각 탭에서 설정 | 상 |

### 2.3 세미나 관리 - 홍보 제작 (PromotionTab)

**현재 기능:**
- AI 기반 5가지 홍보 콘텐츠 자동 생성 (보도자료, SNS, 이메일, 카카오, 해시태그)
- 실시간 스트리밍 생성, 클립보드 복사/다운로드
- 콘텐츠 저장 및 히스토리 관리
- 보도자료 게시판 직접 게시

**Pain Points:**
| 문제 | 영향 | 심각도 |
|------|------|--------|
| 보도자료만 게시판 게시 가능 | SNS/이메일 등은 수동 배포 필요 | 중 |
| 이메일 대량 발송 미연동 | 생성된 이메일 콘텐츠를 뉴스레터 시스템에 수동 복사 | 상 |
| 커스텀 프롬프트 편집 불가 | AI 결과 조정이 제한적 | 하 |
| 포스터 커스터마이징 불가 | AI 생성 포스터의 디자인 수정 불가 | 중 |

### 2.4 세미나 관리 - 운영 타임라인 (TimelineTab)

**현재 기능:**
- 온/오프라인 세미나 별 타임라인 템플릿
- D-day 기반 일정 관리 (D-30, D-14, D-7 등)
- 담당자 배정 및 완료 추적
- 리마인더 메시지 템플릿 (D-2, 당일, 사후)
- 일괄 완료 기능

**Pain Points:**
| 문제 | 영향 | 심각도 |
|------|------|--------|
| 리마인더 자동 발송 안됨 | 템플릿만 제공, 수동 복사하여 발송 | 상 |
| 알림/푸시 연동 없음 | 담당자에게 할당 알림 없음 | 상 |
| 자동 완료 처리 없음 | 연동된 작업(예: 홍보 게시) 완료 시 수동 체크 필요 | 중 |
| 반복 세미나 시 템플릿 재생성 | 기존 세미나 타임라인 복제 불가 | 중 |

### 2.5 세미나 관리 - 참석자 관리 (RegistrationsTab + ReportTab)

**현재 기능:**
- CSV/Google Forms 참석자 일괄 등록
- 필드 자동 매핑 (이름→name, 학번→studentId 등)
- 신청→참석→체크인→수료 파이프라인 시각화
- 소속/학기/기수별 분석 대시보드
- CSV 내보내기, 공유/인쇄 기능

**Pain Points:**
| 문제 | 영향 | 심각도 |
|------|------|--------|
| 중복 등록 감지 없음 | CSV 재업로드 시 중복 참석자 발생 가능 | 상 |
| 대량 등록 롤백 불가 | 잘못된 CSV 업로드 시 수동 삭제 필요 | 중 |
| 실시간 알림 없음 | 신규 참석 신청 시 운영진 알림 없음 | 중 |
| 대기자 자동 전환 없음 | 취소 발생 시 대기자 수동 전환 필요 | 중 |

### 2.6 세미나 관리 - 수료증/명찰 (CertificateGenerator)

**현재 기능:**
- 수료증(C-번호) / 감사장(A-번호) 이중 모드
- 출석자 필터링 (미출석자 수료증 발급 차단)
- 일괄 발급 + 진행률 추적
- 수기 연사 추가 (감사장 모드)
- PPT 스타일 정렬 도구 (편집 모드)
- 이미지/PDF 다운로드

**Pain Points:**
| 문제 | 영향 | 심각도 |
|------|------|--------|
| 체크인 후 자동 발급 불가 | 세미나 종료 후 운영진이 수동 발급 | 상 |
| 디자인 템플릿 제한적 | 커스텀 디자인 불가 (고정 레이아웃) | 중 |
| 강사 후기 페이지와 디자인 비동기 | speaker-review의 감사장은 별도 고정 디자인 사용 | 중 |

### 2.7 세미나 상세페이지 (seminars/[id])

**현재 기능:**
- Hero 섹션: 상태 뱃지, 제목, 일시, 장소, 참석 현황
- 발표자 카드: 사진, 이름, 소속, 약력
- 세션 프로그램: 다중 세션, 시간표
- 참석 신청/취소 (정원 초과 시 대기)
- 후기 열람 (참석자/연사/운영진 구분)
- 자료 탭: LMS 문서
- 운영진 도구: 인라인 편집 다이얼로그
- AI 보도자료 생성, 공유 버튼

**Pain Points:**
| 문제 | 영향 | 심각도 |
|------|------|--------|
| 실시간 업데이트 없음 | 참석자 수 등 새로고침 필요 | 중 |
| 캘린더 연동 없음 | 참석 신청 시 Google/Apple 캘린더에 자동 추가 안됨 | 중 |
| 알림 구독 없음 | 세미나 변경/취소 시 참석자 알림 수동 | 상 |
| SEO 메타 데이터 부족 | 외부 공유 시 프리뷰 정보 제한적 | 하 |

### 2.8 세미나 공간 - LMS (seminars/[id]/lms)

**현재 기능:**
- 탭 구성: 개요, 발표자 소개, 세션, 자료실, 후기
- 자료 업로드/다운로드 (운영진)
- 후기 관리 (운영진)
- 홍보 콘텐츠 뷰어
- 포스터 갤러리
- 타임라인 트래커

**Pain Points:**
| 문제 | 영향 | 심각도 |
|------|------|--------|
| 녹화/영상 재생 미지원 | 사후 학습 콘텐츠 제공 불가 | 상 |
| 참석자 전용 콘텐츠 없음 | 체크인한 사람만 자료 접근하는 기능 없음 | 중 |
| 학습 완료 추적 없음 | 자료 다운로드/열람 기록 미관리 | 중 |
| 댓글/질문 기능 없음 | 세미나 관련 토론 공간 부재 | 중 |

## 3. 고도화 방안 (To-Be)

### Phase 1: 워크플로우 자동화 (Quick Win, 2~3주)

#### 3.1.1 세미나 생성 원스텝 자동화
**현재**: 세미나 생성 → 수동으로 타임라인 설정 → 수동으로 홍보 생성
**개선**: 세미나 생성 완료 시 자동으로:
- 타임라인 템플릿 자동 적용 (온/오프라인에 따라)
- AI 홍보 콘텐츠 5종 자동 생성 (백그라운드)
- 참석 신청 폼 기본 필드 자동 설정

```
[세미나 생성] ─→ [타임라인 자동 생성]
             ├→ [AI 홍보 5종 생성 (백그라운드)]
             ├→ [신청폼 기본 설정]
             └→ [운영진 알림 발송]
```

#### 3.1.2 세미나 복제 기능
- 세미나 목록에서 "복제" 버튼 추가
- 기본 정보(제목, 장소, 시간대, 발표자 틀) 복사
- 날짜만 변경하면 동일 형태 세미나 즉시 생성
- 타임라인 템플릿도 함께 복제

#### 3.1.3 임시저장 (Draft) 기능
- 세미나 생성 중 `status: "draft"` 상태로 자동 저장 (30초 간격)
- 세미나 목록에서 "임시저장" 필터 추가
- 드래프트 상태에서는 공개 페이지에 노출되지 않음

#### 3.1.4 타임라인 자동 알림
- D-day 기반 알림 자동 발송 (이메일 + 인앱 알림)
  - D-2: 참석자에게 리마인더 이메일 자동 발송
  - D-day: 체크인 QR 링크 포함 알림
  - D+1: 후기 작성 요청 + 감사 메시지
- 담당자 배정 시 인앱 알림
- 마감 임박 미완료 항목 경고

### Phase 2: 참석 경험 혁신 (3~4주)

#### 3.2.1 캘린더 원클릭 추가
- 참석 신청 시 Google Calendar / Apple Calendar 이벤트 자동 생성
- ICS 파일 다운로드 옵션
- 세미나 변경 시 캘린더 이벤트 자동 업데이트

#### 3.2.2 대기자 자동 전환
- 참석 취소 발생 시 대기자 1순위 자동 전환
- 전환 시 이메일 + 푸시 알림
- 24시간 내 미확인 시 다음 대기자로 자동 전환

#### 3.2.3 체크인 후 자동 프로세스
```
[체크인 완료] ─→ [출석 확인 알림 (참석자)]
             ├→ [실시간 대시보드 업데이트]
             └→ [세미나 종료 시]
                  ├→ [수료증 자동 발급]
                  ├→ [후기 작성 링크 발송]
                  └→ [연사 감사장 + 후기 요청 발송]
```

#### 3.2.4 실시간 참석 현황
- Firebase Realtime Database 또는 Firestore onSnapshot 활용
- 세미나 상세페이지에서 참석자 수 실시간 반영
- 체크인 대시보드 실시간 업데이트 (새로고침 불필요)

### Phase 3: 데이터 기반 운영 (4~5주)

#### 3.3.1 세미나 대시보드 (관리자 홈)
세미나 목록 탭을 대시보드 형태로 개편:

```
┌─────────────────────────────────────────────┐
│  세미나 관리 대시보드                          │
├─────────┬──────────┬──────────┬─────────────┤
│ 예정 3건 │ 진행 1건 │ 완료 24건 │ 총 참석 482명│
├─────────┴──────────┴──────────┴─────────────┤
│                                             │
│  [다가오는 세미나]                             │
│  ┌─ 4/25 AI교육 세미나 ─ 신청 32/40 ─ D-13  │
│  │  타임라인: ■■■■■□□ 5/7 완료              │
│  │  홍보: ✓보도자료 ✓SNS ○이메일             │
│  └──────────────────────────────────────     │
│                                             │
│  [성과 트렌드]                                │
│  평균 참석률: 78% (↑3%) | 후기율: 45% (↓2%)  │
│  평균 만족도: 4.3/5.0 (→)                    │
│                                             │
│  [빠른 작업]                                  │
│  [+ 새 세미나] [복제] [검색]                   │
└─────────────────────────────────────────────┘
```

**핵심 위젯:**
- **파이프라인 현황**: 신청 → 참석 → 체크인 → 수료 전환율
- **트렌드 차트**: 월별 참석자 수, 만족도 추이
- **운영 효율**: 타임라인 완료율, 미처리 항목 수
- **인기 키워드**: 관심 분야 워드클라우드

#### 3.3.2 발표자(연사) 데이터베이스
- 연사 프로필 DB 구축 (이름, 소속, 전문 분야, 사진, 약력)
- 과거 초청 이력 자동 기록
- 연사 후기의 "다음 세미나 추천 주제/연사" 데이터 집계
- 세미나 생성 시 연사 검색 및 자동 완성
- 연사 만족도 추적 (후기 평점 평균)

#### 3.3.3 참석자 인사이트
- 회원별 세미나 참석 이력 (마이페이지 연동)
- 관심 분야 기반 세미나 추천
- 충성 참석자 식별 (3회 이상 참석)
- 세미나별 참석자 프로필 분석 (학기, 관심 분야 분포)

#### 3.3.4 세미나 성과 리포트 자동화
- 세미나 종료 D+3에 자동 생성되는 성과 보고서:
  - 참석률, 체크인율, 후기 작성률
  - 만족도 점수 및 주요 피드백
  - 이전 세미나 대비 비교
  - 다음 세미나 개선 포인트 (AI 분석)

### Phase 4: LMS 고도화 (5~6주)

#### 3.4.1 영상 콘텐츠 지원
- YouTube/Vimeo 임베드 지원
- 녹화 영상 업로드 (Cloud Storage)
- 타임스탬프 기반 세션별 구간 이동
- 영상 시청 완료 추적

#### 3.4.2 참석자 전용 콘텐츠
- 체크인 완료한 참석자만 접근 가능한 자료 영역
- 발표자료 PDF, 참고 자료, 추가 학습 자료
- 다운로드 이력 추적

#### 3.4.3 Q&A / 토론 공간
- 세미나별 댓글/질문 게시판
- 실시간 질문 (라이브 세미나 중)
- 발표자 답변 하이라이트
- 인기 질문 상단 고정

#### 3.4.4 학습 완료 인증
- 영상 시청 + 자료 열람 + 후기 작성 = 학습 완료
- 학습 완료 시 디지털 배지 발급
- 마이페이지 학습 이력 통합 표시

### Phase 5: UX 개선 (병행 진행)

#### 3.5.1 모바일 최적화
- 세미나 관리 모바일 반응형 개선 (현재 데스크톱 최적화)
- 체크인 페이지 모바일 UX 강화
- 터치 친화적 타임라인 관리

#### 3.5.2 알림 센터 통합
- 인앱 알림 센터 (헤더 벨 아이콘)
- 알림 유형: 세미나 리마인더, 후기 요청, 수료증 발급, 대기자 전환
- 이메일 + 인앱 + 푸시(PWA) 멀티채널

#### 3.5.3 검색 및 필터
- 세미나 목록 통합 검색 (제목, 발표자, 주제)
- 날짜 범위 필터, 상태 필터, 태그 필터
- 정렬 옵션 (최신순, 참석자순, 만족도순)

#### 3.5.4 접근성 개선
- WCAG 2.1 AA 기준 준수
- 키보드 네비게이션 지원
- 스크린리더 호환성

## 5. 자동화 워크플로우 전체 흐름도

```
세미나 라이프사이클 자동화 (To-Be)
═══════════════════════════════════════════════════════

[1. 기획]
  세미나 생성 (또는 복제)
  ├→ 타임라인 자동 생성
  ├→ AI 홍보 콘텐츠 5종 백그라운드 생성
  ├→ 신청폼 기본 설정
  └→ 운영진 알림: "새 세미나가 생성되었습니다"

[2. 홍보] (D-30 ~ D-7)
  타임라인 자동 알림 → 담당자에게 홍보 시작 알림
  ├→ 보도자료 자동 게시 (승인 후)
  ├→ 이메일 초대장 → 뉴스레터 시스템 연동 발송
  ├→ SNS/카카오 콘텐츠 → 클립보드 복사 안내
  └→ 포스터 → 세미나 상세페이지 자동 반영

[3. 접수] (D-30 ~ D-1)
  참석 신청 접수
  ├→ 참석자에게 확인 이메일 자동 발송
  ├→ 캘린더 이벤트 추가 안내 (ICS 첨부)
  ├→ 정원 초과 시 대기열 자동 관리
  │   └→ 취소 발생 시 대기자 자동 전환 + 알림
  └→ 운영진 대시보드 실시간 반영

[4. 리마인더] (D-2)
  자동 발송
  ├→ 참석자: 일시/장소/QR체크인 링크
  ├→ 운영진: 준비사항 체크리스트
  └→ 발표자: 세션 정보 + 자료 업로드 안내

[5. 당일 운영] (D-day)
  체크인 (QR / 셀프)
  ├→ 실시간 대시보드 업데이트
  ├→ 출석 확인 알림 (참석자)
  └→ 운영진 사진 업로드 → 포토 앨범 자동 연동

[6. 사후 관리] (D+1 ~)
  세미나 종료 감지
  ├→ 수료증 자동 일괄 발급 (체크인자)
  ├→ 감사장 발급 알림 (운영진에게)
  ├→ 후기 작성 요청 발송 (참석자 + 연사)
  ├→ 연사 감사 메시지 + 감사장 + 포토갤러리 링크
  ├→ D+3: 성과 리포트 자동 생성
  └→ LMS 콘텐츠 공개 (영상 + 자료)

[7. 분석]
  대시보드 자동 집계
  ├→ 참석률/후기율/만족도 트렌드
  ├→ 연사 추천 데이터 축적
  ├→ 인기 주제 분석
  └→ 다음 세미나 기획 인사이트 제공
```

## 6. 기술 구현 가이드

### 6.1 자동화 엔진 (Cron Jobs)

현재 `api/cron/seminar-reminder`와 `api/cron/seminar-status`가 존재합니다.
추가 Cron Job 제안:

| Cron | 주기 | 역할 |
|------|------|------|
| `seminar-auto-notify` | 매일 09:00 | D-day 기반 자동 알림 발송 |
| `waitlist-auto-convert` | 매 1시간 | 대기자 자동 전환 처리 |
| `certificate-auto-issue` | 매일 22:00 | 당일 종료 세미나 수료증 일괄 발급 |
| `report-auto-generate` | 매일 09:00 | D+3 세미나 성과 리포트 생성 |

### 6.2 데이터 모델 확장

```typescript
// 발표자 DB
interface Speaker {
  id: string;
  name: string;
  affiliation: string;
  position: string;
  bio: string;
  photoUrl?: string;
  expertise: string[];
  seminarHistory: { seminarId: string; date: string; rating?: number }[];
}

// 알림
interface Notification {
  id: string;
  userId: string;
  type: "reminder" | "review_request" | "certificate" | "waitlist" | "assignment";
  title: string;
  message: string;
  link?: string;
  read: boolean;
  createdAt: string;
}

// 학습 진도
interface LearningProgress {
  id: string;
  userId: string;
  seminarId: string;
  videoWatched: boolean;
  materialsDownloaded: string[];
  reviewWritten: boolean;
  completedAt?: string;
}
```

### 6.3 외부 연동

| 서비스 | 용도 | 구현 방식 |
|--------|------|-----------|
| Google Calendar API | 캘린더 이벤트 추가 | ICS 파일 생성 (서버 불필요) |

| YouTube Data API | 영상 임베드 | iframe + oEmbed |

## 9. 결론

연세교육공학회 세미나 시스템은 이미 **기획→홍보→접수→운영→사후관리**의 완전한 라이프사이클을 지원하는 높은 완성도를 보유하고 있습니다. 본 고도화 프로젝트의 핵심은 **각 단계 사이의 수동 연결을 자동화**하고, **축적된 데이터를 기반으로 지속적 개선 루프를 구축**하는 것입니다.

특히 Phase 1의 **세미나 복제, 임시저장, 타임라인 자동 적용**은 낮은 구현 비용으로 즉시 체감 가능한 효율 개선을 가져올 수 있어, 가장 먼저 착수할 것을 권장합니다.
