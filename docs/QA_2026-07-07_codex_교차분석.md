# codex 교차 점검 × QA v3 교차 분석 (2026-07-07)

- 방법: codex CLI(독립 2차 감사자, 읽기 전용)로 ① 전수 코드 점검 ② 서비스 갭 분석 → QA v3(2026-07-05) 결과와 대조. codex 상위 발견은 오케스트레이터가 룰·코드·운영 인덱스에서 직접 재검증.
- 핵심: QA v3는 "쿼리가 룰을 **증명 못 해 깨짐**"(과소권한) + 특정 API 스푸핑을 봤고, codex는 "룰이 코드 의도보다 **넓어 위조 가능**"(과대권한)을 체계적으로 봄 → 상보적, 겹침 적음. codex가 **쓰기 위조 계열 룰 결함**과 **추가 인덱스 누락**을 새로 발견.

## 🔴 Critical (신규 — QA v3 미발견, 검증 완료)

1. ✔ **로그인 회원이 임의 회원의 users 문서 원본 get 가능** — `firestore.rules:62` `allow get: if isAuthenticated()`. QA v3는 비로그인 get만 막았고 **로그인 회원 간**은 열려 있음 → 승인·미승인 회원이 프로필 UID로 `getDoc(users/타인uid)` 하면 securityAnswerHash·birthDate·calendarToken·학번·전화 등 투영 API가 감추는 필드 전부 노출. 수정: get 을 본인/staff 로 제한하고 타인 프로필은 `/api/profile/[id]/public` 경유 강제.
2. ✔ **세미나 참석·수료증 위조** — `firestore.rules:195` `allow create: if isAuthenticated()` (userId·checkedIn 무검증). 회원이 `addDoc(seminar_attendees,{seminarId, userId:본인, checkedIn:true})` 직접 생성 → seminar-status 크론이 checkedIn 참석자로 보고 수료증·후기요청 발급. 수정: create 시 `userId==auth.uid && checkedIn==false && !("checkedInBy" in data)` 강제(체크인은 스태프/토큰 경로만).

## 🟠 High (신규 보안 룰 — 쓰기 위조 계열, 검증 완료)

3. ✔ **회비 납부내역 전 회원 열람** — `firestore.rules:355` `fee_payments read: if isAuthenticated()`. 일반·미승인 계정이 `getDocs(fee_payments)` → 전원 납부상태·금액·학번·메모 노출. 수정: read 를 본인/staff 로.
4. ✔ **타인에게 알림 스푸핑 (룰 레벨)** — `firestore.rules:408` `notifications create: if isAuthenticated()` (대상 userId 무검증). QA v3는 waitlist-notify API 하나만 막았으나 룰이 열려 있어 아무 type·link·메시지의 알림을 타인 알림센터에 주입 가능. 수정: create 시 `userId==auth.uid || isStaffOrAbove()`(서버 fan-out은 Admin SDK라 무관).
5. ✔ **리더보드 점수 조작** — `firestore.rules:1324` streak_events create 가 `userId==본인`만 보고 `points` 무검증 → `points:1000000` 직접 생성 시 리더보드 합산. 수정: points 상한·type 화이트리스트, 또는 점수 적립을 서버 경로로.
6. ✔ **활동 신청/참여 배열 위조·삭제** — `firestore.rules:333` `isApplyOnlyUpdate()`가 applicants/participants 를 **통째 덮어쓰기** 허용(append-only 아님) + `isGuestApplyUpdate()`는 인증 체크 없이 호출 → 비로그인도 `applicants:[]`로 신청자 전체 삭제, 회원이 participants 임의 조작. 수정: 배열 변경을 서버 라우트로 몰거나 append-only 증명 추가.
7. ✔ **활동 참여 API 신원 미대조** — `src/app/api/activities/route.ts:103` PATCH 가 `requireAuth` 후 body 의 joinUserId/leaveUserId 를 인증 사용자와 대조하지 않음 → `PATCH {id, leaveUserId:타인}`으로 타인을 활동에서 탈퇴시킴. 수정: `joinUserId/leaveUserId === authResult.id` 강제(스태프 예외).

## 🟠 High (기능 — 복합 인덱스 누락, 운영 대조 확인)

8. ✔ **복합 인덱스 누락 다수 — 운영에 없음 확인** — `parseSort`가 실제 Firestore `orderBy`를 생성하므로 `where(A)+orderBy(다른필드)` 쿼리는 인덱스 필수. 운영 인덱스 조회 결과 아래 전부 **없음**:
   - `onboarding_checklist(enabled==, order asc)` — **방금 시드한 신규 온보딩 항목이 위젯에 안 뜸** (실사용 파손 확정)
   - `course_reviews(courseOfferingId/authorId, createdAt desc)` — 강의후기·신규회원 체크리스트 로드 실패
   - `seminar_waitlist(seminarId, position asc)` — 대기열 표시·자동 승격 실패
   - `roadmap_stages(published==, order asc)` — 로드맵 CMS 대신 fallback
   - `business_card_exchanges`, `promotion_contents(seminarId, createdAt desc)`, `interview_response_comments(responseId, createdAt asc)`, `agent_workflow_runs(userId, createdAt desc)`
   - QA v3에서 10종 추가했으나 이 계열은 잔여. 수정: reconcile 스크립트 REQUIRED 확장 → 생성·파일 동기화. (일부는 서버 정렬 제거+클라 정렬로 회피 가능)

## 🟡 Medium (신규, 검증 완료/유력)

9. ✔ **`filter[field][in]` 미지원 → 전체 로드** — `src/lib/bkend.ts:139` parseFilters 가 `[in]` 형식을 무시 → `classSessionsApi.listByCourses`의 courseOfferingId in 쿼리가 class_sessions 전체 1000건 로드, 내 수업 일정이 남의 일정까지 섞임/누락.
10. **guide_items published 필터 부재** — `bkend.ts:2126` 회원 쿼리가 trackId 단독 → 룰은 published==true 만 허용이라 `/steppingstone/onboarding` permission-denied (추정, 룰 대조 필요).
11. **게시글/세미나 등록 폼 더블클릭 중복** — `PostForm.tsx:496`, `SeminarForm.tsx:488` submit 재진입 미차단 → 공지/세미나 2건 + fan-out 2회(전 회원 중복 알림). QA v3는 참석 토글만 고침, 생성 폼은 미점검.
12. **weekly-digest 중복 방지 비원자적** — `weekly-digest/route.ts:649` email_logs 조회→발송이 원자 예약 아님 → 재시도·수동 동시 호출 시 중복 발송.
13. **대기열 순번 비트랜잭션** — `useSeminar.ts:224` maxPos+1 동시 저장 시 동일 순번.

## 서비스 갭 (codex 갭 분석 → 검증)

**유효한 갭**
- **심사·디펜스 단계가 "연습" 편중, 운영/제출/결과 관리 부재** — 개인별 심사 일정·위원·제출물·수정결과 보드 없음. 논문학기생·운영진 공통. (여정 갭 중 가장 실질적)
- **progress-meetings 허브 진입 약함** — 생성은 되나 메인 nav·활동 페이지에서 발견 경로 없음(InlineMeetingTimer 외).
- **content_creations 검증 큐 부재** — 포트폴리오에서 사실상 verified처럼 표시되나 콘솔 검증 경로 없음(certificate route만 소비).
- **온보딩 visited가 localStorage 국한** — 운영진이 완료율 집계 불가(서버 미러 없음).

- 디펜스 연습: ThesisJourney stage5·ResearchJourneyGuide·커맨드팔레트에 CTA 존재.
