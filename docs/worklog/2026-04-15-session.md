### 3. Service Worker 캐시 전략 수정 (`public/sw.js`)
- **문제**: 모바일에서 디자인 업데이트 미반영
- **원인**: v1이 HTML(`/`)을 cache-first로 캐싱 → 옛날 페이지 영구 노출
- **해결**:
  - `CACHE_NAME` v1 → v2 (기존 캐시 무효화)
  - HTML 요청(navigate / `Accept: text/html`)은 **network-first**, 실패 시 캐시 폴백
  - 정적 자산은 기존대로 cache-first 유지
  - STATIC_ASSETS에서 `/` 제거 (HTML 강제 캐시 방지)
