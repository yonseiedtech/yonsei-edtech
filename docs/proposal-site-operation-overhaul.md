# 연세교육공학회 사이트 서비스 개편 제안서

> **부제**: 학회 운영 효율화를 위한 통합 플랫폼 재설계
> **작성일**: 2026-04-13
> **대상**: 회장단 · 운영진 · 지도교수
> **범위**: yonsei-edtech.vercel.app 운영 전반 (관리자 / 운영자 / 회원)

---

## 1. 추진 배경 (Why Now)

연세교육공학회 사이트는 지난 3년간 공개 홍보 · 회원 관리 · 학술 활동을 하나의 Next.js 앱으로 빠르게 확장해왔습니다. 그 결과 기능은 풍성해졌지만 **운영자 1인당 반복 업무 시간이 기수당 평균 15~20시간**에 달하며, 회장단 교체 시마다 인수인계 공백으로 인한 학습 비용이 2~3주 발생하고 있습니다.

본 제안서는 "**운영진이 반복 업무에 쓰는 시간을 절반으로 줄이고, 회원의 학회 생활을 단절 없이 연결하는**" 것을 목표로 단기·중기·장기 로드맵을 정의합니다.

---

## 2. 현황 진단 (As-Is)

### 2.1 정량 지표
| 지표 | 현재 | 개선 목표 |
|------|:----:|:---------:|
| 회비 납부 확인 소요 시간 (분기 1회) | 약 4시간 (엑셀 대조) | **20분 이내** |
| 수료증·감사장 발급 소요 시간 (세미나 1회) | 약 3시간 (수기) | **자동화 (0분)** |
| 신규 회원 승인 1건당 클릭 수 | 평균 7회 | **2회 이하** |
| 회장단 인수인계 학습 기간 | 2~3주 | **3일 이내** |
| 문의 응답 평균 시간 | 36시간 | **6시간 이내** |

### 2.2 정성 문제
- **경로 파편화**: `/admin`, `/academic-admin`, `/staff-admin` 3개 허브로 분산되어 운영진이 "어디서 뭘 하는지" 학습에 장기간 소요
- **일괄 처리 부재**: 회원 승인·회비 확인·세미나 상태 변경이 모두 1건씩 처리되는 구조
- **인수인계 공백**: Handover 필드는 스키마에 있으나 UI/자동 리포트 미지원
- **데이터 활용 미흡**: 직업군·학번·후기·참여율 등 수집된 필드가 대시보드/추천에 연결되지 않음
- **회원 여정 단절**: 회원가입 → 첫 활동 → 수료증 수령까지의 온보딩 연결 없음

## 3. 개편 비전 (To-Be)

### 3.1 3대 원칙
1. **원스톱 (One-Stop)** — 운영진은 `/admin` 단일 허브에서 모든 업무를 완결
2. **자동화 (Zero-Touch)** — 반복되는 문서·메일·통계는 시스템이 대신 생성
3. **연결 (Connected Journey)** — 회원은 가입부터 수료까지 하나의 타임라인에서 경험

### 3.2 핵심 서비스 4축
```
┌────────────────────────────────────────────────────────┐
│  [축 1] 통합 관리자 허브  (운영 효율화의 중심)         │
│  [축 2] AI 운영 비서     (문서·보도자료·보고서 자동화) │
│  [축 3] 회원 타임라인    (내 학회생활 대시보드)        │
│  [축 4] 인수인계 자산관리 (회장단 교체 리스크 제거)    │
└────────────────────────────────────────────────────────┘
```

## 4. 세부 과제 (우선순위별)

### 🟢 P0 — Quick Wins (2주 스프린트)
| 과제 | 기대 효과 | 예상 공수 |
|------|-----------|:--------:|
| 관리자 허브 단일화 (`/admin` 탭 통합) | 학습 기간 70% 단축 | 3일 |
| Empty State · Loading Skeleton 표준화 | UX 일관성 + 이탈률 감소 | 2일 |
| Footer 회원 바로가기 + Ctrl+K 통합 검색 | 고빈도 기능 접근성 상승 | 2일 |
| 관리자 대시보드 대기건 배지 (승인/문의/미납) | 누락 업무 즉시 인지 | 1일 |

### 🟡 P1 — 운영 자동화 (4~6주)
| 과제 | 기대 효과 | 예상 공수 |
|------|-----------|:--------:|
| 수료증·감사장 PDF 자동 발급 + 일괄 이메일 | 기수당 8시간 절감 | 1.5주 |
| 회원 일괄 승인 (체크박스 + 1클릭) | 승인 업무 80% 단축 | 3일 |
| 회비 납부 자동 대조 (엑셀 업로드 → 파싱 → 매칭) | 분기당 4시간 → 20분 | 1주 |
| 문의 AI 답변 초안 + 일괄 발송 | 응답 시간 36h → 6h | 1주 |
| 인수인계 Markdown 에디터 + 기수 리포트 PDF | 회장단 교체 리스크 제거 | 1주 |

### 🔵 P2 — 회원 경험 강화 (6~10주)
| 과제 | 기대 효과 | 예상 공수 |
|------|-----------|:--------:|
| `/mypage` 타임라인 대시보드 (회비·활동·수료증 한눈에) | 회원 리텐션 +20% | 2주 |
| 추천 활동 엔진 (직업군·관심분야 기반) | 참여율 +15% | 2주 |
| 통합 검색 (회원·자료·댓글 확장) | 자료 재활용률 상승 | 1.5주 |
| AI 포스터·보도자료 자동 생성 | 홍보 업무 60% 절감 | 3주 |

### 🟣 P3 — 플랫폼화 (분기 단위)
| 과제 | 기대 효과 | 예상 공수 |
|------|-----------|:--------:|
| 운영 애널리틱스 대시보드 (참여율·납부율·조직 부하도) | 데이터 기반 의사결정 | 3주 |
| 디자인 시스템 v2 (토큰·다크모드·Storybook) | 장기 유지보수 비용 감소 | 4주 |
| 챗봇 FAQ + 과거 Q&A 임베딩 | 문의 30% 자체 해결 | 3주 |

## 6. 재사용 가능 자산 (이미 확보)

본 개편은 **새로 만드는 것보다 기존 자산을 연결하고 다듬는 비중이 70%** 이상입니다.

- `useMembers`, `useActivities`, `useOrgChart`, `useBoard` — React Query 훅 완비
- `requireAuth`, `siteSettingsApi` — 인증·설정 표준 경로
- `AppreciationCertificate` — 감사장 템플릿 (html2canvas + jsPDF)
- `/api/search`, `EmptyState`, `GlobalSearch` — 이미 구축 (2026-04 업데이트)
- shadcn/ui 컴포넌트셋 (Dialog, Sheet, Skeleton, Badge 등) 완비
