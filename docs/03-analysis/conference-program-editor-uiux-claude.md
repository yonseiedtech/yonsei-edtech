> 작성일: 2026-05-06  
> 프레임워크: Next.js 16 (App Router) + React 19 + Tailwind CSS v4 + shadcn/ui + framer-motion + @dnd-kit

## 1. 현황 진단 (디자이너 시각)

### 1-1. 편집기 위계 및 정보 밀도

현재 편집기는 **단일 긴 세로 스크롤** 구조다. 최상단 "프로그램 메타" 카드(제목·원본 자료·AI 추출·안내 메모) → 일자 카드 배열(N개) → 각 일자 내부에 SessionRow 반복이라는 3단 위계를 갖는다.

문제는 **일자가 3개 이상, 세션이 20개 이상**이 되는 순간 편집기가 끝없는 폼 목록으로 변한다는 점이다. 운영진은 특정 세션을 수정하려면 페이지를 수십 개 스크롤해야 한다. 현재 구조에서 "일자 N의 세션 K를 수정한다"는 동선에 네비게이션 수단이 없다. 일자 탭이나 앵커 점프가 없어 인지 부하가 크다.

SessionRow 내부의 정보 밀도도 문제다. 한 행에 시작/종료 시각, 카테고리 select, 트랙 입력, 이동/삭제 버튼이 가로로 나열되고, 그 아래에 제목·발표자·소속·장소·초록·자료 URL까지 총 7~8개 필드가 펼쳐진다. 세션 하나가 최대 약 200px 세로 높이를 차지하며, 세션이 많을수록 편집기 전체가 매우 길어진다.

### 1-2. 입력 동선

- **시간 입력**: HTML `<input type="time">`을 직접 사용한다. 브라우저 기본 time picker는 운영체제마다 UI가 달라서 일관성이 없고, 특히 크롬에서는 AM/PM 포맷 혼동, 키보드 입력 시 포커스 이동 등 미세한 불편이 있다.
- **발표자 입력**: 쉼표 구분 문자열(plain text)로 처리한다. "홍길동, 김철수"처럼 입력하면 배열로 분리한다. 발표자가 많거나 이름에 쉼표가 포함될 경우 오입력 가능성이 있고, 개별 발표자의 소속·역할을 따로 지정할 수 없다.
- **자료 URL 입력**: Textarea에 줄바꿈으로 구분해 여러 URL을 입력한다. `/^https?:\/\//` 검증을 인라인에서 수행하지만 유효하지 않은 URL은 조용히 필터링되어 사용자가 인식하기 어렵다.
- **카테고리**: 기본 HTML `<select>`를 사용한다. 색상이 카테고리별로 정의되어 있지만 select 옵션 내에서는 색상이 표시되지 않는다.

### 1-3. 토큰 일관성

- `Card` + `CardHeader` + `CardContent`는 shadcn 패턴을 일관되게 따른다.
- `SessionRow`는 `rounded-lg border bg-muted/20 p-3` 패턴을 사용해 shadcn Card와 미묘하게 다르다. 이중 경계선(Card 안에 muted border 박스)이 시각적으로 복잡해 보인다.
- 버튼 크기가 `size="sm"` / `h-7 w-7 p-0` / `h-8` 등 여러 단계가 혼재한다.
- 저장/일자추가 버튼은 중간 구분선 없이 나란히 위치하여 "저장"의 중요도가 시각적으로 강조되지 않는다.

### 1-4. 모바일 vs 데스크톱 적합성

운영진은 학술대회 책자를 PDF나 사진으로 받은 뒤 PC 환경에서 입력하는 경우가 대부분이다. 현재 레이아웃(`max-w-5xl`, `sm:grid-cols-2`)은 데스크톱에 최적화되어 있다.

모바일에서 SessionRow 첫 줄(시간 2개 + 카테고리 select + 트랙 + 이동/삭제 버튼)은 `flex-wrap`으로 처리되지만, 너비 640px 미만에서는 버튼들이 2~3줄로 내려가 정렬이 깨진다. 특히 시간 입력 필드(`w-28`)가 모바일 키보드에서 조작하기 어렵다.

### 1-5. 운영진의 인지 부하

- 첫 등록 시 "일자 추가 → 세션 추가 → 세션 내용 입력"의 3단계가 명시적이지 않다. 빈 상태에서 "첫 일자 추가" 버튼만 표시되고, 추가 후에야 세션 입력 폼이 나타나는 점프가 있다.
- AI 추출 기능이 "프로그램 메타" 카드 내 필드 중 하나로 배치되어 있어 기능의 위상이 낮아 보인다. AI로 전체 시간표를 자동 구성하는 고가치 기능임에도 발견하기 어렵다.
- 저장 시 성공/실패 피드백은 `savedAt` 타임스탬프 텍스트 표시에 의존한다. 자동 저장 없이 수동 저장만 지원하므로 운영진이 저장을 잊을 경우 데이터 소실 위험이 있다.
- 세션 순서 변경이 ChevronUp/ChevronDown 버튼으로만 가능하다. 세션이 10개 이상이면 순서 조정에 클릭을 수십 번 해야 한다.

## 2. 디자이너 관점 UI 개선점 7건

### 개선 1. 일자 탭 네비게이션 고정

편집기 상단에 일자별 수평 탭을 고정(sticky)해 현재 편집 중인 일자의 세션만 표시한다. ConferenceProgramView의 sticky 탭 패턴을 편집기에도 적용한다. 일자가 5개 이상이면 탭을 `overflow-x-auto`로 스크롤 처리한다.

```
[1일차 · 2026-05-09 (12개)] [2일차 · 2026-05-10 (18개)] [+일자 추가]
```

편집기를 탭 단위로 분할하면 한 화면에 표시되는 세션 수가 제한되어 인지 부하가 크게 줄어든다.

### 개선 2. SessionRow → SessionCard 접기/펼치기

세션 목록에서 기본적으로 제목·시간·카테고리만 표시하는 **접힌(collapsed) 상태**를 기본값으로 하고, 클릭하면 상세 필드(발표자·소속·초록·자료 URL)가 펼쳐지는 패턴으로 전환한다. 이렇게 하면 세션 20개가 있어도 한 화면에 목록이 보인다.

접힌 상태에서 카테고리 색상 배지, 시간, 제목, 트랙 배지를 한 줄에 표시한다. 오른쪽에 편집(연필 아이콘)·삭제 아이콘을 배치한다.

### 개선 3. 카테고리 색상 배지를 Select 대신 ColorPicker 버튼 그리드로 교체

현재 기본 `<select>`는 카테고리 색상을 시각화하지 못한다. 11개 카테고리를 색상 원형 배지 그리드로 표시하는 팝오버로 교체한다. 선택된 카테고리는 강조 ring으로 표시한다.

```
[기조강연(보라)] [심포지엄(파랑)] [패널(인디고)] ... (2~3열 그리드)
```

이미 `CONFERENCE_SESSION_CATEGORY_COLORS`에 색상이 정의되어 있으므로 구현 비용이 낮다.

### 개선 4. 시간 입력 UX — 드롭다운 슬롯 선택 + 텍스트 병행

30분 단위 슬롯을 드롭다운으로 빠르게 선택할 수 있도록 하되, 직접 입력도 허용한다. 예를 들어 `<select>` + 텍스트 입력 혼합 컴포넌트 `TimeSlotInput`을 만들어 "09:00, 09:30, 10:00..." 형식으로 빠른 선택을 제공한다. 시작 시각 선택 후 종료 시각의 기본값을 자동으로 +60분으로 설정하면 입력 속도가 향상된다.

### 개선 5. 발표자 입력 — 태그 입력(Multi-chip Input) 패턴

쉼표 구분 문자열 대신 Enter/Tab/쉼표 입력 시 태그(chip)로 확정되는 multi-chip input으로 교체한다. 각 발표자 이름이 배지로 표시되고 × 버튼으로 개별 삭제 가능하다. 입력 오류 방지와 시각적 피드백 개선 효과가 크다. dnd-kit이 이미 의존성에 있으므로 태그 순서 드래그 재정렬도 가능하다.

### 개선 6. 세션 드래그 앤 드롭 순서 변경

@dnd-kit/sortable이 패키지에 포함되어 있다. `SortableContext`로 세션 목록을 감싸면 드래그 핸들로 순서를 직관적으로 변경할 수 있다. 현재의 ChevronUp/Down 버튼 클릭 방식은 세션 10개 이상에서 비효율적이다.

### 개선 7. 인라인 시간 충돌 경고 (편집기 내)

ConferenceProgramView에는 세션 간 시간 충돌 감지 로직(`conflictsBySessionId`)이 구현되어 있지만 편집기에는 없다. 같은 트랙 내 세션 시간이 겹치는 경우 SessionCard에 경고 배지를 표시하고, 저장 시 충돌 목록을 요약해 toast로 알린다.

## 3. UX 흐름 개선점 5건

### 개선 A. 첫 등록 온보딩 플로우

현재 빈 상태에서는 "첫 일자 추가" 버튼 하나만 보인다. 첫 등록 시 2단계 온보딩을 제안한다.

- **1단계 선택**: "AI로 자동 추출" vs "직접 입력" 두 경로를 카드 UI로 분기
- **2단계**: AI 경로는 파일 업로드 후 추출 결과 검토 → 저장, 직접 입력 경로는 일자 수와 날짜 범위를 먼저 설정한 뒤 세션 입력

이 구조는 AI 추출 기능의 발견 가능성을 크게 높이고, 직접 입력 시에도 일자 골격을 먼저 만들어 주어 편집 효율을 높인다.

### 개선 B. 자동 저장 (Debounced Auto-save)

입력이 멈춘 후 3초 뒤 자동으로 `handleSave()`를 호출하는 debounce 자동 저장을 추가한다. 저장 상태는 상단 상태 표시줄에 "저장 중..." / "저장됨 · 12:34" / "저장 실패 — 재시도"로 표시한다. 운영진이 브라우저를 실수로 닫아도 최대 3초 분량만 소실된다.

### 개선 C. 트랙 매트릭스 뷰 (선택적 보기 모드 전환)

세션이 많고 트랙이 여러 개인 대형 학회를 위한 **매트릭스 뷰** 토글을 추가한다. 가로축 트랙, 세로축 시간대로 구성된 그리드에서 세션 블록을 확인하며 편집할 수 있다. 이 뷰는 시간표 책자와 동일한 구조이므로 원본 PDF를 보면서 입력 오류를 검증하기 쉽다. 구현 난이도가 높으므로 Phase 2 이후로 배정한다.

### 개선 D. 일괄 가져오기 — CSV 템플릿

AI OCR 외에 **CSV 가져오기** 경로를 추가한다. 다운로드 가능한 CSV 템플릿(날짜, 시작시각, 종료시각, 트랙, 카테고리, 제목, 발표자, 소속, 장소)을 제공하고, 운영진이 Excel에서 채운 뒤 업로드하면 시간표가 일괄 생성된다. 학회 측에서 Excel 형식의 프로그램을 미리 보유하고 있는 경우가 많아 실용성이 높다.

### 개선 E. 저장 전 일관성 검증 + 요약 패널

저장 버튼 클릭 시 아래 항목을 검증하고 요약 모달을 표시한다.

- 세션 제목이 빈 경우 목록 표시
- 시작 시각이 종료 시각보다 늦은 세션 목록
- 같은 트랙 내 시간 겹침 세션 쌍
- 발표자 없는 논문/포스터 발표 세션 목록

검증 결과를 "경고 N건 — 그래도 저장" / "수정 후 저장" 두 가지 선택지로 제시한다. 데이터 품질을 운영진이 저장 전에 인지할 수 있다.

    │

**Quick Win 구역** (높은 영향도 + 낮은 난이도): 일자 탭, 카테고리 피커, 시간 슬롯, SessionCard 접기, 자동 저장

## 6. 즉시 적용 Quick Wins 5선

### QW-1. DayTabBar — 일자 탭 고정 (1~2시간)

ConferenceProgramView의 sticky 탭 패턴을 Editor에 그대로 이식한다. 탭 클릭 시 해당 일자 세션만 렌더링(`activeDayIdx` state). 성능도 개선된다.

### QW-2. SessionCard 기본 접힘 상태 (2~3시간)

SessionRow에 `useState(false)` expanded 플래그를 추가한다. 기본 상태에서 제목·시간·카테고리·트랙만 표시, 클릭 시 전체 폼 펼침. framer-motion `AnimatePresence` + `motion.div`로 부드러운 토글 애니메이션 적용 (의존성 이미 포함).

### QW-3. CategoryPickerButton — 색상 배지 팝오버 (1~2시간)

기존 `<select>` 대신 shadcn `Popover` + `Button` 그리드 조합으로 카테고리 선택기를 구현한다. `CONFERENCE_SESSION_CATEGORY_COLORS` 값을 그대로 활용한다.

### QW-4. AutoSaveIndicator + Debounced Save (2시간)

`useRef`로 debounce 타이머를 관리한다. `draft` state 변경 시 3초 debounce 후 `handleSave()` 자동 호출. 상태 문자열(`idle | saving | saved | error`)을 헤더 우측에 작은 텍스트로 표시.

### QW-5. 저장 전 시간 역전 경고 (1시간)

`handleSave()` 내부에서 `days.flatMap(d => d.sessions).filter(s => s.startTime >= s.endTime)` 검사를 추가한다. 해당 세션이 있으면 `toast.warning()`으로 목록을 표시하고 저장은 계속 진행한다(강제 차단 아님).
