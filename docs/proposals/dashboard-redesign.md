## 1. Executive Summary

`/dashboard`는 현재 통계·최근활동·미니 캘린더 등 7개 섹션이 평행 나열되어 있어 "지금 내가 뭘 해야 하는지" 한눈에 보이지 않습니다. 이번 IA 변경으로 **마이페이지의 활동 4종(학술활동·수료증·내 글·인터뷰)이 `/mypage/activities`로 분리**되었으므로, 대시보드는 더 이상 "내 활동 모음"이 아니라 **학회 전체의 운영 상황을 빠르게 파악·행동 유도**하는 운영 보드로 역할을 재정의합니다.

핵심 원칙 3가지:
1. **Above-the-fold 3초 룰** — 화면 진입 후 3초 안에 "오늘 봐야 할 것"이 눈에 들어와야 함
2. **Role-aware visibility** — 일반 회원/운영진/학회장에게 같은 그리드를 보여주되 위젯 priority가 다름
3. **Empty state는 행동 유도** — "데이터 없음" 대신 "지금 해볼 수 있는 일"을 안내

## 2. 현황 분석 (As-Is)

### 2-1. 위젯 인벤토리 (현재 `/dashboard`)

| # | 섹션 | 데이터 소스 | 현재 비중 | 문제점 |
|---|------|------------|----------|--------|
| 1 | 인사말 카드 | static | 상단 100% | 매일 같은 정보, 감정적 가치 |
| 2 | 통계 4종 (회원/세미나/활동/문의) | 다중 API | 25% | 추세·비교 없음 → 의사결정 불가 |
| 3 | 다가오는 세미나 | seminars | 33% | "내가 신청한" 표시 없음 |
| 4 | 최근 게시글 | posts | 33% | 게시판 첫 화면과 중복 |
| 5 | 미니 캘린더 | seminars+activities | 33% | 클릭 시 풀 캘린더 이동만 |
| 6 | 학술활동 진행률 | activities | 25% | 모수가 모호 |
| 7 | 운영진 To-Do (staff+) | todos | 50% | 콘솔과 중복, depth 부족 |

### 2-2. 사용자 페인 포인트
- **분리된 동선**: "내 활동 보러 가야 하는데 대시보드/마이페이지 어디?" → §IA 변경으로 1차 해결됨
- **정보 밀도 vs 의사결정**: 숫자만 있고 "그래서?"가 없음 — 신청 전환율, 미열람 알림 등 행동 단서 부족
- **모바일 적합도**: 데스크탑 4-그리드가 모바일에서 1열로 늘어지면서 스크롤 피로
- **운영진 워크플로우 단절**: 콘솔(/console)과 대시보드의 역할 중복 — "대시보드에서 To-Do를 볼 수 있는데 왜 또 콘솔로 가야 하나?"

- **콘텐츠 발견율** (대시보드 → 게시글/활동 클릭) — 12% → 25%
- **운영진 평균 응답 시간** (문의 등록 → 답변) — 24h → 6h

## 5. 위젯 카탈로그

### P0-1: 이번 주 핵심 활동 (NEW)
- **목적**: "지금 봐야 할 가장 중요한 3가지"를 한눈에
- **데이터**: seminars(date≤+7d) + interview posts(deadline≤+7d) + notice posts(unread)
- **표시**: 카드 3장(다가오는 세미나 / 마감 임박 인터뷰 / 미열람 공지) — 각 카드에 1차 CTA 버튼
- **인터랙션**: 클릭 → 해당 상세, 더보기 → 각 list 페이지
- **우선순위**: P0
- **모든 역할**

### P0-2: 운영진 워크보드 (NEW, role-gated)
- **목적**: 콘솔에 들어가지 않고도 "오늘의 처리 항목"을 즉시 인지
- **데이터**: inquiries(pending) + members(approved=false) + todos(due_today)
- **표시**: 3개 라인 — `[badge] 미응답 문의 3건` `[badge] 승인 대기 2명` `[ ] 오늘의 To-Do 4건`
- **인터랙션**: 라인 클릭 → 콘솔 해당 페이지로 deep link
- **우선순위**: P0
- **운영진+ 만**

### P0-3: 학회장 인사이트 (NEW, role-gated)
- **목적**: 거시 지표로 학회 건강도 판단
- **데이터**: profiles(activity log 30d) + seminars(attendance rate trend) + signups(7d/30d)
- **표시**: sparkline 3종 + 전주 대비 % 변화
- **우선순위**: P0
- **학회장 만**

### P1-1: 나의 다음 액션
- **목적**: "내가 신청은 했는데 액션이 남은" 항목 노출
- **데이터**: my seminars(checked_in=false) + my activity applications(pending) + my interview responses(draft)
- **표시**: max 3개 카드, 각 "행동" CTA
- **우선순위**: P1
- **로그인 사용자**

### P1-2: 학회 활동 지표
- **목적**: "지금 학회가 뭘로 활발한가"
- **데이터**: posts(this_month count) + active discussions(comments>5) + popular interview responses(reactions>3)
- **표시**: 3개 카드형 ranking
- **우선순위**: P1
- **모든 역할**

### P1-3: 캘린더 미니뷰
- **목적**: 일정 overview
- **데이터**: 기존 미니 캘린더 그대로
- **개선**: 클릭 시 inline tooltip → 더 보기로 풀뷰

### P2-1: 환영 카드
- **유지** — 신규 회원에게는 onboarding tour 트리거 추가

### P2-2: 최근 게시글
- **제거 검토** — 게시판 첫 화면과 중복. 대신 "학회 활동 지표"의 "이달의 인기 토론"으로 통합

## 6. UX 원칙

### 6-1. Above-the-fold 3초 룰
- 페이지 로드 후 처음 보이는 영역(데스크탑 720px / 모바일 568px)에는 P0 위젯 2개만 노출
- 통계 숫자는 보조 — 항상 "trend(↑12%)"와 "다음 행동 CTA" 동반

### 6-2. 모바일 우선 카드 stacking
- 모든 위젯은 모바일 1열에서도 정보 밀도 유지
- 카드 내 정보 우선순위: **타이틀 → 핵심 숫자 → CTA → 보조 메타**

### 6-3. Empty state 가이드
- "X가 없습니다" 대신 "지금 X 해보세요" + 1차 CTA
- 예: "다가오는 세미나가 없습니다" → "[+] 세미나 둘러보기" 버튼

### 6-4. Skeleton + 점진 로딩
- 모든 비동기 위젯은 skeleton 먼저, 데이터 도착 시 fade-in
- 이미 캐시된 데이터는 instant render
