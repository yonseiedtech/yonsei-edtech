# 연세교육공학회 학술활동 관리 고도화 프로젝트 제안서

**작성일**: 2026-04-11  
**관점**: UX 전문 PM (자동화 및 효율화 중심)  
**대상 범위**: 세미나, 스터디, 대외 학술대회 관리 기능 전반

---

## 1. 현황 분석 (As-Is)

### 1.1 세미나 관리 현황

현재 세미나 관리는 **10단계 라이프사이클**을 지원하며, 학회 운영의 핵심 기능으로 자리잡았다.

| 단계 | 기능 | 자동화 수준 | 평가 |
|------|------|-------------|------|
| 생성 | 폼 입력 + 일정 충돌 감지 | 반자동 | 양호 |
| 홍보 | AI 콘텐츠 5종 생성 (보도자료/SNS/이메일/카카오톡/해시태그) | AI 자동 | 우수 |
| 포스터 | AI 포스터 생성 (3종 사이즈) | AI 자동 | 우수 |
| 타임라인 | 오프라인/온라인 템플릿 + 담당자 배정 | 템플릿 기반 | 양호 |
| 참가 신청 | 회원 직접 + 구글폼 CSV/엑셀 일괄 등록 | 반자동 | 양호 |
| 리마인더 | 카카오톡 메시지 템플릿 (수동 복사) | **수동** | 개선 필요 |
| 출석 체크 | QR 코드 스캔 + 수동 체크인 | 자동 | 양호 |
| 후기 수집 | 3종 후기 (참석자/연사/운영진) + 커스텀 질문 | 반자동 | 양호 |
| 수료증 | 일괄 발급 + 자동 번호 채번 | 반자동 | 양호 |
| 분석 | 출석률, 소속 분포, 학기 분포, 사전 질문 | 자동 | 양호 |

**주요 Pain Points:**
- 리마인더가 **수동 복사-붙여넣기** 방식 (D-3, D-Day, D+1 메시지를 직접 카카오톡 전송)
- 세미나 상태 전환(upcoming→ongoing→completed)은 크론으로 자동화되었으나, **후속 작업 트리거가 부재** (완료 후 후기 요청, 수료증 발급 등이 수동)
- 타임라인 체크리스트의 진행률이 **전체 세미나 목록에서 보이지 않아** 한눈에 준비 상태 파악 불가
- 세미나 간 **데이터 비교/트렌드 분석** 기능 부재 (개별 세미나 분석만 가능)

### 1.2 학술활동(스터디/프로젝트/대외) 현황

| 기능 | 현재 상태 | 평가 |
|------|-----------|------|
| CRUD | 기본 생성/수정/삭제 | 최소 기능 |
| 참여 관리 | 직접 참여(스터디/프로젝트) + 심사 기반(대외) | 기본 |
| 모집 상태 | 수동 전환 (recruiting→closed→in_progress→completed) | **수동** |
| 진행 관리 | 없음 (상태 필드만 존재) | **부재** |
| 산출물 관리 | 없음 | **부재** |
| 활동 보고서 | 신청자 통계만 (대외 학술대회) | **미흡** |
| 알림 | 없음 | **부재** |
| 연계 | 세미나·게시판과 연동 없음 | **부재** |

**주요 Pain Points:**
- 스터디/프로젝트 생성 후 **진행 상황 추적 도구 없음** (상태 필드만으로 관리)
- 모집 마감, 활동 시작/종료 등 **상태 전환이 모두 수동**
- 활동 결과물(발표자료, 보고서)을 **별도로 관리할 공간 부재**
- 세미나와 스터디/프로젝트 간 **교차 연계 불가** (예: 스터디 결과를 세미나에서 발표)

### 1.3 종합 UX 진단

```
    ┌───────────────┬───────────────┬───────────────┐
    │               │               │               │
  세│  리마인더     │  출석 체크    │  AI 홍보      │
  미│  (수동 복사)  │  (QR 자동)   │  (완전 자동)  │
  나│               │  상태 전환    │  포스터 생성   │
    │               │  (크론 자동)  │               │
    ├───────────────┼───────────────┼───────────────┤
    │               │               │               │
  활│  전체 수동    │  신청/참여    │               │

    │               │               │               │
    └───────────────┴───────────────┴───────────────┘

```

## 2. 고도화 제안 (To-Be)

### 개요

3개 트랙, 12개 과제로 구성한다. 우선순위는 **운영 시간 절감 효과**와 **구현 난이도**를 기준으로 산정하였다.

| 트랙 | 과제 수 | 핵심 목표 |
|------|---------|-----------|
| A. 세미나 자동화 심화 | 5개 | 반복 수작업 제거, 운영 워크플로우 완전 자동화 |
| B. 학술활동 고도화 | 4개 | 스터디/프로젝트/대외활동에 진행 관리 도구 추가 |
| C. 통합 분석 & 연계 | 3개 | 데이터 기반 의사결정, 기능 간 연계 강화 |

---

### 트랙 A: 세미나 자동화 심화

#### A-1. 세미나 운영 자동 워크플로우 (Auto-Workflow Engine)

**문제**: 세미나 완료 후 후기 요청, 수료증 발급, 결과 보고서 생성 등 후속 작업을 운영진이 일일이 수동 실행해야 한다.

**제안**: 세미나 상태 전환 시 자동으로 후속 작업을 트리거하는 워크플로우 엔진 구축

```
[세미나 생성] ──► 자동: 타임라인 템플릿 적용 + 전체 회원 알림
       │
[D-3 도래] ──► 자동: 참석자 리마인더 이메일 발송
       │
[D-Day 시작] ──► 자동: status → "ongoing" (기존 크론)
       │
[세미나 종료] ──► 자동: status → "completed"
       │         ──► 자동: 참석자에게 후기 작성 요청 알림 + 이메일
       │         ──► 자동: 연사에게 감사 이메일 + 후기 링크 발송
       │
[후기 3건 이상] ──► 자동: 수료증 일괄 발급 준비 알림 (운영진)
       │
[D+7] ──► 자동: 미제출 후기 리마인더 발송
```

**구현 범위**:
- `vercel.json` 크론 확장: 리마인더 발송, 후기 요청, 미제출 독촉
- 이메일 발송 자동화: Resend API 연동 (기존 inquiry-reply 패턴 재활용)
- 워크플로우 설정 UI: 세미나별 자동 실행 항목 on/off 토글

**예상 효과**: 세미나당 운영진 후속 작업 시간 **약 30분 → 5분** (수동 확인만)

**우선순위**: ★★★ (최우선) | **난이도**: 중

---

#### A-2. 이메일 리마인더 자동 발송

**문제**: 현재 리마인더는 카카오톡 메시지 템플릿을 **수동 복사**하여 전송하는 방식이다. 참석자 수가 많을수록 운영 부담이 크다.

**제안**: 세미나 참석자에게 이메일 리마인더를 자동 발송하는 기능 추가

**발송 시점 (설정 가능)**:
- D-3: "세미나가 3일 후입니다" + 장소/시간 안내
- D-1: "내일 세미나가 있습니다" + 준비물/유의사항
- D-Day 2시간 전: "오늘 세미나 시작 안내" + 온라인 링크 (온라인 세미나 시)

**구현 범위**:
- 이메일 템플릿 3종 (HTML, 기존 inquiry-reply 디자인 재활용)
- 크론잡 확장 (`/api/cron/seminar-reminder` 기존 파일 활용)
- 세미나별 발송 설정: 발송 시점 선택, 수신 대상 (전체 회원/참석자만)
- 발송 이력 저장 (중복 발송 방지)

**예상 효과**: 리마인더 발송 시간 **15분 → 0분** (완전 자동)

**우선순위**: ★★★ (최우선) | **난이도**: 하

---

#### A-3. 타임라인 진행률 대시보드

**문제**: 타임라인 체크리스트가 개별 세미나 LMS 안에만 있어, **여러 세미나의 준비 상태를 한눈에 파악할 수 없다**.

**제안**: 세미나 목록에 타임라인 진행률 요약 표시 + 전체 대시보드

**세미나 목록 카드에 추가**:
```
┌─────────────────────────────────┐
│ 📅 4월 세미나: AI와 교육       │
│ 연사: 홍길동 | 4/25 14:00      │
│ ████████░░ 80% (8/10 완료)     │  ← 진행률 바 추가
│ ⚠️ D-2: 장비 점검 미완료       │  ← 미완료 긴급 항목
└─────────────────────────────────┘
```

**전체 대시보드 (seminar-admin 메인)**:
- 예정 세미나별 타임라인 완료율 정렬
- 기한 초과 항목 하이라이트 (빨간색)
- 담당자별 미완료 항목 집계

**예상 효과**: 준비 상태 파악 시간 **5분 → 즉시**, 누락 항목 사전 방지

**우선순위**: ★★☆ | **난이도**: 하

---

#### A-4. 세미나 복제 (Clone) 기능

**문제**: 정기 세미나는 형식이 유사하나, 매번 처음부터 세미나를 생성해야 한다.

**제안**: 기존 세미나를 복제하여 새 세미나를 빠르게 생성

**복제 대상 필드**:
- 기본 정보: 제목 패턴, 설명 템플릿, 장소, 시간
- 타임라인: 템플릿 (완료 상태 초기화)
- 신청 폼: 필드 구성
- 후기 질문: 참석자/연사/운영진 질문 목록
- 세션 구조: 카테고리/시간 배분 (연사 정보 제외)

**복제 제외 필드**: 날짜, 연사, 참석자, 후기, 수료증, 포스터

**예상 효과**: 세미나 생성 시간 **15분 → 3분**

**우선순위**: ★★☆ | **난이도**: 하

---

#### A-5. 출석 데이터 실시간 동기화 개선

**문제**: 현재 체크인 대시보드는 Zustand 로컬 스토어 기반으로, **여러 운영진이 동시에 체크인할 때 데이터 불일치** 가능성이 있다.

**제안**: Firestore 실시간 리스너 기반 체크인 동기화

**개선 사항**:
- `onSnapshot` 실시간 구독으로 멀티 디바이스 동기화
- 체크인 충돌 감지 (동일 참석자 중복 체크인 방지)
- 오프라인 체크인 큐잉 + 온라인 복귀 시 자동 동기화

**예상 효과**: 대형 세미나(50명+) 운영 시 데이터 정합성 보장

**우선순위**: ★☆☆ | **난이도**: 중

### 트랙 B: 학술활동 고도화

#### B-1. 활동 진행 관리 보드 (Activity Progress Board)

**문제**: 스터디/프로젝트 생성 후 **진행 상황을 추적할 도구가 전혀 없다**. 상태 필드(upcoming/ongoing/completed)만으로는 실질적 관리가 불가능하다.

**제안**: 각 활동에 간이 진행 관리 기능 추가

**주차별 진행 기록**:
```
┌─────────────────────────────────────────┐
│ 📚 AI 교육 스터디 - 진행 현황          │
├─────────────────────────────────────────┤
│ Week 1 (3/7)  ✅ OT + 주제 선정       │
│ Week 2 (3/14) ✅ 논문 리뷰 #1         │
│ Week 3 (3/21) ✅ 논문 리뷰 #2         │
│ Week 4 (3/28) 🔄 중간 발표 준비 중    │
│ Week 5 (4/4)  ⬜ 최종 발표            │
│                                         │
│ [+ 주차 추가]  진행률: 60% ████████░░  │
└─────────────────────────────────────────┘
```

**데이터 모델 확장**:
```typescript
interface ActivityProgress {
  id: string;
  activityId: string;
  week: number;
  date: string;
  title: string;
  description?: string;
  status: "planned" | "in_progress" | "completed";
  attachments?: string[]; // 파일 URL
  createdAt: string;
}
```

**예상 효과**: 활동 관리 가시성 **0 → 100%**, 운영진 현황 파악 즉시 가능

**우선순위**: ★★★ (최우선) | **난이도**: 중

---

#### B-2. 활동 상태 자동 전환

**문제**: 모집 상태와 활동 상태를 운영진이 **수동으로 전환**해야 한다. 날짜가 지나도 "모집중"으로 표시되는 경우가 발생한다.

**제안**: 날짜 기반 자동 상태 전환 + 알림

**자동 전환 규칙**:
| 조건 | 동작 |
|------|------|
| 시작일(date) 도래 | status: upcoming → ongoing |
| 종료일(endDate) 경과 | status: ongoing → completed |
| 모집 정원 도달 | recruitmentStatus: recruiting → closed |
| 시작일 도래 & 모집중 | recruitmentStatus: recruiting → closed |

**구현**: 기존 `seminar-status` 크론잡 패턴 확장, activities 컬렉션 대상 추가

**예상 효과**: 상태 관리 수동 작업 **완전 제거**

**우선순위**: ★★☆ | **난이도**: 하

---

#### B-3. 활동 산출물 관리

**문제**: 스터디 발표자료, 프로젝트 결과물, 학술대회 발표 논문 등을 **관리할 공간이 없다**. 별도 클라우드 드라이브를 사용해야 한다.

**제안**: 세미나 Materials 섹션과 동일한 패턴으로 활동 산출물 관리 기능 추가

**기능**:
- 파일 업로드 (발표자료, 보고서, 논문 등)
- 주차별/카테고리별 분류
- 참여자 누구나 업로드 가능 (리더/운영진 삭제 권한)
- 이미지 미리보기, PDF 뷰어

**데이터 모델**: 기존 `SeminarMaterial` 인터페이스 재활용 (activityId 필드 추가)

**예상 효과**: 외부 도구 의존 제거, 산출물 아카이빙 자동화

**우선순위**: ★★☆ | **난이도**: 하

---

#### B-4. 활동 모집 공고 자동 생성

**문제**: 스터디/프로젝트 모집 공고를 게시판에 **수동으로 별도 작성**해야 한다.

**제안**: 활동 생성 시 게시판 공고 자동 생성 옵션

**워크플로우**:
```
활동 생성 → [✅ 게시판 공고 자동 등록] → 자유게시판에 모집 공고 자동 게시
                                          (활동 상세 링크 포함)
```

**자동 생성 내용**:
- 제목: `[모집] {활동 제목}`
- 본문: 활동 설명 + 일정 + 모집 인원 + 신청 링크
- 카테고리: `free` (자유게시판)

**예상 효과**: 공고 작성 시간 **10분 → 0분**

**우선순위**: ★☆☆ | **난이도**: 하

**제안**: 관리자 대시보드에 학술활동 통합 현황 위젯 추가

#### C-2. 회원 학술활동 이력 통합 뷰

**문제**: 개별 회원의 세미나 참석, 스터디 참여, 프로젝트 경험 등을 **통합적으로 조회할 방법이 없다**.

**제안**: 회원 상세 페이지에 학술활동 이력 탭 추가

**표시 내용**:
- 세미나 참석 이력 (출석 횟수, 후기 작성 수, 수료증 수)
- 스터디/프로젝트 참여 이력
- 대외 학술대회 참가 이력
- 활동 참여도 요약 (총 참여 횟수, 최근 활동일)

**활용**:
- 우수 활동 회원 선정 기초 자료
- 졸업생 추천서 작성 시 참고
- 회원 기여도 시각화

**예상 효과**: 회원 활동 이력 조회 **불가능 → 즉시 가능**

**우선순위**: ★☆☆ | **난이도**: 중

#### C-3. 학기별 활동 보고서 자동 생성

**문제**: 학기 말 학술활동 보고서를 운영진이 **수동으로 집계하여 작성**해야 한다.

**제안**: AI를 활용한 학기 활동 보고서 자동 생성

**보고서 포함 내용**:
- 학기 중 세미나 실적 (횟수, 총 참석자, 평균 출석률, 평균 만족도)
- 스터디/프로젝트 실적 (횟수, 참여 인원, 완료율)
- 대외 학술대회 참가 실적
- 주요 연사 및 주제 목록
- 전년 동기 대비 성장률
- AI 요약: 핵심 성과 및 개선점

**출력 형식**: 웹 페이지 + PDF 다운로드 + 클립보드 복사

**예상 효과**: 보고서 작성 시간 **2시간 → 10분** (검토 및 보완만)

**우선순위**: ★☆☆ | **난이도**: 중
