# 서비스 갭 심층 분석 — 학술활동 · 대학원 생활 (codex 교차검증)

- 2026-07-07 · codex 도메인 심층 분석 + 오케스트레이터 독립 스캔·코드 재검증
- 검증 표기: ✔ 코드로 확정 · (부분) 일부만 유효 · (설계) 판단 영역

## 강점 (교차로 확인 — 갭 아님)

- **스터디/프로젝트 도구는 빈약하지 않음** ✔ — 주차·출석·과제·회의·성찰·사전수업·자료아카이브·참여리포트 완비(`ActivityProgress`, `StudySession*Card`, `StudyParticipationReport`). "세미나 대비 빈약"이라는 흔한 인상은 코드상 사실 아님.
- **세미나 전주기** ✔ — 신청·QR출석·후기·연사후기·수료증·홍보물·콘솔.
- **외부학회 폭** ✔ — 프로그램·세션선택·노트·워크북·봉사·연사·세션분석.

## 🟠 High — 실질 파급 큼

### 1. ✔ 졸업요건 충족 추적이 전무 (도메인 B)
- 근거: `requiredCredit`·졸업요건·필수과목 매트릭스 타입/라우트 **grep 0건**. 수강은 시간표·후기·일정 **"기록"**에 그치고, "이수학점·필수/선택 요건을 얼마나 채웠나"를 보여주는 곳이 없음.
- 영향: 재학생 전원 — 매 학기 수강 계획을 스스로 엑셀·기억으로 관리.
- 제안: 20명 규모에 맞춘 **수동 설정형 졸업요건 체크표**(전공/공통/논문학점 구간 + 이수/잔여). 수강 이력과 연결해 자동 카운트. → 가장 실용적인 신규 도구.

### 2. ✔ 활동 참여 레코드가 자동으로 안 쌓임 (도메인 A — 숨은 최상위)
- 근거: `activity_participations` 모델·`activityParticipationsApi.create`·증명서 PDF·인사이트·포트폴리오는 전부 있으나, **세미나 출석 완료·스터디 이수·발표자 지정 시 create 호출이 어디에도 없음**(seminar·activities feature grep 0건). 하류 소비처(증명서·포트폴리오·MemberReport·me/export)가 전부 이 컬렉션에 의존.
- 영향: 포트폴리오·경력증명이 필요한 회원(비어 있거나 수동 입력), 검증 운영진.
- 제안: 출석 확정(mirror 지점과 동일 훅)·스터디 이수·발표 지정 시 `activity_participations` upsert. 이미 있는 파이프의 **입력단만 연결**하면 증명서·포트폴리오가 저절로 채워짐 — 신규 대형 개발 아님, 배선 작업.

## 🟡 Medium

### 3. ✔ 재학생 통합 상태 카드 부재 (도메인 B)
- 근거: `accumulatedSemesters`·`enrollmentStatus`·`thesisJourneyStage` 필드와 수업·종합시험·논문 위젯은 있으나 **한 카드로 묶은 "현재 학업 상태" 통합판 없음**(grep 0건). 정보가 대시보드 여러 위젯에 흩어짐.
- 제안: 학기·이수학점·종합시험·논문단계·다음 행동을 한 카드로. (졸업요건 #1과 결합 시 시너지)

### 4. ✔ 논문 심사·디펜스 행정 마일스톤 기록 부재 (도메인 B)
- 근거: 연습·템플릿·STT는 강하나 심사위원·제출본·예심/본심 일정·결과 레코드 없음(`defense.ts`는 연습셋 중심).
- **→ 이미 제안서 작성됨: `PROPOSAL_2026-07-07_심사디펜스_운영보드.md`**

### 5. (부분) 종합시험 준비·결과 관리가 얕음 (도메인 B)
- 근거: `ComprehensiveExamRecord`가 예정학기·상태·선택과목 2개·메모 중심. D-day/리마인더는 있음.
- 제안: 같은 레코드에 과목별 준비 체크리스트·기출/자료 링크·결과 메모를 얇게 확장(신규 컬렉션 불필요).

### 6. ✔ 세미나 후기 추천이 재참여 루프로 안 이어짐 (도메인 A)
- 근거: 후기에 `recommendedTopics`·`recommendedSpeakers` 수집되나 **다음 세미나 기획·개인 추천으로 연결하는 집계/라우트 없음** — 수집만 되고 소비 안 됨(데이터 활용 갭).
- 제안: 운영진 "다음 세미나 기획" 보드에 추천 집계 노출 + 관심사 기반 개인 재참여 CTA.

### 7. (부분) 스터디/프로젝트 발표자 경험이 얇음 (도메인 A)
- 근거: 외부학회는 `SpeakerAssignment` 콘솔, 세미나는 연사후기가 있으나 스터디/프로젝트는 `presenterUserIds` 주차 필드 수준. 발표자료 제출·동료 피드백·포트폴리오 산출물 연결 없음.
- 제안: 주차 발표에 자료 제출 + 동료 피드백 + 참여레코드(#2) 연결.

### 8. (부분) 대학원 실무 정보가 상태 기반 안내까지 못 감 (도메인 B)
- 근거: `GuideItem`/`GuideProgress`로 장학·행정·규정을 담을 수는 있으나 "마감·대상학기·서류를 개인 상태별로 노출"하는 구조는 아님. 장학/행정 전용 정보 허브 부재.
- 제안: 행정 마감·대상학기·관련서류를 구조화해 개인 학기 상태에 맞춰 노출.

## ⚪ Low

- **활동 수료증이 세미나 편중** ✔ — `Certificate`는 `activityId`를 지원하나 발급 UI·`certificatesApi.list`가 seminarId 중심. 스터디/프로젝트/외부학회 활동별 발급 패널 부재.
- **스터디/프로젝트 모집 홍보 도구가 세미나보다 약함**(포스터·홍보 콘솔은 세미나 전용). 활동용 홍보 템플릿 얇게 추가.
- **외부학회 봉사 현장 운영 기록 약함** — `VolunteerDuty` self-check가 Phase 2 미구현.
- **스테핑스톤 트랙 명칭 혼선**(추정) — `comprehensive_exam` 트랙 라벨이 "학술대회 대비"와 섞임. 확인 후 분리.
- **졸업/동문 전환 체크리스트 부재** — 졸업 시 동문 프로필·학위논문·포트폴리오 정리로 잇는 handoff 없음.

## 종합 · 우선순위

1. **졸업요건 체크표(#1)** — 재학생 전원 매 학기 니즈, 신규 가치 최대.
2. **활동 참여 레코드 자동 적재(#2)** — 배선 작업으로 증명서·포트폴리오·인사이트를 한 번에 살림. ROI 최고.
3. **재학생 통합 상태 카드(#3)** — #1과 결합하면 "학업 대시보드"로 완성.
4. 심사 보드(#4, 제안서 작성됨) · 종합시험 확장(#5) · 세미나 재참여 루프(#6).
5. Low는 개별 필요 시.

- codex는 스터디/외부학회를 **강점으로 정확히 인정**(내 독립 스캔과 일치) — "스터디가 빈약" 같은 통념은 기각.
