# 방학 시간표 활용 제안서

작성일: 2026-07-02 · 전제: 수강과목 학기 시작/종료(semesterStartDate/EndDate) 기능 배포 완료 — 종강 후 시간표·캘린더·수업 알림 자동 숨김.

## 문제 인식

시간표는 학기 중 대시보드에서 가장 눈에 잘 띄는 자리(좌상단 F-패턴)를 차지하는 "매일 보는" 위젯이다. 방학이 되면 이 자리가 4개월 가까이(6말~8말, 12말~2말) 비게 된다 — 학기 기준 연중 약 1/3. 이 공간을 비워두는 것은 리텐션 관점에서 큰 손실이고, 반대로 방학에 맞는 콘텐츠로 전환하면 "방학에도 매일 들어올 이유"가 된다.

## 제안 (우선순위순)

### 1. 방학 모드 v2 — 개강 카운트다운 + 방학 목표 게이지 (공수 小)
현재 구현된 종강 안내를 발전:
- **개강 D-day**: 다음 학기 개강일(9/1·3/1 관례 또는 학사일정 컬렉션)까지 카운트다운. "개강까지 58일" — 시간 감각을 유지시키는 가장 값싼 장치.
- **방학 주간 목표 게이지**: "이번 주 연구 타이머 목표 10시간" 설정 → study_sessions 합산으로 진행률 표시. 잔디·타이머 인프라 재사용, 새 컬렉션 없이 프로필 필드 1개.

### 2. 방학 루틴 블록 — 시간표 그리드의 재사용 (공수 中, 핵심 제안)
수업 대신 **내가 정한 루틴**(논문 읽기, 통계 공부, 운동, 아르바이트)을 요일·시간 블록으로 등록해 같은 주간 그리드에 표시.
- 데이터: `routine_blocks` {userId, label, weekdays[], startMin, endMin, color, activeFrom, activeTo} — parseSchedule/주간 그리드 렌더러 그대로 재사용.
- 학기 중에는 수업과 함께, 방학에는 루틴만 표시. "시간표 위젯"이 "시간 관리 위젯"으로 승격되는 구조적 확장.
- 루틴 블록 시간에 연구 타이머 원클릭 시작 연결(타이머 인프라 재사용) → 계획→실행→잔디 적립의 폐루프.

### 3. 계절학기·특강 지원 — 추가 개발 없이 즉시 가능 (공수 0)
이번 기간 기능으로 **이미 가능**: 계절학기/방학 특강을 CourseOffering으로 등록하고 개강·종강일을 방학 기간(예: 7/7~8/1)으로 설정하면 시간표·캘린더·수업 알림이 그 기간에만 정확히 작동한다. 운영 안내만 하면 됨.

### 4. 방학 스터디를 그리드에 (공수 中)
방학은 학회 스터디의 성수기. activities(스터디) 주차 일정에 요일·시간이 있으면 시간표 그리드에 수업처럼 표시(현재 일부 표시되는 활동 로직 강화). 방학 시간표의 실질 콘텐츠가 된다.

### 5. 시간 사용 리포트 — 계획 대비 실제 (공수 中)
주간 그리드에 실제 연구 타이머 세션을 반투명 블록으로 겹쳐 그려 "계획(루틴) vs 실제(세션)"를 시각화. 방학이 끝나면 "방학 리포트"(총 몰입시간·잔디·읽은 논문)를 자동 생성해 개강 첫 주에 보여준다 — 방학 활동의 성취를 회수하는 순간.

### 6. 다음 학기 수강 장바구니 (공수 中)
방학 중 다음 학기 개설 과목을 미리 담아두는 "수강 계획" 탭. 개강일이 되면 장바구니가 시간표로 자동 전환. 방학 후반의 자연스러운 재방문 이유.

### 7. 개강 리마인더 (공수 小)
크론: 개강 D-7 "다음 학기 시간표를 확인하세요" / D-1 알림. 방학 이탈 회원을 개강 시점에 회수하는 안전망.

## 권장 순서
1(카운트다운+목표) → 3(계절학기 운영 안내) → 2(루틴 블록) → 7(개강 리마인더) → 4·5 → 6.
1·3·7만으로도 방학 공백의 체감이 사라지고, 2가 들어가면 시간표가 연중 상시 기능이 된다.
