# ADDIE 모델, 이론에서 실전으로 — 우리가 놓치고 있는 5가지

> 카테고리: 자료실 또는 자유게시판 / 운영진 또는 학회 회원 작성
> 작성: 운영진 / 2026-05-13
> 길이: ~10분 읽기

교육공학 전공자라면 누구나 ADDIE (Analysis · Design · Development · Implementation · Evaluation) 모델을 배웁니다. 이론으로는 명료하지만, 본인이 실제 교육 자료·세미나·강의를 설계할 때 ADDIE 를 그대로 따라가본 적이 있으신가요? 본인이 학회 운영 + 교육 콘텐츠 제작 5년 경험에서 발견한 "이론과 실전의 5가지 간극"을 공유합니다.

## 1. Analysis 단계가 가장 짧게 끝나버린다

이론: 학습자 분석·과제 분석·환경 분석에 충분한 시간 투자.
실전: 마감 압박 때문에 "대상은 신입생, 주제는 OO" 한 줄로 끝.

**해결 방법**: Analysis 산출물에 **최소 3가지 필수 질문**을 강제하기:
- 학습자가 이미 무엇을 알고 있는가? (사전지식)
- 학습자가 이 콘텐츠 후 무엇을 할 수 있어야 하는가? (수행 목표)
- 학습자가 어디서·언제·어떻게 학습할 것인가? (수행 맥락)

이 세 질문에 한 문장씩 답하는 것만으로도 후속 단계가 30% 더 명확해집니다.

## 2. Design 과 Development 가 동일시된다

이론: Design (설계도 — 학습 목표·평가 방법·교수 전략) → Development (실제 자료 제작) 분리.
실전: "PPT 만들면서 설계도 같이 짠다" — 두 단계가 섞이고 결국 Design 이 부실해진다.

**해결 방법**: Development 시작 전에 **Design 1쪽 요약 문서** 만들기:
```
1. 학습 목표 (가능하면 ABCD 형식: Audience-Behavior-Condition-Degree)
2. 평가 기준 (학습자가 무엇을 보여야 통과인가)
3. 핵심 교수 전략 (강의 / 토론 / 실습 / 사례 등)
4. 시간 배분 (10분 강의 + 5분 활동 + …)
```
이 1쪽이 있으면 Development 가 절반의 시간으로 끝납니다.

## 3. Implementation 단계의 정의가 모호하다

이론: 학습자에게 실제 적용.
실전: "세미나 진행 = Implementation 완료" 라고 끝나버린다.

**해결 방법**: Implementation 을 **3단계로 세분화**:
- Pilot (소규모 사전 실행) — 동료 1~2명 대상
- Full delivery (정식 실행) — 대상 청자 전체
- Follow-up (사후 지원) — Q&A·자료 보강·후속 학습 가이드

연세교육공학회 세미나 운영에서 본인이 가장 큰 차이를 만든 것은 Follow-up 단계입니다. 세미나 종료 직후 24시간 내에 학습자에게 "오늘 배운 것 중 어떤 것을 실제로 적용할 계획인가요?" 질문을 보내면 학습 전이가 약 3배 늘어납니다.

## 4. Evaluation 이 "만족도 설문 1장" 으로 끝난다

이론: Kirkpatrick 4단계 (반응 · 학습 · 행동 · 결과) 다층 평가.
실전: 세미나 종료 직후 5점 만족도 설문 → 끝.

**해결 방법**: 가능한 범위에서 **2단계 평가** 최소화:
- Level 1 (반응): 즉시 만족도 + 자유 응답 1개 ("가장 도움이 된 부분")
- Level 2 (학습): 1주 후 빠른 quiz 3문항 (학습 검증)

Level 2 를 추가하는 데 추가 시간은 본인 경험상 30분 미만입니다. 그러나 이 30분이 본인의 콘텐츠 효과성을 객관적으로 평가할 수 있는 유일한 데이터입니다.

## 5. ADDIE 가 "선형적"이라는 오해

이론 vs 현실: ADDIE 는 본질적으로 반복적·순환적. 그러나 교과서에는 5단계가 순서대로 그려져 있어 "한 번 하면 끝" 처럼 보인다.

**해결 방법**: 본인 작품을 **버전 1.0 부터 시작**하기. 학회 세미나 자료는 학기마다 진화해야 하며, 각 학기 후 Evaluation 결과를 다음 학기 Analysis 입력으로 바로 사용. ADDIE 는 사다리가 아닌 **나선형 (Spiral)** 입니다.

## 결론

ADDIE 는 단순 체크리스트가 아닌 **사고의 도구**입니다. 5단계를 정확히 따르는 것보다, 각 단계의 핵심 질문에 정직하게 답하는 것이 더 중요합니다.
