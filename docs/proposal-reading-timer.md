> 작성일: 2026-04-18 | 대상 서비스: 연세교육공학회 (yonsei-edtech)
> 참고 서비스: 논문길 (Thesis-Writer-Clean) 타이머 아키텍처

---

## 1. 기능 개요

### 1-1. 배경 및 목적

현재 연세교육공학회 사이트의 연구 활동 추적은 제한적이다:
- **논문 읽기**: 날짜 단위(readStartedAt / readCompletedAt)로만 추적 — 실제 투자 시간 불명
- **논문 작성**: 저장 횟수(WritingPaperHistory)만 기록 — 실제 작성 시간 불명

연구 활동 타이머는 단순한 시간 측정이 아니라, **논문을 읽고 쓰는 동안 집중할 수 있도록 돕는 도구**이다.

### 1-2. 핵심 가치

| 가치 | 설명 |
|------|------|
| **집중 유도** | 타이머가 돌아가고 있다는 인식 자체가 집중력을 높임 |
| **연구 습관 형성** | 일별·주별 읽기/작성 시간 시각화로 꾸준한 연구 습관 장려 |
| **데이터 기반 성찰** | 논문별 투자 시간, 평균 집중도 등 정량적 지표 제공 |
| **연구리포트 연계** | 읽기·작성 활동이 연구리포트에 자동으로 반영되어 연구 성과 가시화 |

## 2. 사용자 시나리오

### 시나리오 A: 논문 읽기 타이머

```
1. 사용자가 마이페이지 > 연구 > 읽기 탭에서 논문 카드를 본다
2. 논문 카드의 "읽기 시작" ▶ 버튼을 클릭한다
3. 화면 하단에 미니 타이머 바가 나타난다 (논문 제목 + 경과 시간 + 일시정지/종료 버튼)
4. 사용자는 다른 페이지로 이동해도 타이머가 유지된다 (하단 바 고정)
5. 읽기를 마치면 "종료" 버튼을 클릭한다
6. 종료 팝업: 집중도 자기평가(1~5점) + 메모(선택) 입력 후 저장
7. 세션이 저장되고, 해당 논문의 총 읽기 시간이 갱신된다
```

### 시나리오 B: 논문 작성 타이머

```
1. 사용자가 마이페이지 > 연구 > 작성 탭에서 논문 작성 에디터에 진입한다
2. 작성 에디터 상단의 "작성 시작" ▶ 버튼을 클릭한다
3. 하단에 미니 타이머 바가 나타난다 (작성 중 논문 제목 + 경과 시간)
4. 다른 페이지로 이동해도 타이머 유지
5. 작성을 마치면 "종료" → 집중도/메모 입력 후 저장
6. 작성 세션이 기록되고, 작성 시간이 누적된다
```

### 시나리오 C: 수기 입력 (종료를 못 누른 경우)

```
1. 논문 카드(또는 작성 에디터)의 "..." 메뉴에서 "시간 수기 입력"을 선택
2. 다이얼로그: 활동 유형(읽기/작성) / 날짜 / 시작 시각 / 종료 시각 / 메모 입력
3. 시간 겹침 검증 후 저장
4. "수기" 라벨이 붙은 세션으로 기록된다
```

### 시나리오 D: 집중 모드 (팝업형 전체화면)

```
1. 타이머 바의 "집중 모드" 🎯 버튼을 클릭한다
2. 전체화면 오버레이가 열린다:
   - 중앙: 큰 타이머 (HH:MM:SS)
   - 상단: 논문 제목 + 활동 유형 배지 (읽기/작성)
   - 하단: 오늘의 연구 목표 진행률 바
   - 배경: 차분한 그라데이션 (네이비 계열)
3. 25분 경과 시 부드러운 알림: "5분 휴식 어떠세요?" (포모도로 연계)
4. ESC 또는 "나가기" 버튼으로 일반 모드 복귀 (타이머 유지)
```

## 3. UI/UX 상세 설계

### 3-1. 미니 타이머 바 (Floating Timer Bar)

화면 하단에 고정되는 슬림한 바. 연구 활동 중 어떤 페이지에서든 표시된다.

```
┌─────────────────────────────────────────────────────────────────────┐
│  📖 읽기  「교육공학에서의 AI 활용 동향」  ⏱ 00:47:23   ⏸ ⏹ 🎯  │
└─────────────────────────────────────────────────────────────────────┘
  [활동유형]  [논문 제목 (잘림 시 ...)]   [경과 시간]  [일시정지] [종료] [집중모드]
```

- **높이**: 48px (모바일), 44px (데스크톱)
- **위치**: `fixed bottom-0` (모바일 네비 위)
- **색상**: 읽기=네이비 배경, 작성=딥블루 배경 + 흰색 텍스트
- **활동 배지**: `📖 읽기` / `✏️ 작성` (시각적 구분)
- **애니메이션**: 시작 시 아래에서 슬라이드 업
- **일시정지**: 30분 이상 일시정지 시 자동 종료 (유휴 감지)

### 3-2. 종료 다이얼로그

```
┌──────────────────────────────┐
│   📖 읽기 세션 완료           │

│   47분 23초                  │
│                              │
│   집중도는 어땠나요?         │
│   ★ ★ ★ ★ ☆  (4/5)         │
│                              │
│   메모 (선택)                │
│   ┌──────────────────────┐   │
│   │ Ch.3 방법론 부분     │   │
│   │ 재독 필요            │   │
│   └──────────────────────┘   │
│                              │
│   ☐ 완독 처리하기 (읽기 시)  │
│                              │
│   [취소]         [저장하기]   │
└──────────────────────────────┘
```

- **집중도**: 1~5 별점 (탭/클릭)
- **메모**: 2줄 textarea (선택)
- **완독 체크**: 읽기 세션일 때만 표시. 체크 시 readStatus → "completed" 자동 변경
- **취소**: 세션은 저장하되 집중도/메모 없이 저장

### 3-7. 유휴 감지 (Grace Period)

```
사용자가 일시정지 후 30분 경과 시:

┌──────────────────────────────┐
│   ⏰ 아직 작업 중이신가요?    │
│                              │
│   「교육공학에서의 AI 활용」   │
│   일시정지 30분 경과           │
│                              │
│   [계속하기]    [세션 종료]   │
│                              │
│   15초 후 자동 종료됩니다     │
│   ████████████░░░  (12초)     │
└──────────────────────────────┘
```

- 일시정지 30분 후 Grace Period 카운트다운 (15초)
- 응답 없으면 자동 종료 (일시정지 시점까지만 시간 기록)
- 브라우저 탭 비활성 5분 이상 시에도 Grace Period 진입

## 4. 데이터 모델

### 4-1. StudySession (신규 컬렉션)

```typescript
type StudySessionType = "reading" | "writing";

interface StudySession {
  id: string;
  userId: string;
  type: StudySessionType;     // "reading" | "writing"
  paperId?: string;           // ResearchPaper.id (읽기 시)
  writingPaperId?: string;    // WritingPaper.id (작성 시)
  targetTitle: string;        // 논문 제목 (UI 표시용 스냅샷)
  startTime: string;          // ISO datetime
  endTime: string | null;     // ISO datetime (진행 중이면 null)
  durationMinutes: number;    // 종료 시 계산 (분 단위, 소수점 1자리)
  source: "timer" | "manual"; // 타이머 vs 수기 입력
  focusScore?: number;        // 1~5 (종료 시 입력)
  memo?: string;              // 자유 메모
  createdAt: string;
  updatedAt: string;
}
```

### 4-2. ResearchPaper / WritingPaper 확장 필드

```typescript
// ResearchPaper에 추가
totalReadingMinutes?: number;  // 읽기 세션 합산 (캐시)

// WritingPaper에 추가
totalWritingMinutes?: number;  // 작성 세션 합산 (캐시)
```

### 4-3. 일일 목표 설정 (User 프로필 확장)

```typescript
interface User {
  // ... 기존 필드 ...
  dailyStudyGoalMinutes?: number;  // 일일 연구 목표 (분), 기본 60
}
```

## 5. 연구리포트 연계

### 5-1. 요약 지표 추가

현재 4개 BigStat에 **2개 추가**:

| 지표 | 설명 | 산출 |
|------|------|------|
| **총 연구 시간** | 읽기+작성 시간 합계 | `SUM(sessions.durationMinutes)` |
| **평균 집중도** | 전체 세션 평균 집중 점수 | `AVG(sessions.focusScore)` |

### 5-2. 활동 요약 확장

```
작성 활동 요약                      읽기 활동 요약
├─ 최대 연속 작성일  5일            ├─ 총 등록          12건
├─ 총 저장 횟수     48회            ├─ 완독             8건
├─ 작성 분량        12,400자        ├─ 읽는 중          3건
├─ 진행 챕터        3/5             ├─ 읽을 예정        1건
├─ 총 작성 시간     18시간 30분 NEW ├─ 평균 완독 소요   5일
└─ 평균 세션 시간   42분        NEW ├─ 총 읽기 시간     24시간 30분 NEW
                                    └─ 평균 세션 시간   45분        NEW
```

### 5-3. 히트맵 연계

WritingHeatmap의 호버 툴팁에 연구 시간 추가:

```
2026년 04월 18일(금)
논문 작성 : YES!
작업 이력 : 3건
연구 시간 : 2시간 45분 (읽기 1h25m + 작성 1h20m)   ← 신규
```

## 6. API 설계

### 6-1. 세션 시작

```
POST /api/study-sessions/start
Body: { type: "reading" | "writing", paperId?: string, writingPaperId?: string, targetTitle: string }
Response: { id: string, startTime: string }
```

### 6-2. 세션 종료

```
POST /api/study-sessions/:id/end
Body: { focusScore?: number, memo?: string, markCompleted?: boolean }
Response: { durationMinutes: number }
```

### 6-3. 수기 입력

```
POST /api/study-sessions/manual
Body: { type, paperId?, writingPaperId?, targetTitle, date, startTime, endTime, memo? }
Response: { id: string, durationMinutes: number }
```

### 6-4. 세션 조회

```
GET /api/study-sessions?type=reading&paperId=xxx
GET /api/study-sessions?from=2026-04-01&to=2026-04-30
Response: { data: StudySession[], total: number }
```

## 9. 엣지 케이스 처리

| 상황 | 처리 방안 |
|------|-----------|
| 종료 버튼을 안 누르고 브라우저 닫음 | localStorage에 세션 정보 저장. 다음 접속 시 복원 팝업: "이전 세션이 있습니다. 종료 시각을 입력해주세요" |
| 4시간 이상 활성 세션 | Ghost 세션으로 판단. 자동 종료 처리 (마지막 활동 시점까지만 기록) |
| 같은 대상에 동시 세션 | 불가. 이미 활성 세션이 있으면 "진행 중인 세션이 있습니다" 알림 |
| 다른 논문으로 전환 | 기존 세션 자동 종료 (간이 다이얼로그) → 새 세션 시작 |
| 읽기↔작성 전환 | 기존 세션 종료 후 새 세션 시작 (동시에 두 종류 불가) |
| 모바일에서 백그라운드 | Web Visibility API로 감지. 백그라운드 5분 이상 시 Grace Period 진입 |
| 수기 입력 시간 겹침 | 같은 날 기존 세션과 시간대 겹치면 경고 + 저장 차단 |
