# 세미나 관리 UX 개선 제안서

> 작성일: 2026-04-11 | 분석 범위: `/seminar-admin` 전체 8개 탭

---

## 현황 분석

### 현재 세미나 운영 워크플로우

```
생성 → 타임라인 설정 → 신청 관리 → 홍보 제작 → 당일 운영(체크인) → 참석자 분석 → 수료증 발급
```

### 주요 Pain Points

| # | 문제 | 영향 |
|---|------|------|
| 1 | **탭 간 컨텍스트 단절** — 세미나를 선택한 뒤 다른 탭으로 이동하면 다시 세미나를 선택해야 함 | 매번 드롭다운 클릭 반복, 실수로 다른 세미나 작업 위험 |
| 2 | **워크플로우 순서 불명확** — 8개 탭이 나열되어 있지만 어떤 순서로 진행해야 하는지 가이드 없음 | 신규 운영진이 학습에 시간 소요 |
| 3 | **세미나 목록과 관리가 이원화** — `/admin/seminars`(관리자 패널)와 `/seminar-admin`(세미나 관리) 두 곳에서 세미나 관리 | 어디서 뭘 해야 하는지 혼란 |
| 4 | **수동 반복 작업 다수** — 참석자 데이터를 엑셀/구글시트에서 수동 임포트, 수료증 대상자 수동 선택 | 50명 이상 세미나 시 20분+ 소요 |
| 5 | **홍보와 포스터 분리** — 같은 세미나의 마케팅 자료인데 별도 탭에서 각각 세미나 선택 필요 | ~~해결 완료 (합병)~~ |
| 6 | **진행 상태 한눈에 안 보임** — 타임라인 진행률은 목록에만 작게 표시, 전체 준비 상황 대시보드 없음 | 마감 임박 항목 놓침 위험 |

## 개선 제안: 3개 트랙

### Track A. 세미나 허브 — 컨텍스트 유지형 단일 페이지

> 목표: 세미나 하나를 선택하면 모든 관리 기능을 한 화면에서 접근

#### A-1. 세미나 컨텍스트 바 (Sticky Context)

**현재**: 각 탭마다 세미나 드롭다운 → 매번 선택
**개선**: 상단에 고정된 컨텍스트 바에서 세미나 한 번 선택 → 모든 하위 탭에 자동 반영

```
┌─────────────────────────────────────────────────┐
│ 📌 [AI와 교육의 미래 ▾]  2026-04-15  김교수     │
│     상태: 예정  |  참석: 23/40  |  준비: 75%     │
├─────────────────────────────────────────────────┤
│ 타임라인 | 신청관리 | 홍보제작 | 참석자 | 수료증  │
└─────────────────────────────────────────────────┘
```

- Zustand 스토어에 `selectedSeminarId` 저장
- 모든 하위 컴포넌트가 동일 ID 참조
- 세미나 변경 시 하위 데이터 자동 리페치

**효과**: 탭 이동 시 세미나 재선택 제거 → 작업 시간 30%+ 단축

#### A-2. 단계별 진행 가이드 (Stepper)

**현재**: 탭 나열만 있어 순서 불명확
**개선**: 세미나 상태에 따라 현재 단계를 하이라이트하는 스테퍼 UI

```
① 생성 ──✓── ② 타임라인 ──●── ③ 홍보 ──○── ④ 당일 ──○── ⑤ 정산
```

- 각 단계에 완료 조건 자동 판별 (예: 타임라인 100% → 다음 단계로)
- 미완료 단계 클릭 시 해당 탭으로 이동 + 필요 작업 하이라이트
- 완료 시 축하 토스트 ("홍보 준비 완료!")

**효과**: 신규 운영진 온보딩 시간 50% 단축, 누락 방지

#### A-3. 세미나 대시보드 카드 (목록 뷰 개선)

**현재**: 테이블 형태로 정보 밀집
**개선**: 카드 기반 대시보드 + 칸반 뷰 옵션

- 칸반: `예정` | `진행중` | `완료` | `취소` 열로 분류
- 각 카드에 미니 진행률 바, D-day 카운트다운, 미완료 태스크 수
- 드래그로 상태 변경 (optional)

**효과**: 전체 세미나 상황을 한눈에 파악

---

### Track B. 자동화 — 반복 작업 제거

#### B-1. 원클릭 세미나 세팅

**현재**: 생성 후 타임라인 수동 초기화 → 홍보 수동 시작
**개선**: 세미나 생성 시 자동으로:

1. 온/오프라인 타입에 맞는 타임라인 템플릿 자동 적용
2. 기본 리마인더 메시지 생성
3. 홍보 콘텐츠 초안 AI 자동 생성 (백그라운드)

사용자는 생성 후 바로 "검토" 단계로 진입 가능

**효과**: 세미나 초기 설정 시간 10분 → 1분

#### B-2. 자동 수료증 발급

**현재**: 완료 후 수동으로 참석자 선택 → 일괄 발급
**개선**: 세미나 완료 + 체크인 확인 후 자동 발급 옵션

- 설정: "세미나 완료 시 체크인 참석자에게 자동 수료증 발급" 토글
- 수료증 번호 자동 채번 (기존 로직 활용)
- 발급 후 이메일/알림 자동 발송
- 예외 처리: 수동 검토 필요 시 "대기" 상태로 표시

**효과**: 수료증 발급 소요 시간 20분 → 0분 (자동)

#### B-3. 참석자 데이터 자동 연동

**현재**: 구글 폼 → 구글 시트 → 수동 임포트
**개선**: 구글 시트 URL 등록 → 주기적 자동 동기화

- 세미나에 구글 시트 URL 연결 (1회)
- "자동 동기화" 토글 ON → 1시간마다 신규 행 자동 반영
- 중복 체크 자동 수행
- 마지막 동기화 시간 및 건수 표시

**효과**: 수동 임포트 반복 작업 완전 제거

#### B-4. 홍보 일괄 생성 + 예약 발행

**현재**: 보도자료, SNS, 이메일, 카카오톡을 각각 생성
**개선**: "일괄 생성" 버튼으로 5개 포맷 동시 생성

- 세미나 선택 → "전체 홍보 패키지 생성" 클릭
- AI가 5개 포맷을 병렬 생성 (30초 내)
- 결과를 카드형 그리드로 한눈에 비교
- 각 카드에서 편집/복사/발행 가능
- 보도자료: "게시판 발행" 예약 (D-7 자동 발행 등)

**효과**: 홍보 콘텐츠 제작 시간 15분 → 2분

---

### Track C. 통합 및 정리 — 구조 최적화

#### C-1. 관리자 패널 세미나 탭과 통합

**현재**: `/admin/seminars`와 `/seminar-admin`이 별도 존재
**개선**: `/admin/seminars`에서 세미나 클릭 → `/seminar-admin`으로 이동 (컨텍스트 유지)

- 관리자 패널의 세미나 탭은 "간략 뷰" (목록 + 빠른 액션)
- 상세 관리는 세미나 관리 페이지로 원클릭 이동
- 뒤로가기 시 관리자 패널로 복귀

**효과**: 이원화 해소, 사용자 혼란 제거

#### C-2. 콘텐츠 제작 탭 명칭 변경

**현재**: "콘텐츠 제작" (수료증 + 명찰 생성)
**개선**: "수료증/명찰"로 명칭 변경 — 기능을 직관적으로 전달

**효과**: 탭 이름만으로 기능 예측 가능

#### C-3. 참석자 관리 + 신청 관리 통합

**현재**: "참석자 관리"와 "신청 관리"가 별도 탭
**개선**: "참석자" 하나의 탭으로 합치고, 내부에 서브탭 구성

```
참석자
├── 신청 목록 (기존 신청 관리)
├── 출석 현황 (기존 참석자 관리)
└── 분석 (통합 분석)
```

**효과**: 관련 기능 그룹핑, 탭 수 6개로 감소 → 인지 부하 감소

#### C-4. 최종 탭 구조 (개선 후)

```
① 세미나 목록  ② 세미나 생성  ③ 홍보 제작  ④ 운영 타임라인  ⑤ 참석자  ⑥ 수료증/명찰
```

기존 8개 → 6개 탭으로 정리. 세미나 컨텍스트 바 도입 시 "세미나 생성"도 목록 내 버튼으로 흡수 가능 (5개 탭).
