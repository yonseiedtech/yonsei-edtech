# 연세교육공학회 서비스 고도화 제안서 — 연동(Integration) 중심

작성일: 2026-07-02 · 근거: 코드베이스 전수 조사 5종(기능 인벤토리 / 참여장치 / UI·UX / 연동 추적 4개 영역 / 대시보드 데이터 감사)

## 0. 한 줄 결론

이 서비스의 다음 도약은 **새 기능 추가가 아니라 "이미 만들어진 기능들 사이의 배선(wiring)"**이다. 개별 기능은 성숙하지만, 트랙 간 연결이 끊겨 사용자의 활동이 축적·순환되지 않는 지점이 많다. 특히 **모임·행사(네트워킹 트랙)는 거의 완전한 사일로**이고, **대시보드는 수집된 데이터의 절반만 보여주며**, **같은 지식(번역투·통계방법)이 2~3곳에 중복 하드코딩**되어 있다.

---

## 1. 연동 매트릭스

범례: ✅ 연동됨 · ➡️ 일방향 · ⚠️ 부분/중복 · ❌ 단절(사일로)

### A. 연구 여정 내부

| 연결 | 상태 | 비고 |
|---|---|---|
| 연구 타이머 ↔ 논문 작성/논문 읽기 | ✅ 양방향 | writingPaperId/paperId 기록, 누적시간 역노출 — 모범 사례 |
| 지도 노트 → 논문 에디터 (장별 미반영 배지) | ➡️ | 잘 됨. 반영 처리는 로그 화면에서만 |
| 연구 계획서 → 논문 (서론·방법 시딩) | ➡️ | seedFromProposal. 역방향 없음 |
| 읽은 논문 → 계획서/리포트 (참고문헌 선택) | ➡️ | APA7 목록 생성 |
| 읽은 논문 → 논문 에디터 | ⚠️ | ReadingDrawer 클립보드 복사만, 구조화 삽입 없음 |
| 연구 리포트 ↔ 연구 계획서 | ❌ | 서로 읽지 않음 (연구모형을 매개로만 간접) |
| 연구 모형 ↔ 리포트 | ✅ | Variable/QuestionSync 양방향 |
| 연구 모형 → 계획서 | ⚠️ | 연구문제 생성만, 변인 동기화 패널 없음 |
| 연구 모형 ↔ 논문 에디터 | ❌ | 정적 링크뿐. 에디터 연구문제 패널과 미동기화 |
| 초록 패널 ↔ 본문/계획서/졸업논문 | ❌ | 완전 고립 도구 |
| 논문 읽기 기록 이원화 (research_papers vs paper_reading_logs) | ❌ | 두 시스템 병존, 상호 참조 없음 |
| thesis_references / thesis_claims | 💀 | API만 존재, UI 소비자 0 (죽은 데이터) |

### B. 아카이브 ↔ 도구

| 연결 | 상태 | 비고 |
|---|---|---|
| 통계/연구방법 파인더 → 졸업생 논문 예시 | ✅ | statMethodIds 역조회. 단, 외부 PDF 링크(내부 상세 페이지 아님) |
| 파인더 → 결과 저장/공유/에디터 | ❌ | 결과가 휘발성 state. URL 공유·연구문제 태그 연계 없음 |
| 문체 점검(STYLE_RULES) ↔ writing-tips 카드 | ⚠️ 이중화 | 같은 번역투 지식의 병렬 인코딩. 규칙→카드 링크 없음 |
| 에디터 통계 템플릿 ↔ archive_statistical_methods | ⚠️ 삼중화 | 파인더·에디터·아카이브에 지식 3벌. seedKey로 선택만 연동, 링크는 검색쿼리(부정확) |
| archive_variables/measurements ↔ 연구모형·PaperVariables | ❌ | 변인이 전부 자유 텍스트. 공용 어휘 미사용 |
| 아카이브 즐겨찾기 → 재노출 | ⚠️ | /archive 랜딩만. 에디터·대시보드 미노출 |
| foundation-terms → 플래시카드 | ❌ | concepts는 ✅(수동 버튼). 기초용어는 경로 자체 없음 |
| 졸업생 논문 ↔ 아카이브 문서 (개념/변인/방법) | ✅ 양방향 | 단, 역링크가 2곳에 이중 관리(비정규화 위험) |
| /archive/graph | ✅ | 실제 관계 기반. 졸업논문·팁은 그래프 밖 |

### C. 학술활동·커뮤니티 ↔ 개인 기록

| 연결 | 상태 | 비고 |
|---|---|---|
| 세미나/스터디 참여 → 공개 프로필 | ✅ 자동 | 출석·참여 자동 재구성 |
| 참여 → /mypage/portfolio | ❌ | 포트폴리오는 수동 입력 전용. ActivityParticipation 스키마는 고아(작성자 0) |
| 후기·회고 → 개인 성장 기록 | ⚠️ | 커뮤니티/잔디에는 반영, "내 회고 모음" 없음 |
| paper-review 게시판 ↔ 논문 읽기 | ❌ | `review_board` enum만 정의, 배선 안 됨 |
| 진단 결과 → 약점 개념·플래시카드 | ✅ | 폐루프(learning-loop)까지 있음 — 모범 사례 |
| 진단 결과 → 단계 추천/로드맵/온보딩 CTA | ❌ | 추천 패널은 학기수만 봄, 진단 미반영 |
| 협업 연구(collab) ↔ 논문 에디터·문체점검 | ❌ | 협업은 맨텍스트 에디터 별도, lint 미적용 |
| 면담(progress_meetings)·지도노트 → 캘린더/.ics | ❌ | 캘린더 소스에 없음 |
| 잔디(스트릭) 소스 | ⚠️ 광범위 | 11종 반영. 단 아카이브 열람은 잔디 미적립 |
| 저널 ↔ collab ↔ 개인 논문 | ⚠️ | collab→저널 컨테이너 ✅, 본문 자동이관 ❌, 개인 논문 ↔ 저널 ❌ |

### D. 모임·행사 (네트워킹 트랙) — 가장 큰 사일로

내부 완성도는 높음(일정투표·프로그램·회비·콘솔 정산). 외부 연동은 전무:

| 연결 | 상태 |
|---|---|
| RSVP → /calendar · /mypage/calendar · .ics | ❌ |
| 참석 → 잔디/포인트/포트폴리오/수료증 | ❌ |
| 리마인더 (크론 19종 중 네트워킹 0) — rsvpDeadline 필드는 있으나 미사용 | ❌ |
| 행사 후기/리뷰 (세미나는 후기 요청 크론까지 있음) | ❌ |
| 참석자 ↔ 디렉토리/명함/DM ("누구를 만났나" 흐름) | ❌ |
| 사진첩(PhotoAlbum) — seminarId/activityId만 있고 네트워킹 eventId 없음 | ❌ |
| /network 관계망 지도와의 연계 | ❌ (의도적 용어 분리, 데이터 연계도 없음) |
| 회비 ↔ 학회 회계(transactions) | ❌ 의도적 분리 — **유지** |

### E. 대시보드 — 수집됐지만 안 보이는 데이터

미노출(가치순): 플래시카드 오늘 복습 수 → 지도노트 미반영 건수 → 글자수 추이 스파크라인 → 진단 점수 추이(현재 최신 1회만) → 리더보드 내 순위 → 프로필 조회/좋아요 수 → 협업 연구 현황 → 학습트랙 진도 → 수료증 수 → 과제 마감 → 모임 내 RSVP 상태 → 내 댓글 활동.

정비 필요: `spacedRepetition` 위젯이 라벨(간격 반복)과 달리 실제론 옛 게시글 재노출(진짜 SRS는 마이페이지에만) · `dailyTimeline`/`miniCalendar` 죽은 토글 2개 · 세미나 카운트 3~4중 표시 · course_todos 동일 컬렉션 4회 중복 조회 · 데이터 최다 블록(CommandCenter·잔디 등)은 위젯 설정에서 토글 불가.

## 2. 구조적 발견 (연동 이전에 정리할 것)

1. **지식의 SSOT 부재**: 번역투 지식 2벌(정규식 lint vs 카드), 통계방법 지식 3벌(파인더 rationale, 에디터 하드코딩 템플릿, 아카이브 문서). 아카이브를 정본으로 삼고 도구는 참조하는 구조로 전환 필요.
2. **고아·죽은 스키마**: thesis_references/claims(소비자 0), ActivityParticipation(작성자 0), PaperReadingSource="review_board"(호출자 0), next-themes(죽은 의존성). 쓰거나 지우거나 결정 필요.
3. **읽기 기록 이원화**: research_papers(서지·연구노트)와 paper_reading_logs(타이머·잔디)가 같은 행위를 두 곳에 기록.
4. **역링크 이중 관리**: 통계방법↔졸업논문 관계가 method.alumniThesisIds(정방향)와 thesis.statMethodIds(역방향) 양쪽에 따로 저장 — 불일치 위험.

## 3. 고도화 로드맵 (4단계)

### Phase 1 — 기반: 속도와 정합성 (효과 大 / 공수 中)
"연결"을 얹기 전에 바닥을 다진다.
- 성능: 콘텐츠성 페이지 서버 컴포넌트 전환, 라우트 loading.tsx/중첩 error.tsx, 100KB급 에디터·PDF·차트 dynamic() 분리, 목록 페이지네이션 감사.
- 대시보드 정비: 죽은 토글 2개 제거/구현, spacedRepetition을 진짜 플래시카드 SRS로 교체, 세미나 카운트·course_todos 중복 정리, 하드코딩 블록의 위젯 레지스트리 편입.
- 죽은 스키마 정리(§2-2): 사용 계획 없으면 격리.
- PWA 퀵윈: maskable PNG 아이콘 + apple-touch-icon, Search Console verification.

### Phase 2 — 모임·행사 트랙 통합 (효과 大 / 공수 中) ★ 최대 공백
네트워킹 중심이라는 성격은 유지하되, "참여의 흔적"이 서비스에 남게 한다.
- 캘린더 통합: /calendar·.ics 소스에 networking_events 추가 (union에 "networking" 1종 추가).
- 리마인더: rsvpDeadline D-1·행사 D-1 크론 + 푸시/이메일 (세미나 크론 패턴 복제).
- 잔디·포인트: StreakEventType에 networking-attend 추가, 참석 확정 시 적립.
- 행사 후기: 세미나 review 패턴 이식(후기 요청 크론 포함) — 다음 행사 기획 데이터로 환류.
- "누구를 만났나": 행사 종료 후 참석자 목록(옵트인) → 명함 교환/DM 팔로업 → received_business_cards에 eventId 부여. 네트워킹 트랙의 본질 가치를 완성하는 연결.
- 사진첩에 networkingEventId 필드 추가.

### Phase 3 — "오늘 카드" + 소셜 루프 (효과 大 / 공수 中)
매일 들어올 이유를 첫 화면 한 곳에 압축.
- 오늘 카드 히어로: 플래시카드 due N장 · 이어쓰기(마지막 장·절) · 지도노트 미반영 N건 · 세미나/모임 D-day(내 RSVP 상태 포함) · 새 댓글/멘션. 전부 Phase 1~2에서 배선된 데이터 재사용.
- @멘션: 게시글·댓글·Q&A `@이름` 자동완성 + 알림(인앱/푸시). 재방문 트리거 최강 단일 기능.
- 리더보드 내 순위 칩, 프로필 조회수 타일, 주간 다이제스트에 동료 하이라이트 섹션.
- 진단 → 추천 배선: StageRecommendationPanel·onboarding CTA가 진단 약점을 반영하도록.

### Phase 4 — 지식 SSOT와 연구 여정 완성 (효과 中~大 / 공수 大)
- 문체 점검 규칙에 tipId 연결 → 이슈 카드에서 해당 writing-tips 카드로 딥링크(향후 규칙을 아카이브에서 생성).
- 에디터 통계 가이드의 archiveHref를 정확한 문서 id 딥링크로, 해석 포인트·참고문헌은 아카이브 문서에서 로드.
- 파인더 결과 영속화: 결과 URL 공유 + "이 방법을 내 연구문제에 태그하기" + 내부 논문 상세 페이지 링크.
- 연구모형 ↔ 논문 에디터 동기화(Sync 패널 이식), 계획서에 VariableSyncPanel 추가.
- 변인 어휘 연결: VariablesInput에 archive_variables 자동완성(자유 입력 병행).
- 읽기 기록 통합: paper_reading_logs와 research_papers 상호 참조(타이머 종료 시 서지 생성 제안).
- foundation-terms → 플래시카드 버튼, 아카이브 열람 잔디 적립(소액).
- collab 챕터 에디터에 문체점검(styleIssues) 적용, 개인 논문 ↔ 저널 투고 브리지.
- 참여 → 포트폴리오 자동 편입(프로필 자동 집계 재사용, verified 플로우 유지).
- 면담·지도노트 일정 → 캘린더/.ics.

### 성공 지표
북극성: 주간 활성 회원율(WAU/전체). 보조: D7 재방문, 오늘 카드 CTR, 모임 RSVP→후기 전환율, 멘션·댓글 수, 푸시 옵트인율. loyalty-snapshot 크론과 user_activity_logs로 전후 비교 측정 가능.
