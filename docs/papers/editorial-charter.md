## 1. AI 탐지 회피 (글쓰기지침 §1)

### 1.1. 문장 길이 편차
- 단문(15자 이하)을 본문에 주기적으로 삽입한다. "단정은 위험하다.", "방향은 분명하다." 류.
- 모든 문장이 50자 이상 장문이면 탐지 위험이 올라간다.

### 1.2. 서수 열거 금지 — 서론 본문
- 서론 본문에서 **"첫째, 둘째, 셋째, 넷째" 사용 0회**가 원칙이다.
- **유일한 예외**: 본문 말미의 "본 논문의 기여는 다음과 같다" 직후 정식 기여 진술. 학회지 관례.
- 대체 표현 풀: 우선 / 더 중요한 것은 / 또한 / 한편 / 무엇보다 / 이뿐만 아니라 / 이와 함께
- **현 final.md 위반 사항** — 서론에서 "첫째, 사이트는 단일 LMS가 아니라…", "둘째, 운영 주체가…", "셋째, 7개월의 PDCA 사이클…" 의 3중 열거가 발견됨. v3에서 자연어 흐름으로 풀어쓰기 필수.

### 1.3. 정형 접속 회피 (§1.6 + §12.5)
- "이는 ... 시사한다" 4회 이상 연속 금지.
- 종결문 풀: 보여 주는 결과이다 / 확인된다 / 재확인된다 / 뒷받침된다 / 관찰된다 / 의미한다 / 지지한다 / 입증한다 / 시사한다 / 암시한다 / 근거가 된다 / 단서가 된다.

### 1.4. 명사화 밀도 완화
- "정합성이 높다" → "맞물린다"
- "통합도가 가장 높다" → "가장 폭넓게 적용된다"
- 술어문을 혼입해 리듬을 조절한다.

### 1.5. 상투 표현 금지
| 금지 | 대체 |
|------|------|
| 이 같은 특성·맥락·구조 | 이런 속성·조건·구조, 또는 문장 재구성 |
| 이러한 ~을 고려할 때 | 이를 고려하면, 이 점을 고려하면 |
| 본 연구에서는 ... | 문장 재구성 (자기지시 절제) |
| ~에 반해 (연속 사용) | 한 문단 내 동일 접속 2회 이상 금지 |

## 2. 예고 문장 금지 (§2)

- "본 장 X절에서 다룬다", "후술한다", "뒤에서 상세히 논의할 것이다" 금지.
- 예외: 장(章) 도입부의 1문단(3~5문장) 개관문은 허용. 절(節) 이하 개관문은 금지.
- 절·항 번호를 괄호로 본문에 박지 않는다. "...(4.2 참조)" 식 표기는 figure·table 호명에 한정.

## 3. 단정문 출처 부착 — 학술적 톤 (§10.4 + §10.13 + 김수봉 §3)

### 3.1. 원칙
- 서론 첫 문단부터 결론 직전까지, **모든 단정형 문장은 인용·데이터·통계량 중 하나**를 동반한다.
- 본인의 해석·판단은 "~로 볼 수 있다", "~로 해석된다", "본 매트릭스는 ~로 읽힌다" 등 분석적 표현으로 전환한다.

### 3.2. 자가 점검 절차
- v3 작성 후 별도 패스로 **출처 없는 단정문 전수 점검**을 수행한다.
- 인용을 붙일 수 없는 단정문은 표현을 약화하거나 삭제한다.

### 3.3. 문단 오프닝은 인용부터 (§4.0)
- 핵심 문단의 첫 문장은 선행연구 인용으로 시작한다.
- 결과 섹션에서 사이트 디자인 결정 묘사 후, 반드시 이론적 출처를 1차 인용으로 묶는다.

## 4. 결론·논의 작성 원칙 (§12.2~§12.5, §12.9, §12.10)

### 4.1. 결론 명제의 3단 전개
- Ⅵ장(결론)은 3~5개의 거시 명제로 구성하며, 각 명제는 **(결과 진술 → 이론적 위치 → 실천적 함의)** 의 3단으로 서술한다.
- 명제 첫 문장은 굵게 처리하여 결과 요점을 단언한다.

### 4.2. 한계 ↔ 제언 일대일 대응
- 한계 N개와 제언 N개는 1:1로 짝지어 배치한다.
- 한계 N개를 모두 "후속 연구에서는 ~ 해야 한다" 패턴으로 끝내지 않도록 종결문을 다양화한다.

### 4.3. 결론의 자립성
- 결론은 요약의 재진술이 아니라, **이론적 위치 + 실천적 함의**를 새로 제출하는 입체적 진술이어야 한다.
- 결론에서만 출현하는 신개념(예: 본 연구가 명명하는 "이론 통합 디자인" 개념)을 명확히 한다.

### 4.4. 일반화 진술의 절제
- 본 연구는 단일 사례임을 명심하고, "모든 학회", "한국 학술 인프라 전반" 같은 진술은 피한다.
- 표집·사례 한계와 충돌하는 광범위 일반화 금지.

## 5. 사례연구·디자인 합리화 특화 원칙 (yonsei-edtech 맞춤)

### 5.1. Insider perspective 명시
- 저자가 운영진이라는 사실은 한계 섹션에서 한 번 진술하는 것을 넘어, **inter-rater(외부 평가자) 절차**로 보완 증거를 댄다.
- final.md는 이미 Cohen's κ = 0.78 진술을 포함하지만, **외부 평가자 신원·소속·평가 기간**의 구체 정보를 부록에 추가한다.

### 5.2. 매핑 강도 평가의 재현 가능성
- 4단계 rubric(●●● 핵심 / ●● 중요 / ● 부분 / ◐ 잠재)을 본문 §3.3에 명시한다.
- 부록 B의 매핑 검증 절차를 90개 셀 중 **임의 추출 표본 3~5개**의 실제 평가 과정으로 예시화한다.

### 5.3. 효과크기 명명 (§12.6) — 본 사례 맞춤
- 본 논문은 ANCOVA 효과크기가 아니므로 Cohen 기준은 직접 적용되지 않는다.
- 대신 매핑 매트릭스의 **행 평균(이론 통합도)·열 평균(도메인 이론 밀도)** 수치는 **소수점 둘째 자리까지 일관 표기**하고, "가장 높은 / 중간 / 가장 낮은" 명명은 행/열 평균의 분위(quartile) 기준으로 명시한다.

### 5.4. 표·본문 수치 정합성 (§12.7)
- §4.2 표 1·표 2의 수치(행 평균·열 평균·매핑된 도메인 수)와 §4.1 본문 묘사의 수치가 1:1로 일치하는지 점검.
- 점검 시트: `docs/papers/tables_check.csv` 작성 후 v4 빌드 전 대조.

### 5.5. 데이터 placeholder 정책 (§12.12)
- §4.4 운영 7개월 활동 데이터는 현재 `N/M/X%` 자리표시자 상태.
- v3에서 두 가지 경로 중 하나를 채택:
  - **경로 A (Real Data)**: Firestore 집계 스크립트 실행 → 실측치 삽입
  - **경로 B (Methodological Reframing)**: §4.4 절을 삭제하고, 운영 데이터를 본 논문 범위에서 제외함을 §3.2·§5.5 한계에서 명시. 매트릭스 검증은 코드·설계 문서·인터뷰의 3원천만으로 한정.
- 본 논문은 **경로 B를 1차 채택**한다. 이유: (a) 본 연구의 핵심 contribution은 매트릭스이지 효과 검증이 아님, (b) 운영 데이터 분석은 후속 6개 트랙의 과제로 이미 §5.6에 명시되어 있음. 경로 B 선택을 5.1 한계 절에서 한 문단으로 변론한다.

### 5.6. 인용 검증 — placeholder 잔존 차단
- final.md 참고문헌의 `[Placeholder — 저자가 RISS/KCI에서 검증 후 정확한 출처로 교체 필요]` 표기(Kim et al., 2017)는 **삭제 후 본문 인용도 함께 제거**하거나, RISS/KCI 검증 후 정확한 출처로 교체한다.

- 모든 인용은 APA7 형식으로 통일.
