# Plan: IA 개편 — 구성원 메뉴 분산 + GNB 제거 (ia-members-redistribution)

> **작성일**: 2026-05-05
> **PDCA 단계**: Plan
> **추정 작업량**: 4-6h
> **사용자 의도**: 회원 명단을 "사람 vs 도구" 청자에 맞게 분산하고 GNB를 단순화. 이동에 따른 라우팅·운영콘솔 회귀 방지.

---

## 1. 변경 요약

### 1.1 GNB 변경

**현재 (7개)**
```
학회소개 │ 구성원 │ 대학원 생활 │ 연구 활동 │ 학술 활동 │ 커뮤니티 │ 문의
```

**변경 후 (6개)**
```
학회소개 │ 대학원 생활 │ 연구 활동 │ 학술 활동 │ 커뮤니티 │ 문의
```

### 1.2 메뉴 항목 이동

| 기존 위치 | 항목 | 이동 후 |
|----------|------|--------|
| **구성원 → 학회소개** | 주임교수 | 학회소개 그룹 하위 |
| **구성원 → 학회소개** | 운영진 | 학회소개 그룹 하위 |
| **구성원 → 대학원 생활** | 재학생 회원 | 대학원 생활 그룹 하위 |
| **구성원 → 대학원 생활** | 졸업생 회원 | 대학원 생활 그룹 하위 |
| **구성원 자체** | (메뉴) | **제거** |

### 1.3 변경 후 그룹별 항목

```
학회소개:
  - 인사말
  - 학회 소개
  - 활동 분야
  - 연혁
  ─── (구분선) ───
  - 주임교수            ← 신규
  - 운영진              ← 신규

대학원 생활:
  ─── 학사 도구 ───
  - 인지디딤판
  - 내 수강과목
  - 캘린더
  ─── 회원 명단 ───
  - 재학생 회원        ← 신규
  - 졸업생 회원        ← 신규
```

> sub-section 헤더로 청자(학회 정보/사람) 시각 분리 — 인지부담 경감.

## 3. 사용자 의도 — "관련 기능 회귀 방지"

사용자 명시: "GNB 메뉴 개편과 함께 이동하는 메뉴들과 관련된 기능을 개선해달라 버그가 안나오게"

→ 의미: GNB 변경 자체로 회귀가 발생하지 않게 검증. 운영콘솔 등 회원 관련 기능은 그대로 작동해야 함.

### 회귀 검증 체크리스트
- [ ] `/members?tab=professor` 등 직접 URL 접근 정상
- [ ] 학회소개 드롭다운에서 주임교수 클릭 → /members?tab=professor 정상
- [ ] 대학원 생활 드롭다운에서 재학생 클릭 → /members?tab=student 정상
- [ ] 운영 콘솔(/console)의 회원 관리 기능 영향 없음
- [ ] 모바일 햄버거 메뉴에서도 동일 분산 표시
- [ ] sub-section 헤더가 dark mode에서도 가독성 유지

## 4. 결정 포인트

### Q1. sub-section 시각 분리 적용?
- **A. 적용 (권장)**: 학회소개에 "[학회 정보]" + "[주요 구성원]" / 대학원 생활에 "[학사 도구]" + "[회원 명단]" 헤더
- **B. 미적용**: 평탄 리스트 (단순)

### Q2. /members 페이지 자체 라우트 유지?
- **A. 유지 (권장)**: `/members?tab=*` 그대로 — 변경 위험 0
- **B. 분할 이전**: `/about/professors`, `/grad-life/students` 등으로 페이지 분할 — 큰 작업, 북마크 영향

### Q3. 학회소개에 "주임교수/운영진" 위치
- **A. 학회소개 끝 (권장)**: 인사말/학회소개/활동분야/연혁 → 주임교수/운영진 (학회 정보부터 보고 사람)
- **B. 학회소개 처음**: 주임교수/운영진 → 인사말/학회소개/... (사람부터)

### Q4. 모바일 햄버거 sub-section 표시
- **A. 적용**: 데스크톱과 동일한 sub-section 헤더 표시
- **B. 평탄**: 모바일은 sub-section 없이 평탄 (인지부담 줄이기)
