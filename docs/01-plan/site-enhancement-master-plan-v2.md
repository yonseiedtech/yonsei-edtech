# 연세교육공학회 사이트 고도화 마스터플랜 v2

> 작성: 2026-04-18 / 대상: yonsei-edtech (Next.js 16 + Firestore + bkend.ai + Vercel)

## 1. 비전

> **"강의는 LearnUs, 학자(學者)로서의 모든 것은 학회에서."**

대학원 LMS(LearnUs)는 강의 운영에 최적화되어 있다. 그러나 대학원생의 진짜 시간은 강의 밖에 있다 — 세미나·스터디·프로젝트·논문 큐레이션·인터뷰·대외 학술대회·공동연구·졸업 후 네트워킹. 본 사이트는 **현실(실험실/연구실/모임/사적 학습)과 LMS 사이의 중간 운영체제(Lab Life OS)** 로 자리잡는다.

핵심 포지셔닝 3축:
1. **LMS 보조** — 강의 일정/공지를 우리 사이트의 통합 캘린더에 일방향 수렴(ICS export 등). 시스템 연동은 불가하므로 **사용자가 한 번에 조회·기억할 수 있는 단일 표면**을 제공한다.
2. **학자 정체성의 Single Source of Truth** — 누구든 본인의 학자 활동(연구·학술·산출물·수상·대외활동)을 한 페이지에서 입증·재사용·공유할 수 있다.
3. **세대 간 자산화** — 재학생-졸업생 네트워크와 졸업생 논문/참고문헌이 후속 연구자의 문헌 탐색·멘토 매칭에 직접 활용된다.

## 2. 트랙 구조 (5 Track)

### Track 0 — 진행 중·완료된 작업의 정합화 (즉시)
- `bright-sparking-cerf.md` 마스터 로드맵의 단계 1·2 (board-community-v2 마무리, 학회보 PDF/매거진 뷰)는 본 v2와 병행 가능.
- 본 v2는 단계 3 이후의 우선순위를 **"학자 정체성·네트워킹·지식 그래프"** 축으로 재정렬한다.

### Track 1 — 학자 일상 캘린더 + 알림 (2주, P0)
**목표**: 강의·세미나·스터디·과제 마감·대외학술대회 마감을 한 화면에서 보고, 누락 없이 알림 받는다.
- 통합 캘린더 (`/calendar`): 세미나·활동·인터뷰·발표·논문 마감·외부 학술대회 임포트
- ICS export (`/api/calendar/ics?token=...`): 본인 토큰 기반 → Google Calendar/Apple/네이버에 1회 등록 후 자동 동기화 (일방향)
- 외부 학술대회 임포트: 운영진이 콘솔에서 등록 → 회원의 캘린더에 자동 표시
- Web Push (PWA) + Email(Resend) 듀얼 알림 — 사용자별 채널 on/off
- 카카오 알림톡은 V2 (단가·발신 프로필 등록 비용 검토 후)

### Track 2 — 개인 학술 포트폴리오 + 활동 깊이화 (3~4주, P0) ★ 본 문서 핵심
**목표**: 모든 활동이 "참여 여부"가 아니라 **"역할 + 산출물 + 성장 + 연계 수상"** 으로 누적되어, 졸업 시 자동으로 학술 포트폴리오 PDF가 나온다.
- 활동 참여 모델 확장: `ActivityParticipation` (role, outputs[], growthNotes, period)
- 수상 모델: `Award` (활동 연계 + 단독 수상 모두)
- **대외활동** (External Engagement) 모델: "연세대학교 교육대학원 교육공학 석사과정생" 신분으로 수행한 강연·기고·콘퍼런스·커뮤니티 활동 — 증빙 + 검증
- **콘텐츠 제작 이력** (ContentCreation): 인터뷰(Interviewer/Interviewee)·논문 큐레이션·뉴스레터 기고·블로그·영상
- 개인 상세 페이지 (`/profile/[id]`) 재설계:
  - 기본정보 / 학술활동(기존 4탭 유지·재가공) / **산출물·기여** / **수상** / **대외활동** / **콘텐츠** / 연구활동
  - 본인용 편집 모드 + 운영자 검증 워크플로
- PDF 출력 (`/profile/[id]/portfolio.pdf`): @react-pdf/renderer + Pretendard 임베드

→ 상세: `docs/01-plan/features/profile-portfolio-system.plan.md`

### Track 3 — 인적 네트워크 시각화 + 추천 매칭 (2주, P1)
**목표**: 재학생이 연구주제·관심분야·기수가 가까운 졸업생을 클릭 한번에 찾고, 졸업생은 후배 멘티를 만난다.
- **노드 엣지 모델**:
  - 노드: 회원(재학·휴학·졸업·교수)
  - 엣지: 같은 활동 참여 / 같은 세미나 / 공저 논문 / 멘토-멘티 / 같은 연구주제 키워드 / 인용 관계 (Track 4와 통합)
- **시각화**: react-force-graph (1차) — 1000명 미만이면 충분. 1000명 이상시 기수/주제 필터로 분할.
- **추천 매칭** (콜드: 키워드 cosine + 활동 자카드 / 웜: 공동 활동 가중치):
  - 재학생 → 추천 졸업생 멘토 5명
  - 졸업생 → 본인 연구분야 후배 후보
  - 운영진 콘솔: 매칭 수락률·실효성 통계
- **연락 채널**: 사이트 내 1:1 메시지 (V1) → 이메일 deep-link (V1.5)
- **6개 매칭 시그널 + "연세교육공학人" 브랜딩**: S1 관심분야 / S2 신분+학교급 / S3 같은 논문 읽기 / S4 졸업논문 참고문헌 겹침 / S5 과거 동수강 / S6 현재 동수강

→ 상세: `docs/01-plan/features/member-network-recommendation.plan.md`

### Track 4 — 졸업생 논문 DB + 참고문헌 매칭 추천 (3주, P1)
**목표**: "내 주제와 가까운 선배 논문"을 자동으로 찾고, 키워드·초록·참고문헌 매칭률로 정량화한다.

**시드 데이터 확보** (2026-04-18):
- `연세대_교육공학과_학위논문_134건.csv` 로 **2000년 8월 ~ 최신까지 논문 134편 즉시 적재 가능**.
- 컬럼: 번호 / 학위수여년월 / 논문저자 / 논문제목 / 지도교수 / 주제(키워드) / 실제URI(dCollection) / 초록 / 목차
- 가장 많은 지도교수: 이명근(59), 이현경(11), 김성완(10), 서순식(8), 김남주(8) — 학회 컨텍스트 핵심
- 누락: 지도교수 18건, 키워드 17건 → 운영진 보강 큐 필요

**졸업유형 구분** (사용자 요구):
- `graduationType: "thesis" | "research_report"` 두 가지
- CSV는 thesis 졸업만 포함. research_report 졸업자는 별도 수집/입력 (V1.5)

**회원-논문 매핑 전략** (학번/입학시점 데이터 부재):
- V1: 논문저자 이름 → 회원 매핑은 운영진 수동 (`/console/alumni-mapping` 큐). 미매핑 논문도 DB에는 적재 후 검색·매칭에 사용.
- V1.5: 동명이인 구분을 위해 본인 인증된 졸업생 회원이 본인의 학위논문을 "내 논문" 으로 클레임하는 UI 추가.
- V2: 학번/입학시점 데이터가 수집되면 자동 매핑 + 기수별 그래프 가능.

**참고문헌·매칭 분석**:
- V1: 키워드(`주제(키워드)`) + 초록 텍스트 키워드화(KOMORAN/Mecab nori) → 자카드 유사도. 초록만으로도 관심분야 매칭 80% 정확도 기대.
- V1.5: 사용자가 본인 논문 참고문헌 수동 입력 → 자카드 매칭률. Crossref/OpenAlex DOI 정규화.
- V2: PDF 자동 파싱(Grobid Cloud Run 배포). 임베딩 기반 추천(OpenAI text-embedding-3-small / KR-SBERT, Pinecone Free 또는 Firestore 벡터 검색).

**연관 기능**:
- 본인이 등록한 관심 키워드/연구주제 → 졸업생 논문 추천 카드 5개
- 논문 상세 페이지: 저자 프로필(매핑된 경우) + 같은 지도교수 논문 + 키워드 관련 논문
- 본 트랙은 **서지관리 도구(Zotero) 대체 아님** — 차별점은 "학회 회원 한정 + 인적 네트워크(Track 3) + 검증된 학과 데이터".

### Track 5 — 운영 효율화 마무리 (병렬 진행, P1)
- 수료증·감사장 PDF 자동 발급 + 일괄 이메일 (Track 2의 PDF 인프라 재사용)
- 회원 일괄 승인
- 회비 엑셀 자동 대조
- 인수인계 마크다운 + 기수 리포트 PDF

→ `bright-sparking-cerf.md` 단계 3과 동일하므로 **Track 2 PDF 인프라 완성 후 진입**.

## 4. 시스템·환경 제약 (Constraint Table)

| 영역 | 제약 | 대응 |
|---|---|---|
| LMS 연동 | 시스템 간 API 연동 불가 (전제) | 일방향 ICS export, 사용자 직접 등록 가이드 |
| 카카오 알림톡 | 발신 프로필 등록 + 단가 부담 | V2로 보류, V1은 Web Push + Resend |
| PDF 한글 폰트 | @react-pdf 기본 폰트는 한글 미지원 → 깨짐 | Pretendard / Noto Sans KR Variable 임베드 (라이선스 OFL) |
| Grobid (PDF 참고문헌 자동 추출) | Java + GPU 필요 → Cloud Run 별도 배포 (월 비용 발생) | V1은 수동 업로드 / DOI 입력. V2에서 Cloud Run + 호출량 한도 설정 |
| 그래프 시각화 성능 | 1000+ 노드 force-directed 렌더링 시 모바일 프레임 드랍 | 기수/주제 필터 강제 + 카드 뷰 fallback |
| 대외활동 진위 검증 | 본인 자가 입력 → 허위 입력 가능 | 증빙(URL/이미지/PDF) 첨부 필수 + 운영진 검증 큐 + verified 플래그 |
| Firestore 비용 | 그래프 매칭은 클라이언트 fetch 시 read 비용 폭증 | 매칭 결과를 staff trigger로 사전 계산 → `/cache/matches/{userId}` 저장 (TTL 24h) |
| 검색 색인 | Firestore 전문 검색 미지원 | V1: 클라이언트 in-memory 필터. V2 필요 시 Algolia 무료 티어 또는 Meilisearch |
| 졸업생 정보 갱신 | 졸업 후 연락 끊김 | 본인 회원가입 + alumni 초대 링크(이메일 발송) — V1.5 |
| 학번·입학시점 데이터 부재 | 134건 시드 CSV에 학번·입학기수 없음 | V1: 논문 DB는 적재만 + 저자명 매칭은 운영진 수동. V1.5: 본인 클레임 UI. V2: 학번 수집 후 자동 매핑·기수 그래프 |
| 논문 vs 연구보고서 졸업 구분 | 졸업유형 두 종류, CSV는 논문만 포함 | `graduationType` 필드 도입. 연구보고서 졸업자는 운영진 수동 입력 (V1.5) |
| 동명이인 매칭 오류 | 졸업생 이름만으로 회원 자동 매핑 시 오인 가능 | 운영진 검증 큐 + 본인 인증된 졸업생의 self-claim 워크플로 |
| Vercel Free 함수 시간 | PDF 렌더 + 외부 API 호출 시 10s 제한 | 포트폴리오 PDF는 클라이언트 측 React-PDF 렌더 + Blob 다운로드 (서버리스 함수 안 씀) |

## 5. 진행 규약

1. 각 트랙·Phase는 **PDCA 1사이클** (`/pdca plan` → `design` → `do` → `analyze`).
2. 데이터 모델 변경은 반드시 `src/types/index.ts` 단일 소스에 반영 + 마이그레이션 노트 기록.
3. **이미 구현된 기능 절대 재구현 금지** — 진입 전 grep 확인.
4. UI 변경 시 `/about/greeting` 의 `max-w-6xl px-4` 표준 + 통일 헤더 패턴(`h-12 w-12 rounded-xl bg-primary/10` + `text-2xl` 제목) 유지.
5. 배포는 항상 `npm run build` 통과 후 `npx vercel --prod` (GitHub Actions 사용 금지).
6. 검증: Track 2의 PDF 출력은 실제 파일 다운로드 + 한글 폰트 렌더링 검수 필수.
