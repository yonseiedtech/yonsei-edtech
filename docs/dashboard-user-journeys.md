# 대시보드 사용자 여정 시나리오 (참고 자료)

> 작성: 2026-06-13 · 목적: 대시보드 개선을 위한 사용자 여정 분석.
> 3영역 커맨드 센터(대학원생활·연구·학술) 위·아래 영역을 어떤 상황의 사용자가
> 무엇을 필요로 하는지에 근거해 개선하기 위한 기준 문서.

## 0. 전제 — 현재 대시보드 구조

```
[헤더: 인사·역할]
[JourneyGreetingHeader]  ← 위 영역 (인사 + 여정 단계 + 논문 진행률)
[3영역 커맨드 센터]       ← 핵심 (대학원생활·연구·학술 카드)  ✓ 좋다는 평가
[신규회원 배너/체크리스트]
[TermBriefHero]          ← 학기 진행 + 학사일정
[위젯 리스트]            ← 아래 영역 (할 일·타임라인·공지·게시판·세미나 등)
```

개선 포인트: **위 영역(인사~커맨드 센터 사이)** 과 **아래 영역(위젯)** 을
사용자의 학기·상황에 맞춰 더 "지금 나에게 필요한 것"으로 만든다.

## 1. 학기별 여정 (JOURNEY_STAGES 기반)

각 학기마다 연구뿐 아니라 **수업·네트워킹·학술활동**이 동시에 진행된다.
대시보드는 "이번 학기 핵심 + 영역별 다음 한 걸음"을 보여줘야 한다.

### 1학기 — 주제 탐색 (적응기)
- **상태/목표**: 입학 적응, 관심 분야 지형 파악, 선행 논문 읽기 습관, 수업 적응.
- **대시보드에서 필요한 것**: 신규 온보딩(체크리스트), 수업 시간표, 개강총회 등 첫 모임,
  논문 읽기 시작 유도, 아카이브 둘러보기.
- **현재 갭**: 신규 온보딩은 있으나, "아직 연구 단계가 아닌" 1학기생에게 논문 진행률 0%가
  부담으로 보일 수 있음. 읽기·네트워킹·수업 중심 안내가 더 적합.
- **개선**: 1학기는 커맨드 센터 아래 "이번 학기 추천" = 선행 논문 읽기 + 개강총회 참석 +
  관심 분야 아카이브 탐색.

### 2학기 — 주제 구체화
- **상태/목표**: 연구 문제 좁히기, 선행연구 체계적 정리, 변인·측정도구 탐색.
- **필요**: 읽기 리스트·분석 노트, 아카이브 개념↔변인↔측정도구, 연구 인터뷰(문제 진단).
- **갭**: 읽은 논문이 쌓이는데 "정리/구조화"로 넘어가는 다리가 약함.
- **개선**: "추천" = 분석 노트 작성, 변인 아카이브, 연구보고서 인터뷰 시작.

### 3학기 — 연구계획서 (종합시험)
- **상태/목표**: 종합시험 통과 + 연구 설계가 정당화된 계획서 완성.
- **필요**: 종합시험 카운트다운·자료, 계획서 에디터, 통계방법 가이드(설계 정당화),
  심사 위원 피드백(지도노트).
- **갭**: 종합시험 위젯은 있으나 기본 OFF. 3학기엔 최우선이어야.
- **개선**: 종합시험 D-day·자료를 상위 노출(상황 감지), 계획서→통계 가이드 동선.

### 4학기 — 데이터 수집·분석
- **상태/목표**: 자료 수집, 올바른 통계로 분석, 본문 1~3장 작성.
- **필요**: 통계방법 가이드(가정·SPSS/jamovi 따라하기·진단), 데이터 분석기, 본문 에디터 진행률.
- **갭**: 분석 도구·가이드가 아카이브에 풍부하나 대시보드에서 바로 닿기 어려움.
- **개선**: "추천" = 본문 작성 이어가기 + 통계 가이드 + 가정 검정 점검.

### 5학기 — 완성·심사 방어
- **상태/목표**: 4~5장 완성, 타당도 위협 방어 논리, 심사 준비.
- **필요**: 논문 심사 연습(따라읽기·STT), 지도노트 미반영 점검, 초록 작성, 진행률 마감.
- **갭**: 심사 연습·초록 기능은 있으나 마감 임박 사용자에게 집중 노출 안 됨.
- **개선**: "추천" = 심사 연습 + 초록 마무리 + 미반영 지도 해소. 진행률 강조.

## 2. 상황별 여정 (학기와 독립적으로 발생)

### A. 신규 가입 직후 (가입 30일 이내)
- 필요: 환영, 6단계 체크리스트, 프로필 완성, 첫 모임·첫 글.
- 현재: 신규 배너·체크리스트·미니멀 프리셋 존재 ✓ (잘 됨).
- 개선: 커맨드 센터의 빈 수치(0)를 "시작하기" CTA로 전환(예: 완독 0편 → "첫 논문 등록").

### B. 종합시험 시즌 (시험 D-30 이내)
- 필요: 종합시험 D-day·범위·자료를 최상단, 다른 위젯은 후순위.
- 개선: comprehensive_exam 일정 감지 시 커맨드 센터 위에 D-day 배너 승격.

### C. 논문 마감 임박 (제출·심사 D-30 이내, 또는 진행률 70%+)
- 필요: 본문 진행률·미반영 지도·심사 연습·초록을 집중 노출.
- 개선: 진행률/지도노트 신호로 "마감 모드" 감지 → 연구 카드 강조 + 추천에 심사·초록.

### D. 세미나·모임 시즌 (다가오는 행사 ≤7일)
- 필요: 임박한 세미나·모임 RSVP·회비를 눈에 띄게.
- 개선: 커맨드 센터 대학원생활/학술 카드에 "D-2 개강총회" 같은 임박 배지.

### E. 협업 연구 참여 중
- 필요: 공동연구 진행·할 일.
- 개선: 참여 중인 협업 연구가 있으면 학술/연구 카드에 표시.

### F. 방학 / 비활동기
- 필요: 부담 낮추고 "읽기·복습·다음 학기 준비" 권유.
- 개선: 수업 없는 기간 감지 시 추천을 읽기·복습 중심으로.

### G. 운영진 (staff+)
- 필요: 승인 대기·미답변 문의·행사 운영·회비 정산을 빠르게.
- 현재: StaffPriorityPanel 존재 ✓. 모임·네트워킹 운영(신규)도 연결 필요.

## 3. 도출된 개선 우선순위

| 우선 | 개선 | 근거 시나리오 | 작업량 |
|---|---|---|---|
| **High** | 커맨드 센터 아래 **"이번 학기 추천 한 걸음"** 스테이지별 패널 (1~5학기 + 상황) | 모든 학기 | 중 |
| **High** | 빈 수치(0)를 **시작 CTA**로 (완독 0 → "첫 논문 등록") | A 신규 | 소 |
| Medium | 임박 행사 **D-day 배지** (세미나·모임 ≤7일) | D | 소 |
| Medium | 종합시험/마감 **상황 감지 승격** | B·C | 중 |
| Low | 운영진 행사 운영 동선 보강 | G | 소 |

→ 본 사이클: **High 2건**(이번 학기 추천 패널 + 빈 수치 CTA)부터 구현.
나머지는 후속 사이클 또는 사용자 우선순위에 따라.
